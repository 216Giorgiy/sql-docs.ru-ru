# <a name="adding-and-modifying-data-sources-using-setup"></a>Добавление и изменение источников данных с помощью программы установки
> [!IMPORTANT]  
>  Этот компонент будет удален в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставляемого корпорацией Oracle.  
  
 Источник данных определяет путь к данным, можно включить сетевую библиотеку, сервера, базы данных и другие атрибуты — в этом случае источник данных — это путь к базе данных Oracle. Для подключения к источнику данных, диспетчер драйверов проверяет реестр Windows на наличие конкретных сведений о подключении.  
  
 Записи реестра, созданные по администратор источников данных ODBC используется драйверы диспетчера драйверов ODBC и ODBC. Эта запись содержит сведения о каждого источника данных и его соответствующий драйвер. Прежде чем подключиться к источнику данных, его сведения о соединении необходимо добавить в реестр.  
  
 Чтобы добавить и настроить источники данных, используйте [администратор источников данных ODBC](../../odbc/admin/odbc-data-source-administrator.md). Администратор ODBC обновляет сведения о подключении к данным. Добавление источников данных, администратор ODBC обновляет данные реестра для вас.  
  
### <a name="to-add-a-data-source-for-windows"></a>Добавление источника данных для Windows  
  
1.  Открытие администратора источников данных ODBC.  
  
2.  В диалоговом окне администратора источника данных ODBC нажмите кнопку "Добавить". Откроется диалоговое окно Создать новый источник данных.  
  
3.  Выберите Microsoft ODBC для Oracle и нажмите кнопку Готово. Откроется Microsoft ODBC для Oracle диалоговым окном.  
  
4.  В поле имя источника данных введите имя источника данных, которым требуется доступ. Он может быть любое выбранное имя.  
  
5.  В поле «Описание» введите описание для драйвера. Это необязательное поле Описание источника данных подключается к драйверу базы данных. Он может быть любое выбранное имя.  
  
6.  В поле имя пользователя введите имя пользователя базы данных (идентификатор пользователя базы данных).  
  
7.  В поле сервер введите псевдоним базы данных или строка для ядра сервера Oracle, который необходимо получить доступ к подключения.  
  
8.  Нажмите кнопку ОК, чтобы добавить этот источник данных.  
  
> [!NOTE]  
>  Откроется диалоговое окно источников данных, а администратор ODBC обновляет данные реестра. Пользователь имя и, указанный в строке подключения становятся значения подключения по умолчанию для этого источника данных при подключении к нему.  
  
1.  Нажмите кнопку Создать параметры дополнительные спецификации о драйвере ODBC для Oracle установки:  
  
    -   **Перевод** — нажмите кнопку Выбрать, чтобы выбрать загруженный транслятор данных. Значение по умолчанию — \<транслятор не используется >.  
  
    -   **Производительность** — включить примечания в функции работы с каталогами флажок определяет, возвращает ли драйвер столбцы примечания для [SQLColumns](../../odbc/microsoft/level-1-api-functions-odbc-driver-for-oracle.md) результирующего набора. Если этот флажок не установлен, драйвер ODBC для Oracle обеспечивает более быстрый доступ.  
  
         Включить СИНОНИМЫ из столбцов SQL флажок указывает ли драйвер возвращает сведения о столбце. **Размер буфера** указывает размер в байтах, выделенной для хранения извлекаемых данных. Процесс загрузки настраивается таким образом, одну загрузку сервер Oracle возвращал достаточно строк, чтобы заполнить буфер указанного размера. Большие значения, как правило, увеличивает быстродействие при загрузке большого объема данных.  
  
    -   **Настройка** — применять ODBC DayOfWeek стандарт флажок указывает ли результирующий набор будет соответствовать указанный день недели формат ODBC (воскресенье = 1; Суббота = 7). Если этот флажок снят, возвращается значение Oracle определенного языкового стандарта.  
  
         SQLDescribeCol **всегда возвращает значение для точности** флажок указывает, возвращать ли драйвер ненулевое значение для *cbColDef* аргумент **SQLDescribeCol**. Этот атрибут строки подключения применяется только к столбцам, где есть нет Oracle масштаба, например вычисляемые числовые столбцы и столбцы, определенные в виде ЧИСЛА без указания точности или масштаба. Объект **SQLDescribeCol** вызов возвращает 130 для точности при Oracle не предоставляет эту информацию. Если этот флажок установлен, драйвер возвращает 0 для этих типов столбцов вместо него.  
  
2.  Нажмите кнопку Добавить, чтобы добавить другой источник данных или нажмите кнопку Закрыть, чтобы выйти.  
  
### <a name="to-modify-a-data-source-for-windows"></a>Чтобы изменить источник данных для Windows  
  
1.  Открытие администратора источников данных ODBC. Щелкните на соответствующую вкладку DSN.  
  
2.  Выберите источник данных Oracle, который вы хотите изменить и нажмите кнопку настроить. Откроется Microsoft ODBC для Oracle диалоговым окном.  
  
3.  Изменить поля источника данных применимых и нажмите кнопку ОК.  
  
 Завершив внесение изменений в этом диалоговом администратора ODBC обновляет данные реестра.
