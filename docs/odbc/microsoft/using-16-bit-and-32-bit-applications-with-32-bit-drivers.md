# <a name="using-16-bit-and-32-bit-applications-with-32-bit-drivers"></a>Использование 16-разрядных и 32-разрядных приложений с 32-разрядными драйверами
> [!IMPORTANT]  
>  Поддержка 16-разрядных приложений будут удалены в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо разработки 32-разрядная или 64-разрядных приложений.  
  
 Компонент доступа к данным ODBC используется 16-разрядных и 32-разрядных приложений с 32-разрядными драйверами. Операционные системы Microsoft® Windows® 95/98 и Microsoft Windows или Windows 2000 поддерживают следующие сочетания приложения и драйверы:  
  
-   16-разрядных приложений с 32-разрядными драйверами  
  
-   32-разрядных приложений с 32-разрядными драйверами  
  
 Использование 32-разрядное приложение с 16-разрядный драйвер не поддерживается.  
  
> [!NOTE]  
>  Начиная с выпуска ODBC версии 3.0, Windows NT 4.0 поддерживается.  
  
 ODBC включает ODBC компоненты, необходимые для поддержки приведенных выше конфигураций, «создание» библиотеки динамической компоновки (DLL) для преобразования 16-разрядные адреса в 32-разрядные адреса и наоборот. Программа установки определяет, какая операционная система используется и устанавливает ODBC компонентов, необходимых для этой системы. Можно также установить компоненты ODBC, используемые всех систем.  
  
 В большинстве случаев переноса приложения или драйвера с 16-разрядной на 32-разрядные включает пять типов изменений:  
  
-   Изменения в коде обработки сообщений  
  
-   Изменения из-за целыми числами и обрабатывает 32 бита  
  
-   Изменения в вызовы Windows API-интерфейсов (API)  
  
-   Изменения драйвер потоками  
  
-   При изменении компонентов ODBC  
  
 Из приложения или драйвера программирования точки зрения между 16-разрядных и 32-разрядных компонентов ODBC основное отличие заключается в том, что они имеют различные имена. С точки зрения системы архитектура для каждого приложения или драйвера подключения и средства, используемые для управления источниками данных отличаются.
