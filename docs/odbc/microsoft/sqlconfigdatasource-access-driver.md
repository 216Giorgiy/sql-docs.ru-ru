# <a name="sqlconfigdatasource-access-driver"></a>SQLConfigDataSource (драйвер доступа)
> [!NOTE]  
>  В этом разделе сведения драйвера доступа. Общие сведения об этой функции см. в соответствующем разделе [Справочник по API ODBC](../../odbc/reference/syntax/odbc-api-reference.md).  
  
 **SQLConfigDataSource** функцию, которая будет использоваться для добавления, изменения или удаления источника данных динамически используются следующие ключевые слова.  
  
|Ключевое слово|Description|  
|-------------|-----------------|  
|COLLATINGSEQUENCE|Последовательность, в которой отсортированы поля.<br /><br /> Таким образом задается один и тот же параметр как **упорядоченную последовательность** в диалоговом окне программы установки.|  
|COMPACT_DB|Сжатие данных на файл базы данных. Имеет следующий формат: COMPACT_DB = < путь >< optionaL_sort_order >\<Необязательное ключевое слово ENCRYPT >.<br /><br /> При использовании ключевого слова COMPACT_DB в одной инструкции с помощью ключевого слова DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом сжатие базы данных и указав имя источника данных состоит из двух этапов.|  
|КЛЮЧЕВОЕ СЛОВО CREATE_DB КАК|Создает файл базы данных. Имеет следующий формат: ключевое слово CREATE_DB как = < путь >\<порядок optional_sort >< optional_ENCRYPT ключевое слово >, где имя пути — полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в диалоговом окне Новая база данных отображается при нажатии кнопки создания в диалоговом окне Установка Microsoft Access. Если порядок сортировки не указан, используется Общие.<br /><br /> При использовании ключевого слова ключевое слово CREATE_DB как в одной инструкции с помощью ключевого слова DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом Создание базы данных и указав имя источника данных состоит из двух этапов. Если при использовании ключевое слово ключевое слово CREATE_DB как пути создания базы данных Microsoft Access содержит один или несколько пробелов, затем полное имя пути должны заключаться в двойные кавычки, как показано в следующих примерах:<br /><br /> «C:\PROGRAM FILES\COMMON FILES\ MyAccess.mdb»<br /><br /> «C:\PROGRAM FILES\Access2.mdb»<br /><br /> CREATE_DB=C:\TEMP\test.mdb (без кавычек требуется)|  
|CREATE_SYSDB|Создает файл базы данных системы. Имеет следующий формат: CREATE_SYSDB =\<путь name >\<необязательно порядок сортировки >, где имя пути — полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в **новую базу данных** диалоговое окно отображается, когда **создать** кнопки в **Установка драйвера ODBC для Microsoft Access** диалоговое окно. Если порядок сортировки не указан, используется Общие.|  
|CREATE_V2DB|Создает файл базы данных, совместимый с Microsoft Access 2.0. Имеет следующий формат: CREATE_V2DB =\<путь name >\<необязательно порядок сортировки >, где имя пути — полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в диалоговом окне Новая база данных отображается при нажатии кнопки создания в диалоговом окне Установка Microsoft Access. Если порядок сортировки не указан, используется Общие.<br /><br /> При использовании ключевого слова CREATE_V2DB в одной инструкции с помощью ключевого слова DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом Создание базы данных и указав имя источника данных состоит из двух этапов.<br /><br /> При использовании ключевого слова CREATE_V2DB, если пути создания базы данных Microsoft Access содержит один или несколько пробелов, затем полное имя пути должны заключаться в двойные кавычки, как показано в следующих примерах:<br /><br /> «C:\PROGRAM FILES\COMMON FILES\ MyAccess.mdb»<br /><br /> «C:\PROGRAM FILES\Access2.mdb»<br /><br /> CREATE_V2DB=C:\TEMP\test.mdb (без кавычек требуется)|  
|DBQ|Имя файла базы данных.<br /><br /> Таким образом задается один и тот же параметр как **базы данных** в диалоговом окне программы установки.|  
|ЕГО ЗНАЧЕНИЯ|Строка пути к файлу базы данных.|  
|DESCRIPTION|Описание данных в источнике данных.<br /><br /> Таким образом задается один и тот же параметр как **описание** в диалоговом окне программы установки.|  
|DRIVER|Строка пути к DLL-Библиотека драйвера.|  
|DRIVERID|Целочисленный идентификатор для драйвера.  25 (Microsoft Access)|  
|FIL|Тип доступа по протоколу MS для Microsoft Access файла|  
|IMPLICITCOMMITSYNC|Определяет ли драйвер Microsoft Access выполнения фиксации внутренней или неявные асинхронно. Это значение изначально установлено значение «Да», это означает, что драйвер Microsoft Access ждет фиксаций в внутренней/неявной транзакции должны завершаться.<br /><br /> Значение этого параметра не должны изменяться без тщательного анализа последствий. Дополнительные сведения об этом параметре см. в разделе *Microsoft Jet базы данных подсистемы Руководство программиста*.<br /><br /> Таким образом задается один и тот же параметр как **ImplicitCommitSync** в диалоговом окне программы установки.|  
|MAXBUFFERSIZE|Размер внутреннего буфера, в килобайтах, используемый для передачи данных на диск и обратно в Microsoft Access. Размер буфера по умолчанию — 2048 КБ (отображается как 2048). Можно использовать любое целочисленное значение, кратным 256. Таким образом задается один и тот же параметр как **размер буфера** в диалоговом окне программы установки.|  
|MAXSCANROWS|Число строк для просмотра при задании типа данных столбца на основе существующих данных.<br /><br /> Можно ввести число от 1 до 16 для строк для сканирования. По умолчанию используется значение 8. Если свойство имеет значение 0, просматриваются все строки. (Число за пределами ограничение будет возвращена ошибка.)<br /><br /> Таким образом задается один и тот же параметр как **строк для просмотра** в диалоговом окне программы установки.|  
|PAGETIMEOUT|Указывает период времени, в миллисекундах, которое страницы (если не используется), остаются в буфере до удаления. Значение по умолчанию — 5-десятые доли секунды (0,5 секунд). Обратите внимание, что этот параметр применяется ко всем источникам данных, использующие драйвер ODBC.<br /><br /> Таким образом задается один и тот же параметр как **время ожидания страницы** в диалоговом окне программы установки.|  
|PWD|Пароль.|  
|READONLY|Значение TRUE, чтобы сделать файл доступным только для чтения; Значение FALSE, чтобы сделать файл не только для чтения.<br /><br /> Таким образом задается один и тот же параметр как **только для чтения** в диалоговом окне программы установки.|  
|REPAIR_DB|Восстановление базы данных повреждены, ошибки, возникшей во время процесса фиксации.<br /><br /> При использовании ключевого слова REPAIR_DB в одной инструкции с помощью ключевого слова DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом восстановление базы данных и указав имя источника данных состоит из двух этапов.|  
|SYSTEMDB|Для драйвера Microsoft Access, пути к файлу базы данных системы.<br /><br /> Таким образом задается один и тот же параметр как **системной базы данных** в диалоговом окне программы установки.|  
|ПОТОКИ|Число фоновых потоков, используемых механизмом. Это значение по умолчанию — 3, но могут быть изменены.<br /><br /> Таким образом задается один и тот же параметр как **потоков** в диалоговом окне программы установки.|  
|UID|Для драйвера Microsoft Access имя идентификатора пользователя, используемый для входа.|  
|USERCOMMITSYNC|Определяет, является ли драйвером Microsoft Access выполнения пользовательские транзакции асинхронно. Это значение изначально установлено значение «Да», это означает, что драйвер Microsoft Access будет ожидать передачи в пользовательской транзакции должны завершаться.<br /><br /> Значение этого параметра не должны изменяться без тщательного анализа последствий. Дополнительные сведения об этом параметре см. в разделе *Microsoft Jet базы данных подсистемы Руководство программиста*.<br /><br /> Таким образом задается один и тот же параметр как **UserCommitSync** в диалоговом окне программы установки.|
