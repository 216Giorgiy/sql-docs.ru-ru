# <a name="applications"></a>Приложения
*Приложения* — программа, которая вызывает API ODBC для доступа к данным. Несмотря на то, что возможны несколько типов приложений, большинство делятся на три категории, которые используются в качестве примеров в этом руководстве.  
  
-   **Универсальные приложения** их также называют упакованной со сжатием или готовых приложениях. Универсальные приложения предназначены для работы со множеством различных DBMS. Примеры включают электронной таблицы или пакет статистики, который использует технологию ODBC для импорта данных для дальнейшего анализа и текстового процессора, который использует технологию ODBC для получения списка рассылки из базы данных.  
  
     Важные разновидность универсальные приложения — сред разработки приложения, например PowerBuilder или Microsoft Visual Basic®. Несмотря на то, что приложения, созданные с помощью этих сред, скорее всего, будет работать только с одной СУБД, самой среде требуется работать с несколькими СУБД.  
  
     То, что все универсальные приложения имеют общий — что они поддерживают возможность взаимодействия между СУБД и необходимые для работы с ODBC относительно универсальным способом. Дополнительные сведения о совместимости см. в разделе [Выбор уровня взаимодействия](../../odbc/reference/develop-app/choosing-a-level-of-interoperability.md).  
  
-   **Вертикальные приложения** вертикальные приложения выполнение одного типа задачи, например ввода заказа или производственные данные, отслеживания и работа со схемой базы данных, которая определяется разработчиком приложения. Для конкретного клиента что приложение работает с одной СУБД. Например малого бизнеса могут использовать приложение с dBase, пока крупные предприятия могут использовать ее с Oracle.  
  
     Приложение использует ODBC таким образом, что приложение не привязан к любой один СУБД, несмотря на то, что может привязываться к ограниченному числу СУБД, обеспечивающие аналогичную функциональность. Таким образом разработчик приложения может продавать приложения независимо от СУБД. Вертикальные приложения могут взаимодействовать, если они разрабатываются, но иногда изменено для включения noninteroperable кода, когда пользователь выбрал СУБД.  
  
-   **Пользовательские приложения** Custom applications используются для выполнения определенных задач в одной организации. Например приложение в большой компании может собирать данные о продажах из нескольких разделов (каждый из которых использует различные СУБД) и создать единый отчет. ODBC используется, так как он представляет собой общий интерфейс и сохраняет программистов от необходимости обучения нескольких интерфейсов. Такие приложения обычно не являются функционально совместимыми и записываются в конкретных СУБД и драйверы.  
  
 Число задач являются общими для всех приложений, независимо от того, как они используют ODBC. Вместе они во многом определяют поток любое приложение ODBC. Следующие задачи:  
  
-   Выбор источника данных и подключение к нему.  
  
-   Идет отправка инструкцию SQL для выполнения.  
  
-   Получение результатов (если таковые имеются).  
  
-   Обработка ошибок.  
  
-   Фиксация или откат транзакции, содержащего инструкцию SQL.  
  
-   Отключение от источника данных.  
  
 Поскольку большинство данных доступа осуществляются с помощью SQL, основная задача, для которой приложения используют ODBC является отправлять инструкции SQL и получение результатов (если таковые имеются) при выполнении этих инструкций. Другие задачи, для которых приложения используют ODBC включают определение и настройка для возможности драйверов и просмотра каталога базы данных.
