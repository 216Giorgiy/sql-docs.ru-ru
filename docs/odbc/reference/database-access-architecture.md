# <a name="database-access-architecture"></a>Архитектура доступа к базе данных
Один из вопросов в разработке ODBC была какая часть архитектуры доступа к базе данных для стандартизации. Программные интерфейсы, описанные в предыдущем разделе SQL — embedded SQL, SQL модули и CLI — входят только один такой архитектуры. На самом деле поскольку ODBC был в основном предназначен для подключения приложения на основе персонального компьютера миникомпьютер и мэйнфрейма СУБД, было также количество сетевых компонентов, некоторые из которых может стандартизован.
