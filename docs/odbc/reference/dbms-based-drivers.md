# <a name="dbms-based-drivers"></a>Драйверов на основе DBMS
Драйверов на основе СУБД используются с источниками данных, например Oracle или SQL Server, предоставляющие изолированный компонент database engine для использования драйвера. Эти драйверы физических данных обращаться из автономного обработчика. то есть они отправлять инструкции SQL и получить результаты из ядра.  
  
 Поскольку драйверов на основе DBMS использовать существующий компонент database engine, они являются обычно легче писать, чем драйверов на основе файлов. Несмотря на то, что драйвер на основе СУБД может быть легко реализован перевод в вызовы ODBC для собственного вызовы API, в результате медленнее драйвера. Для реализации на основе DBMS драйвер лучше использовать базовый протокол потока данных, которой обычно является назначение собственных API. Например драйвер SQL Server следует использовать TDS (поток данных протокола для SQL Server), вместо того чтобы DB-Library (собственный API для SQL Server). Исключением из этого правила является ODBC собственного API. Например автономный обработчик, который находится на том же компьютере, что и приложение и загружается непосредственно как драйвер является Watcom SQL.  
  
 Драйверов на основе СУБД выступает в качестве клиента в конфигурации клиента и сервера где источник данных выступает в роли сервера. В большинстве случаев клиента (драйвер) и сервера (источник данных) находиться на разных компьютерах, несмотря на то, что оба может находиться на том же компьютере, в многозадачной операционной системой. Третья возможность *шлюза,* которой располагается между драйвером и источника данных. Шлюз — это программное обеспечение, которое вызывает один СУБД должен выглядеть как другой. Например приложения, написанные для использования SQL Server можно также доступ к данным DB2 через шлюз DB2 Micro Decisionware; Этот продукт в результате DB2, как SQL Server.  
  
 На следующем рисунке три различные конфигурации для драйверов на основе DBMS. В первой конфигурации драйвера и источник данных находятся на том же компьютере. Во втором драйвер и источник данных находятся на разных компьютерах. В третьем драйвер и источник данных находятся на разных компьютерах и шлюз располагается между ними, размещенный на компьютере еще один.
