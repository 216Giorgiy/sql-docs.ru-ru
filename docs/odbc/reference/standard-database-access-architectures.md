# <a name="standard-database-access-architectures"></a>Стандартные архитектуры доступа к базе данных
Изучить компоненты доступа базы данных, описанные в предыдущем разделе, оказывается, два из них — программирование интерфейсы и протоколы потока данных — хорошо подходят для стандартизации. Другие два компонента — механизма межпроцессного Взаимодействия и сетевых протоколов — не только находятся слишком низкого уровня, но оба сильно зависит от сети и операционной системы. Также имеется третий подход - шлюзы -, которая предоставляет возможности для стандартизации.  
  
 Этот раздел содержит следующие подразделы.  
  
-   [Стандартный интерфейс программирования](../../odbc/reference/standard-programming-interface.md)  
  
-   [Стандартный протокол потока данных](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [Стандартный шлюз](../../odbc/reference/standard-gateway.md)
