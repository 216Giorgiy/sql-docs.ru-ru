# <a name="determining-the-target-dbmss-and-drivers"></a>Определение целевой СУБД и драйверов
Следующий вопрос, которые следует учитывать, каковы целевой СУБД для приложения, и какие драйверы станут доступными, поддерживает эти СУБД? Поскольку универсальные приложения, как правило, поддерживает возможность взаимодействия, лучше всего подходит для пользовательских и вертикальные приложения на вопрос: целевой СУБД. Тем не менее вопрос драйверов целевого применяется ко всем приложениям, так как драйверы существенно различаться по скорости, качества, поддержка функций и доступности. Кроме того Если драйверы должны распространяться вместе с приложением, стоимость и доступности планов лицензирования необходимо учитывать.  
  
 Для многих пользовательских приложений, очевидны целевой СУБД: они представляют существующие СУБД, которые приложение предназначено для доступа. Следует также рассматривать СУБД, к которым планируется будущей миграции. Основной вопрос для этих приложений, то, какой драйвер или драйверы для использования с ними. Для других пользовательских приложений — те, которые не предназначены для доступа к СУБД с существующие — можно выбрать на основе поддержку функций, поддержка параллельных пользователей, доступности драйверов и ценовой целевой СУБД.  
  
 Для вертикальной приложений целевой СУБД обычно выбираются на основе поддержку функций, доступности драйверов и рынка. Например вертикальное приложение, предназначенное для малого бизнеса должны предназначаться для СУБД, доступной для этих бизнеса; вертикальные приложения предназначены как расширение в существующих СУБД должны быть предназначены широко используемые СУБД.  
  
 При выборе целевой СУБД, следует рассматривать различия между настольных компьютеров и серверов баз данных. Настольных баз данных, например dBASE, Paradox и Btrieve имеют меньшую мощность сервера баз данных. Так как обычно они доступны через меньшую мощность ядер SQL в наиболее на основе файлов драйверов, они часто не поддерживают полную транзакцию, поддерживать меньшее число одновременных пользователей и имеют ограниченный SQL. Тем не менее они недорогих и имеют множеству.  
  
 Сервер баз данных, например Oracle, DB2 и SQL Server предоставляют полную поддержку транзакции и поддерживать несколько одновременных пользователей имеют широкие возможности SQL. Они гораздо более затратным и имеют меньшие установленной базы. С другой стороны цены программного обеспечения, как правило, будет больше, немного смещение меньшего размера потенциальный рынок.  
  
 Таким образом целевой СУБД иногда может быть указано в функции, необходимые для приложения и приложения целевого рынка. Например, система для крупных корпораций может не рабочего стола базу данных-получатель, так как они не поддерживают достаточно транзакции. Большинство баз данных на основе стоимости исключить аналогичные система, предназначенная для малого бизнеса. И могут ориентироваться и разработчикам универсальных приложений, но старайтесь не использовать расширенные функции из базы данных сервера.
