# <a name="auto-commit-mode"></a>Режим автоматической фиксации
*В режиме автоматической фиксации* каждой операции базы данных является транзакцией, которая фиксируется при выполнении. Этот режим подходит для многих реальных транзакций, состоящих из одной инструкции SQL. Нет необходимости разделять или указать завершения этих транзакций. В базах данных без поддержки транзакций режим автоматической фиксации имеет единственный поддерживаемый режим. В таких базах данных инструкции фиксируются, когда они выполняются, и нет возможности отката; они таким образом всегда находятся в режиме автоматической фиксации.  
  
 Если базовой СУБД не поддерживает транзакции режим автоматической фиксации, драйвер эмуляции их путем вручную фиксации каждой инструкции SQL, как он выполняется.  
  
 Если пакет инструкций SQL выполняется в режиме автоматической фиксации, это зависящее от источника данных, когда инструкции в пакете не фиксируются. Их можно будет зафиксировать их по или после выполнения всего пакета. Некоторые источники данных могут поддерживать оба эти варианты поведения, а также может служить способ выбора одного или других. В частности при возникновении ошибки в середине пакета, нет, определяемые источником данных использует инструкции уже выполнена фиксации или отката. Таким образом взаимодействующие приложения, которые используют разделы и требуется фиксация или откат всей выполнять пакеты только в режиме ручной фиксации.
