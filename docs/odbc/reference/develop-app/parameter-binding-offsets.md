# <a name="parameter-binding-offsets"></a>Привязка параметров смещения
Приложение может указать, что смещение добавляется связанный параметр буфера адресов и соответствующие длины/индикатора буфера адреса при **SQLExecDirect** или **SQLExecute** вызывается. Результат этих дополнений определяет адреса, используемые в этих операциях.  
  
 Привязка смещения позволяют приложению изменить привязки без вызова **SQLBindParameter** для ранее привязанных параметров. Вызов **SQLBindParameter** повторно привязать параметр изменяет адрес буфера и указатель длины/индикатора. Повторная привязка со смещением, с другой стороны, просто добавляет смещение к существующий адрес буфера привязанного параметра и адрес буфера длины/индикатора. При использовании смещения, привязки «шаблон» как располагаются буферы приложения и приложения можно перемещать «шаблон» к различным областям памяти, изменение смещения. Новое смещение может быть указан в любое время и всегда добавляется изначально связанных значений.  
  
 Чтобы указать смещение привязки, приложение устанавливает атрибут инструкции SQL_ATTR_PARAM_BIND_OFFSET_PTR адрес буфера SQLINTEGER. Прежде чем приложение вызывает функцию, которая использует привязки, она помещает их смещение в байтах в этот буфер, пока параметр адрес буфера ни адрес буфера длины/индикатора 0, который привязанного параметра в инструкции SQL. Сумма адрес и смещение должен быть допустимый адрес. (Это означает, что один или оба смещение и адрес, к которому добавляется смещение может быть недопустимым, при условии, что их сумма является допустимым адресом).  
  
> [!NOTE]  
>  Привязка смещения не поддерживаются ODBC 2. *x* драйверы.
