# <a name="retrieving-results-advanced"></a>Извлечение результатов (Дополнительно)
Приложение может указать, что смещение добавляется связанный адреса буфер данных и соответствующие длины/индикатора буфера адреса при **SQLBulkOperations**, **SQLFetch**,  **SQLFetchScroll**, или **SQLSetPos** вызывается. Результаты этих дополнений определения адреса, используемые в этих операциях.  
  
 Привязка смещения позволяют приложению изменить привязки без вызова **SQLBindCol** ранее связанных столбцов. Вызов **SQLBindCol** для повторной привязки данных изменяет адрес буфера и указатель длины/индикатора. Повторная привязка со смещением, с другой стороны, просто добавляет смещение к существующий адрес буфера привязанного к данным и адрес буфера длины/индикатора. При использовании смещения, привязки «шаблон» как располагаются буферы приложения и приложения можно перемещать «шаблон» к различным областям памяти, изменение смещения. Новое смещение может быть указан в любое время и всегда добавляется изначально связанных значений.  
  
 Чтобы указать смещение привязки, приложение устанавливает атрибут инструкции SQL_ATTR_ROW_BIND_OFFSET_PTR адрес буфера SQLINTEGER. Прежде чем приложение вызывает функцию, которая использует привязки, такие как **SQLBulkOperations**, **SQLFetch**, **SQLFetchScroll**, или **SQLSetPos**, он помещает смещение в байтах в этом буфере, при условии, что адрес буфера данных ни адрес буфера длины/индикатора равно 0, а также при условии, что связанный столбец содержится в результирующем наборе. Сумма адрес и смещение должен быть допустимый адрес. (Это означает, что один или оба смещение и адрес, к которому добавляется смещение может быть недопустимым, при условии, что их сумма является допустимым адресом). Атрибут инструкции SQL_ATTR_ROW_BIND_OFFSET_PTR является указателем, чтобы значение смещения можно применять к более одного набора привязки данных, которые можно изменить, изменив одно значение смещения. Приложению необходимо убедиться, что указатель остается действительным, пока курсор не будет закрыт.  
  
> [!NOTE]  
>  Привязка смещения не поддерживаются ODBC 2. *x* драйверы.
