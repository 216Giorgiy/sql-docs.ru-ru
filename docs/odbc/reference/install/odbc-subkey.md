# <a name="odbc-subkey"></a>Подраздел ODBC
Параметры трассировки ODBC определенные значения в подразделе ODBC. Эти параметры задаются через «трассировка» диалогового окна Администратор источников данных ODBC, отображаемого элементом **SQLManageDataSources**. Подраздел ODBC является необязательным. Формат этих значений является, как показано в следующей таблице.  
  
|Имя|Тип данных|data|  
|----------|---------------|----------|  
|Trace|REG_SZ|**0** &#124; **1**|  
|TraceFile|REG_SZ|*путь файла трассировки*|  
  
 Значения имеют значения, описанные в следующей таблице.  
  
|Значение|Значение|  
|-----------|-------------|  
|Trace|Если трассировка имеет значение 1, если приложение вызывает **SQLAllocHandle** с параметром SQL_HANDLE_ENV трассировка включена для вызывающего приложения.<br /><br /> Если ключевое слово трассировки имеет значение 0, когда приложение вызывает **SQLAllocHandle** с параметром SQL_HANDLE_ENV трассировка отключена для вызывающего приложения. Это значение по умолчанию.<br /><br /> Приложение можно включить или отключить трассировку с помощью атрибута соединения SQL_ATTR_TRACE. Однако делать так не изменить данные для этого значения.|  
|TraceFile|Если трассировка включена, диспетчер драйверов записывает в файл трассировки, заданное в параметре файла трассировки.<br /><br /> Если файл трассировки не указан, диспетчер драйверов записывает файл Sql.log на текущем диске. Это значение по умолчанию.<br /><br /> Трассировку можно использовать только для одного приложения или каждое приложение должно указать файл трассировки другой. В противном случае два или более приложений будет пытаться открыть один и тот же файл трассировки одновременно, что вызывает ошибку.<br /><br /> Приложение может указать новый файл трассировки с помощью атрибута соединения SQL_ATTR_TRACEFILE. Однако делать так не изменить данные для этого значения.|  
  
 Например предположим, что трассировка включена, и файл трассировки — C:\Odbc.log. Значения в подразделе ODBC будет выглядеть следующим образом:  
  
```  
Trace : REG_SZ : 1  
TraceFile : REG_SZ : C:\ODBC.LOG  
  
```
