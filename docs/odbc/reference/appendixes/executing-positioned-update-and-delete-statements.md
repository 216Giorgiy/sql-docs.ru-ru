# <a name="executing-positioned-update-and-delete-statements"></a>Выполнение позиционированных обновлений и инструкций Delete
> [!IMPORTANT]  
>  Этот компонент будет удален в будущих версиях Windows. Избегайте использования этой возможности в новых разработках и запланируйте изменение приложений, которые сейчас ее используют. Корпорация Майкрософт рекомендует использовать функциональность курсора драйвера.  
  
 После как приложение получило блока данных с **SQLFetchScroll**, его можно обновить или удалить данные в блоке. Для выполнения позиционированного обновления или удаления, приложение:  
  
1.  Вызовы **SQLSetPos** для позиционирования курсора на строке, чтобы обновить или удалить.  
  
2.  Создает позиционированного обновления или инструкции delete с помощью следующего синтаксиса:  
  
     **ОБНОВЛЕНИЕ** *имя таблицы*  
  
     **ЗАДАТЬ** *идентификатор столбца* **=** {*выражение* &#124; **NULL**}  
  
     [**,** *идентификатор столбца* **=** {*выражение* &#124; **NULL**}]  
  
     **WHERE CURRENT OF** *имя курсора*  
  
     **УДАЛИТЬ из** *имя таблицы* **WHERE CURRENT OF** *имя курсора*  
  
     Самый простой способ создания **ЗАДАТЬ** предложением инструкции позиционированного обновления — использовать маркеры параметров для каждого столбца и использовать **SQLBindParameter** привязать эти буферы строк для на обновляемую строку. В этом случае тип данных C параметра будет таким же, как тип данных C буфера строк.  
  
3.  Обновляет буферы строк для текущей строки, если она будет выполнена инструкция позиционированного обновления. После успешного выполнения инструкция позиционированного обновления, библиотеку курсоров в свой кэш копирует значения из каждого столбца в текущей строке.  
  
    > [!CAUTION]  
    >  Если приложение не обновляется должным образом буферы строк перед выполнением инструкции позиционированного обновления, данные в кэше могут оказаться неверными после выполнения инструкции.  
  
4.  Выполняет позиционированного обновления или с помощью другой инструкции, чем инструкция, связанная с курсор инструкции delete.  
  
    > [!CAUTION]  
    >  **ГДЕ** предложение, создаваемых библиотеку курсоров для идентификации текущей строки не удается определить все строки, определить другую строку или определить более одной строки. Дополнительные сведения см. в разделе [построение инструкции поиск](../../../odbc/reference/appendixes/constructing-searched-statements.md).  
  
 Все располагается обновления и инструкций delete требуют имя курсора. Чтобы указать имя курсора, приложение вызывает **SQLSetCursorName** перед открытием курсора. Чтобы использовать имя курсора, сформированное драйвером, приложение вызывает **SQLGetCursorName** после открытия курсора.  
  
 После курсора библиотека выполняет позиционированного обновления или инструкции delete, массив состояния, буферы строк и кэш, который поддерживается библиотекой курсоров содержит значения, указанные в следующей таблице.  
  
|Используется инструкция|Значение в массив состояния строк|Значения в<br /><br /> буферы строк|Значения в<br /><br /> буферы кэша|  
|--------------------|-------------------------------|----------------------------------|---------------------------------|  
|Позиционированное обновление|SQL_ROW_UPDATED|Новые значения [1]|Новые значения [1]|  
|Позиционированные delete|SQL_ROW_DELETED|Старые значения|Старые значения|  
  
 [1] приложения необходимо обновить значения в буферы строк перед выполнением инструкции позиционированного обновления; После выполнения инструкции позиционированного обновления, библиотеку курсоров копирует значения в буферы строк в свой кэш.
