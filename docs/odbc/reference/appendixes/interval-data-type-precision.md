# <a name="interval-data-type-precision"></a>Точность интервального типа данных
Точность для типа данных interval включает интервала, точность, точности интервала и точностью.  
  
 В начале поле Интервал является числа со знаками. Максимальное число цифр для ведущих поля определяется количество вызывается *точности, интервала* который является частью объявления типа данных. Например, объявление: HOUR(5) ИНТЕРВАЛ до минуты имеет точность 5; интервала из поля ЧАСА может принимать значения от –99999 до 99999. В поле SQL_DESC_DATETIME_INTERVAL_PRECISION запись дескриптора содержится главного параметра точности интервала.  
  
 Список полей, тип интервала данных состоит из называется *точности интервала*. Это не числовое значение, как может подразумеваться, термин «точность». Например, интервал точность типа INTERVAL DAY TO во-ВТОРЫХ — это список день, час, минута, секунда. Нет поля дескриптора, который содержит это значение; Точность интервал всегда может определяться по типу данных интервала.  
  
 Любой тип данных интервал, имеющий ВТОРОЕ поле имеет *высокую точность секунд*. Это количество цифр дробной части, допустимое в дробной части секунд значения. Это отличается от для других типов данных, где точность определяет количество цифр перед десятичной запятой. Секунды значения типа в тип данных интервал — это количество цифр после десятичной запятой. К примеру Если точность секунды, равным 6, номер 123456 в поле дробной части будет интерпретироваться как.123456 и номер 1230 будет интерпретироваться как.001230. Для других типов данных это называется масштабирования. Точность секунды интервала содержится в поле SQL_DESC_PRECISION дескриптора. Если точность долей секунды в значении значение интервала SQL больше, чем что могут сохраняться в структуре интервала C, он драйвера — определен ли значение долей секунды в интервале SQL округлением или усечены при преобразовании в C Структура интервала.  
  
 Если поле SQL_DESC_CONCISE_TYPE имеет значение к типу данных интервал, поле SQL_DESC_TYPE присваивается SQL_INTERVAL и SQL_DESC_DATETIME_INTERVAL_CODE задан код для типа данных. Поле SQL_DESC_DATETIME_INTERVAL_PRECISION автоматически присваивается точность начальные интервал по умолчанию 2, а поле SQL_DESC_PRECISION автоматически присваивается точность секунд интервал по умолчанию 6. Если любой из этих параметров не подходит, приложение должно явным образом задать поля дескриптора посредством вызова **SQLSetDescField**.
