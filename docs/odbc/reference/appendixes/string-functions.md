# <a name="string-functions"></a>Строковые функции
В следующей таблице перечислены функции обработки строк. Приложение может определить, какие строковые функции поддерживаются драйвером путем вызова **SQLGetInfo** с *типу информации* из SQL_STRING_FUNCTIONS.  
  
## <a name="remarks"></a>Remarks  
 Аргументы обозначается как *строковое_выражение* может быть именем столбца, *символ строковый литерал*, или результат другой скалярной функцией, где базовый тип данных может быть представлено как SQL_CHAR, SQL_ VARCHAR или SQL_LONGVARCHAR.  
  
 Аргументы обозначается как *character_exp* — это строка символов переменной длины.  
  
 Аргументы обозначается как *запустить*, *длина*, или *число* может быть *числовой литерал* или результата другого скалярной функцией, где базовый тип данных может быть представлено как SQL_TINYINT, SQL_SMALLINT или SQL_INTEGER.  
  
 Строковые функции, перечисленные здесь начинаются с 1; Первый символ в строке является символ 1.  
  
 BIT_LENGTH, CHAR_LENGTH, CHARACTER_LENGTH, OCTET_LENGTH и ПОЗИЦИИ строки скалярные функции были добавлены в ODBC версии 3.0 в соответствии со стандартом SQL-92.  
  
|Компонент|Description|  
|--------------|-----------------|  
|**ASCII (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает код ASCII первого символа указанного *строковое_выражение* как целое число.|  
|**BIT_LENGTH (** *строковое_выражение* **)** (ODBC 3.0)|Возвращает длину строкового выражения в битах.<br /><br /> Работает только для строковых типов данных, поэтому будет не неявно преобразовать *строковое_выражение* со строкой, а возвращается (внутренний) размер любого типа данных, ей присваивается.|  
|**CHAR (** *кода* **)** (ODBC 1.0)|Возвращает значение, заданное параметром кода с символом, имеющим ASCII *кода*. Значение *кода* должен находиться в диапазоне от 0 до 255; в противном случае возвращает значение зависит от источника данных.|  
|**CHAR_LENGTH (** *строковое_выражение* **)** (ODBC 3.0)|Возвращает длину в символах строкового выражения, если строковое выражение символьного типа данных; в противном случае возвращает длину в байтах (наименьшее целое число не меньше, чем число битов, разделенное на 8) строкового выражения. (Эта функция является таким же, как функция CHARACTER_LENGTH.)|  
|**CHARACTER_LENGTH (** *строковое_выражение* **)** (ODBC 3.0)|Возвращает длину в символах строкового выражения, если строковое выражение символьного типа данных; в противном случае возвращает длину в байтах (наименьшее целое число не меньше, чем число битов, разделенное на 8) строкового выражения. (Эта функция является таким же, как функция CHAR_LENGTH.)|  
|**CONCAT (** *строковое_выражение1*,*строковое_выражение2***)** (ODBC 1.0)|Возвращает символьную строку, которая является результатом объединения *строковое_выражение2* для *строковое_выражение1*. Полученная в результате строка зависит от СУБД. Например, если столбец, представленный *строковое_выражение1* содержится значение NULL, DB2 и возвращаемое значение NULL, но SQL Server возвратит НЕПУСТАЯ строка.|  
|**РАЗНИЦА (** *строковое_выражение1*,*строковое_выражение2***)** (ODBC 2.0)|Возвращает целочисленное значение, указывающее разницу между значениями, возвращаемому функцией SOUNDEX для *строковое_выражение1* и *строковое_выражение2*.|  
|**Вставка (** *строковое_выражение1*, *запустить*, *длина*, *строковое_выражение2***)** (ODBC 1.0)|Возвращает символ строки where *длина* символы будут удалены из *строковое_выражение1*, начиная с *запустить*и где *строковое_выражение2* была вставлена в *строковое_выражение,* начиная *запустить*.|  
|**LCASE (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает строку, в *строковое_выражение*, все прописные символы преобразованы в нижний регистр.|  
|**LEFT (** *строковое_выражение*, *число***)** (ODBC 1.0)|Возвращает самое левое *число* символов *строковое_выражение*.|  
|**Длина (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает количество символов в *строковое_выражение,* исключая конечные пробелы.<br /><br /> **Длина** принимает только строки. Таким образом, неявно преобразует *строковое_выражение* строку и возвращают длина этой строки (не внутренний размер типа данных).|  
|**НАЙДИТЕ (** *строковое_выражение1*, *строковое_выражение2*[, *запустить*]**)** (ODBC 1.0)|Возвращает начальную позицию первого вхождения *строковое_выражение1* в *строковое_выражение2*. Поиск первого вхождения *строковое_выражение1* начинается с позиции первого знака в *строковое_выражение2* Если необязательный аргумент *запустить*, указан. Если *запустить* указан, поиск начинается с позиции знака значение *запустить*. Позиция первого символа в *строковое_выражение2* определяется значением 1. Если *строковое_выражение1* не найдено в *строковое_выражение2*, возвращается значение 0.<br /><br /> Если приложение может вызвать скалярные функции поиск с *строковое_выражение1*, *строковое_выражение2*, и *запустить* аргументы, драйвер возвращает SQL_FN_STR_LOCATE при  **SQLGetInfo** вызывается с *параметр* из SQL_STRING_FUNCTIONS. Если приложение может вызвать скалярные функции поиск только с *строковое_выражение1* и *строковое_выражение2* аргументы, драйвер возвращает SQL_FN_STR_LOCATE_2 при **SQLGetInfo**вызывается с *параметр* из SQL_STRING_FUNCTIONS. Драйверы, которые поддерживают вызова функции поиск с двух или трех аргументов возвращают SQL_FN_STR_LOCATE и SQL_FN_STR_LOCATE_2.|  
|**LTRIM (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает символов *строковое_выражение*, удалены пробелы в начале.|  
|**OCTET_LENGTH (** *строковое_выражение* **)** (ODBC 3.0)|Возвращает длину строкового выражения в байтах. Результатом является наименьшее целочисленное значение, не меньшее, чем число битов, разделенное на 8.<br /><br /> Работает только для строковых типов данных, поэтому будет не неявно преобразовать *строковое_выражение* со строкой, а возвращается (внутренний) размер любого типа данных, ей присваивается.|  
|**ПОЛОЖЕНИЕ (** *character_exp* **в** *character_exp***)** (ODBC 3.0)|Возвращает позицию первого символа выражения во втором выражении символ. Результатом является точное числовое значение с масштабом 0 и точностью, определяемого реализацией.|  
|**ПОВТОРИТЕ (** *строковое_выражение,* *число***)** (ODBC 1.0)|Возвращает строку символов, состоящие из *строковое_выражение* повторяется *число* раз.|  
|**ЗАМЕНИТЕ (** *строковое_выражение1*, *строковое_выражение2*, *string_exp3***)** (ODBC 1.0)|Поиск *строковое_выражение1* foroccurrences из *строковое_выражение2*и замените *string_exp3*.|  
|**СПРАВА (** *строковое_выражение*, *число***)** (ODBC 1.0)|Возвращает самое правое *число* символов *строковое_выражение*.|  
|**RTRIM (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает символов *строковое_выражение* с замыкающими пробелами удалены.|  
|**Функция SOUNDEX (** *строковое_выражение* **)** (ODBC 2.0)|Возвращает строку символов зависит от источника данных, представляющий звук слов в *строковое_выражение*. Например SQL Server возвращает код SOUNDEX 4 цифры; Oracle возвращает фонетические представление каждого слова.|  
|**ПРОБЕЛ (** *число* **)** (ODBC 2.0)|Возвращает символьную строку, состоящую из *число* пробелы.|  
|**SUBSTRING (** *строковое_выражение*, *запустить*, длина**)** (ODBC 1.0)|Возвращает строку символов, который является производным от *строковое_выражение*, начиная с позиции, указанной параметром *запустить* для *длина* символов.|  
|**UCASE (** *строковое_выражение* **)** (ODBC 1.0)|Возвращает строку, в *строковое_выражение*, со всеми нижний регистр, переведенную в верхний регистр.|
