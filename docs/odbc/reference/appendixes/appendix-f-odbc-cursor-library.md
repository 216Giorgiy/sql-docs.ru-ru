# <a name="appendix-f-odbc-cursor-library"></a>Приложение е. библиотека курсоров ODBC
> [!IMPORTANT]  
>  Этот компонент будет удален в будущих версиях Windows. Избегайте использования этой возможности в новых разработках и запланируйте изменение приложений, которые сейчас ее используют. Корпорация Майкрософт рекомендует использовать функциональность курсора драйвера.  
  
 Библиотека курсоров ODBC (Odbccr32.dll) поддерживает прокручиваемые курсоры блок для любой драйвер, соответствующий уровню соответствия API уровня 1 и может распространяться разработчиками приложений и драйверов. Библиотека курсоров также поддерживает позиционированные обновления и инструкций delete для результирующих наборов, созданных **ВЫБЕРИТЕ** инструкции. Несмотря на то, что он поддерживает только статические и однонаправленные курсоры, библиотеку курсоров удовлетворяет требованиям многих приложений. Кроме того он может предоставлять высокую производительность, особенно полезен для небольшого размера и средних результирующие наборы и для приложений, которые не поддерживает хорошо курсора.  
  
 Библиотека курсоров является библиотеки динамической компоновки (DLL), расположенный между диспетчером драйверов и драйвер. Когда приложение вызывает функцию, диспетчер драйверов вызывает функцию в библиотеку курсора, которая выполняет функцию или вызывает ее в указанный драйвер. Для данного соединения приложение указывает ли библиотека курсоров всегда используется, если драйвер не поддерживает прокручиваемые курсоры или никогда не используется.  
  
 Библиотека курсоров отображается как драйвер диспетчера драйверов. Если библиотека курсоров находится между диспетчером драйверов и ODBC 2. *x* драйверов, библиотеку курсоров отображается как ODBC 2. *x* драйвера. Если библиотека курсоров находится между диспетчером драйверов и ODBC 3*.x* драйверов, библиотеку курсоров отображается как ODBC 3*.x* драйвера. Библиотека курсоров поведения зависит от версии драйвер, который работает с, за исключением смещения привязки, которая поддерживается для обоих ODBC 2. *x* и ODBC 3. *x* драйверы.  
  
 Для реализации блочных курсоров в **SQLFetch** и **SQLFetchScroll**, многократно вызывает библиотеку курсоров **SQLFetch** в драйвере. Для реализации возможности прокрутки, он кэширует данные, извлеченные в памяти и файлы на диске. Когда приложение запрашивает новый набор строк, библиотеку курсоров извлекает его при необходимости из драйвера или из кэша.  
  
 Для реализации позиционированного обновления и удаления, создает библиотеку курсоров **обновление** или **удаление** инструкции с **ГДЕ** предложение, определяющее, кэшированные значение каждого привязанного столбца в строке. При выполнении инструкции позиционированного обновления, библиотеку курсоров обновляет свой кэш от значений в буферы строк.
