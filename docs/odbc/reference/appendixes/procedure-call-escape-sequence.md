# <a name="procedure-call-escape-sequence"></a>Escape-последовательность вызова процедуры
ODBC использует escape-последовательности для вызовов процедур. Ниже приведен синтаксис escape-последовательности:  
  
 **{**[? =]**вызвать** *имя процедуры*[**(**[*параметр*] [, [*параметр*]] ... **)**]**}**  
  
 В форме БЭКУСА-Наура используется следующий синтаксис:  
  
 *ODBC процедуры escape* :: =  
  
 &#124; *ODBC-esc инициатор* [? =] вызова *процедуры ODBC esc признака конца.*  
  
 *процедура* :: = *имя процедуры* &#124; *имя процедуры* (*список параметров процедуры*)  
  
 *Идентификатор процедуры* :: = *определенное имя пользователя*  
  
 *Имя процедуры* :: = *идентификатор процедуры*  
  
 &#124; *имя владельца*. *Идентификатор процедуры*  
  
 &#124; *разделитель каталога имя каталога* *идентификатор процедуры*  
  
 &#124; *разделитель каталога имя каталога* [*имя владельца*]. *Идентификатор процедуры*  
  
 (Третий синтаксис допустим, только в том случае, если источник данных не поддерживает владельцев).  
  
 *Имя владельца* :: = *определенное имя пользователя*  
  
 *Имя каталога* :: = *определенное имя пользователя*  
  
 *разделитель каталога* :: = {*определяемого реализацией*}  
  
 (Разделитель каталога возвращается через **SQLGetInfo** с параметром сведения SQL_CATALOG_NAME_SEPARATOR.)  
  
 *список параметров процедуры* :: = *параметр процедуры*  
  
 &#124; *параметр процедуры*, *список параметров процедуры*  
  
 *параметр процедуры* :: = *динамический параметр* &#124; *литерала* &#124; *пустой строкой*  
  
 *пустой строки* :: =  
  
 *ODBC-esc инициатор* :: = {}  
  
 *ODBC esc признак конца* :: =}  
  
 (Если параметр процедуры является пустая строка, в процедуре используется значение по умолчанию для этого параметра.)  
  
 Чтобы определить источник данных поддерживает процедур и драйвер поддерживает синтаксис вызова процедуры ODBC, приложение может вызвать **SQLGetInfo** с типом SQL_PROCEDURES сведения.
