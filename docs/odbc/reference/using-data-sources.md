# <a name="using-data-sources"></a>Использование источников данных
Источники данных обычно создаются конечным пользователем или специалист в программе вызывается *администратор ODBC*. Администратор ODBC пользователю драйвер для использования, а затем вызывает этого драйвера. Драйвер отображает диалоговое окно, который запрашивает сведения, требуемые для подключения к источнику данных. После ввода пользователем данные, драйвер сохраняет его в системе.  
  
 Позже приложение вызывает диспетчера драйверов и передает его имя источника данных или путь файла, содержащего файл источника данных. При передаче имени источника данных компьютера, диспетчер драйверов ищет системе, чтобы найти драйвер, используемый источником данных. Затем будет загружен драйвер и передает ей имя источника данных. Драйвер использует имя источника данных, чтобы найти сведения, требуемые для подключения к источнику данных. Наконец он подключается к источнику данных, обычно запросила идентификатор пользователя и пароль, которые обычно не хранятся.  
  
 При передаче источника данных, диспетчер драйверов открывает файл и загружает указанный драйвер. Если файл также содержит строку соединения, он передает этот драйвер. Используя сведения в строку подключения, драйвер подключается к источнику данных. Если строка подключения не были переданы, драйвер обычно пользователю необходимые сведения.
