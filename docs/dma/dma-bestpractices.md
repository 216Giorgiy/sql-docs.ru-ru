# <a name="best-practices-for-running-data-migration-assistant"></a>Рекомендации по выполнению помощник по миграции данных
В этой статье содержатся следующие рекомендации для установки, оценки и миграции.

## <a name="installation"></a>Установка
Не установите и запустите помощник по миграции данных непосредственно на хост-компьютере SQL Server.

## <a name="assessment"></a>Оценка
- Запустите оценок в производственных базах данных во время минимальной нагрузки.
- Выполнить **проблемы совместимости** и **новые функции рекомендации** оценки отдельно, чтобы уменьшить длительность оценки.

## <a name="migration"></a>Миграция
- Миграция сервера во время минимальной нагрузки.
- При переносе базы данных, укажите одну общую папку расположении, доступном с исходного сервера и на целевом сервере и по возможности избегайте операции копирования. Операция копирования может вызвать задержку в зависимости от размера файла резервной копии. Операция копирования увеличивается вероятность того, что миграция завершится ошибкой из-за выполнить дополнительное действие. Если указано централизованно, помощник по миграции данных обходит операции копирования.
 
    Кроме того убедитесь, чтобы предоставить необходимые разрешения для общей папки, чтобы избежать сбоев при миграции. В средстве указаны правильные разрешения. Если экземпляр SQL Server выполняется учетными данными, сетевой службы, предоставьте правильные разрешения для общей папки на учетную запись компьютера для экземпляра SQL Server.

- Включить шифрование соединения при подключении к исходным и целевым серверами. С помощью протокола SSL шифрование повышает безопасность передачи данных по сетям между помощник по миграции данных и экземпляр SQL Server, который будет полезно, особенно в том случае, если перенос имен входа SQL Server. Если не используется шифрование SSL и сети нарушена злоумышленником, переносимым имена входа SQL, могут получить перехвачены и/или изменить на лету злоумышленником.

    Однако если весь доступ осуществляется по безопасной внутренней сети, то шифрование не требуется. Включение шифрования снижает производительность, так как дополнительные издержки, требуется для шифрования и расшифровки пакетов. Дополнительные сведения см. в [шифрование соединений с SQL Server](https://go.microsoft.com/fwlink/?linkid=832513).
