# <a name="connect-to-an-access-data-source-sql-server-import-and-export-wizard"></a>Подключения к источнику данных Access (мастер экспорта и импорта SQL Server)
В этом разделе показано, как подключиться к **Microsoft Access** из источника данных **выберите источник данных** или **Выбор назначения** страницы мастера экспорта и импорта SQL Server.

На следующем снимке экрана показан пример подключения к базе данных Microsoft Access. В этом примере нет необходимости вводить имя пользователя и пароль, поскольку целевой базы данных не использует файл рабочей группы.

![Подключение к Access](../../integration-services/import-export-data/media/connect-to-access.jpg)

## <a name="options-to-specify"></a>Параметры для указания

> [!NOTE]
> Параметры соединения для поставщика данных одинаковы ли доступа к источнику или к назначению. То есть варианты, вы увидите одинаковы на обоих **выберите источник данных** и **Выбор назначения** страницах мастера.

**Источник данных**  
Список поставщиков данных может содержать несколько записей для Microsoft Access. Выберите последнюю установленную версию или версию, соответствующую версии доступа, который создал файл базы данных.

|Источник данных|Версия Office|
|-------|-------|
|Microsoft Access (Microsoft.ACE.OLEDB.16.0)|Office 2016|
|Microsoft Access (Microsoft.ACE.OLEDB.15.0)|Office 2013|
|Microsoft Access (ядро СУБД Microsoft Access)|Office 2010 и Office 2007|
|Microsoft Access (базы данных Microsoft Jet)|Office версий более ранних, чем Office 2007|

> [!IMPORTANT]
> Может потребоваться загрузка и установка дополнительных файлов для подключения к базам данных Access. В разделе [получить файлы, необходимые для подключения к доступа](#officeDownloads) на этой странице Дополнительные сведения.

 **Имя файла**  
Укажите путь и имя файла. Например **C:\\MyData.mdb** для файла на локальном компьютере или  **\\ \\продажи\\базы данных\\Northwind.mdb** для файла в общей сетевой папке. или нажмите **Обзор**. 

 >   [!NOTE] 
 > При нажатии кнопки **Обзор** для поиска файла доступа **откройте** фильтры поле диалогового окна для файлов с более старой версии. MDB формат и расширением файла по умолчанию. Однако поставщик данных позволяет открывать файлы с помощью более новой версии. ACCDB формат и расширением файла.
  
 **Обзор**  
 Выберите файл базы данных с помощью диалогового окна **Открыть**.  
  
 **Имя пользователя**  
Если файл рабочей группы связан с базой данных, укажите допустимое имя пользователя.  
  
 **Пароль**  
Если файл рабочей группы связан с базой данных, укажите здесь пароль пользователя.
 
Если база данных защищена одним для всех пользователей паролем, см. раздел [является файл базы данных защищен паролем?](#database_password).
  
 **Дополнительно**  
Укажите дополнительные параметры, например пароль базы данных или информационный файл рабочей группы не по умолчанию, в **свойства связи данных** диалоговое окно.  

## <a name="i-dont-see-access-in-the-list-of-data-sources"></a>Я не вижу доступа в списке источников данных
Если вы не видите доступа в списке источников данных, то выполняется 64-разрядный мастер? Список поставщиков для Excel и Access обычно 32-разрядные и не отображаются в 64-разрядный мастер. Запустите 32-разрядный мастер.

> [!NOTE]
> Чтобы использовать 64-разрядной версии мастера экспорта и импорта SQL Server, необходимо установить SQL Server. SQL Server Data Tools (SSDT) и SQL Server Management Studio (SSMS), 32-разрядных приложений и установить только 32-разрядных файлов, включая 32-разрядную версию мастера.

## <a name="officeDownloads"></a>Получить файлы, необходимые для подключения к доступа  
Может потребоваться скачать сетевые компоненты для источников данных Microsoft Office Access и Excel, в том числе, если они еще не установлены. Загрузите последнюю версию компонентов подключения для Access и Excel файлов здесь: [доступа к базе данных ядро 2016 распространяемый компонент Microsoft](https://www.microsoft.com/download/details.aspx?id=54920).
  
Последнюю версию компонентов позволяет открывать файлы, созданные в предыдущих версиях доступ.

Если компьютер имеет 32-разрядной версии Office, необходимо установить 32-разрядную версию компонентов, а также необходимо гарантировать выполнение пакета в 32-разрядном режиме.

Если у вас есть подписка Office 365, убедитесь, что вы загрузите 2016 распространяемый доступа базы данных ядра и не 2016 среды выполнения Microsoft Access. При запуске программы установки может появиться сообщение об ошибке, что невозможно установить загрузки side-by-side работай компоненты Microsoft Office. Чтобы обойти это сообщение об ошибке, запустите установку в автоматическом режиме, открыв окно командной строки и выполнение. EXE-файла, загруженного с `/quiet` переключения. Например:

`C:\Users\<user name>\Downloads\AccessDatabaseEngine.exe /quiet`

## <a name="database_password"></a>Файл базы данных защищен паролем?
В некоторых случаях база данных Access защищена паролем, но не использует файл рабочей группы. Все пользователи должны предоставить тот же пароль, но не требуется вводить имя пользователя. Чтобы указать пароль базы данных, выполните следующие действия.

1.  На **выберите источник данных** или **Выбор назначения** щелкните **Дополнительно** кнопку, чтобы открыть **свойства связи данных** диалоговое окно.  
2.  В **свойства связи данных** выберите **все** вкладки.  
3.  В списке свойств и значений, выберите **OLEDB:Database пароль Jet**.   
    
    ![Укажите пароль для доступа к, экран 1](../../integration-services/import-export-data/media/specify-access-password-screen-1.jpg) 
4.  Нажмите кнопку **изменить значение** Открытие **изменение значения свойства** диалоговое окно.  
    
    ![Укажите пароль для доступа к, экран 2](../../integration-services/import-export-data/media/specify-access-password-screen-2.jpg)
5.  В **изменение значения свойства** диалогового окна введите пароль базы данных.
6.  Нажмите кнопку **ОК** в соответствующем диалоговом окне, чтобы вернуться к **выберите источник данных** или **Выбор назначения** мастера и продолжить.

## <a name="keep-your-autonumber-values-when-you-export-from-access"></a>Сохранять значения счетчика, при экспорте из Access
Чтобы разрешить значения идентификаторов, существующие в исходных данных должны быть вставлены в столбец идентификаторов целевой таблицы, выберите **Разрешить вставку в столбец идентификаторов** в диалоговом окне **сопоставления столбцов** диалоговое окно. По умолчанию для целевого столбца идентификаторов обычно не позволяет вставлять существующие значения. Чтобы показать **сопоставления столбцов** выберите **изменить сопоставления** по достижении **Выбор исходных таблиц и представлений** мастера. Посмотрите на этих страницах, в разделе [Выбор исходных таблиц и представлений](../../integration-services/import-export-data/select-source-tables-and-views-sql-server-import-and-export-wizard.md) и [сопоставления столбцов](../../integration-services/import-export-data/column-mappings-sql-server-import-and-export-wizard.md).

Если существующие первичные ключи находятся в столбце идентификаторов, столбце автоматической нумерации или аналогичном, нужно выбрать этот параметр, чтобы сохранить такие ключи. В противном случае целевому столбцу идентификаторов обычно присваиваются новые значения.
