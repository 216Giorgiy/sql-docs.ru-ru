# <a name="connect-to-an-excel-data-source-sql-server-import-and-export-wizard"></a>Подключения к источнику данных Excel (мастер экспорта и импорта SQL Server)
В этом разделе показано, как подключиться к **Microsoft Excel** из источника данных **выберите источник данных** или **Выбор назначения** страницы мастера экспорта и импорта SQL Server.

На следующем снимке экрана показан пример подключения к книге Microsoft Excel.

![Подключение через Excel](../../integration-services/import-export-data/media/excel-connection.png) 

## <a name="options-to-specify"></a>Параметры для указания

> [!NOTE]
> Параметры соединения для поставщика данных являются одинаковыми как для Excel является источнике или месте назначения. То есть варианты, вы увидите одинаковы на обоих **выберите источник данных** и **Выбор назначения** страницах мастера.

**Путь к файлу Excel**  
 Укажите путь и имя файла Excel. Например:
-   Для файла на локальном компьютере **C:\\MyData.xlsx**.
-   Для файла в общей сетевой папке  **\\ \\продажи\\базы данных\\Northwind.xlsx**.

или нажмите **Обзор**.  
  
 **Обзор**  
 Выберите электронную таблицу с помощью диалогового окна **Открыть**.  

> [!NOTE]
> Мастер не может открыть защищенный паролем файл Excel.

 **Версия Excel**  
Выберите версию Excel для исходной рабочей книги.

> [!IMPORTANT]
> Может потребоваться загрузка и установка дополнительных файлов для подключения к файлам Excel. В разделе [получить файлы, необходимые для подключения к Excel](#officeDownloads) на этой странице Дополнительные сведения.

**Первая строка содержит имена столбцов**  
Укажите, содержит ли первая строка данных имена столбцов.
-   Если данные не содержит имена столбцов, но этот параметр включен, мастер рассматривает первую строку из источника данных с именами столбцов.
-   Если данные содержат имена столбцов, но отключить этот параметр, мастер считает первая строка данных имена столбцов в строке.

Если указать, что данные не указаны имена столбцов, мастер использует F1, F2 и т. д., как заголовки столбцов.

## <a name="i-dont-see-excel-in-the-list-of-data-sources"></a>Excel не отображается в списке источников данных
Если вы не видите Excel в списке источников данных, то выполняется 64-разрядный мастер? Список поставщиков для Excel и Access обычно 32-разрядные и не отображаются в 64-разрядный мастер. Запустите 32-разрядный мастер.

> [!NOTE]
> Чтобы использовать 64-разрядной версии мастера экспорта и импорта SQL Server, необходимо установить SQL Server. SQL Server Data Tools (SSDT) и SQL Server Management Studio (SSMS), 32-разрядных приложений и установить только 32-разрядных файлов, включая 32-разрядную версию мастера.

## <a name="officeDownloads"></a>Получить файлы, необходимые для подключения к Excel  
Может потребоваться скачать сетевые компоненты для источников данных Microsoft Office Excel и Access, в том числе, если они еще не установлены. Загрузите последнюю версию компонентов подключения здесь файлов Excel и Access: [доступа к базе данных ядро 2016 распространяемый компонент Microsoft](https://www.microsoft.com/download/details.aspx?id=54920).
  
Последнюю версию компонентов позволяет открывать файлы, созданные в предыдущих версиях Excel.

Если компьютер имеет 32-разрядной версии Office, необходимо установить 32-разрядную версию компонентов, а также необходимо гарантировать выполнение пакета в 32-разрядном режиме.

Если у вас есть подписка Office 365, убедитесь, что вы загрузите 2016 распространяемый доступа базы данных ядра и не 2016 среды выполнения Microsoft Access. При запуске программы установки может появиться сообщение об ошибке, что невозможно установить загрузки side-by-side работай компоненты Microsoft Office. Чтобы обойти это сообщение об ошибке, запустите установку в автоматическом режиме, открыв окно командной строки и выполнение. EXE-файла, загруженного с `/quiet` переключения. Например:

`C:\Users\<user name>\Downloads\AccessDatabaseEngine.exe /quiet`
