# <a name="release-notes-for-the-jdbc-driver"></a>Заметки о выпуске для драйвера JDBC
[!INCLUDE[Driver_JDBC_Download](../../includes/driver_jdbc_download.md)]

## <a name="updates-in-microsoft-jdbc-driver-62-for-sql-server"></a>Обновления в драйвере Microsoft JDBC 6.2 для SQL Server
6.2 драйвер JDBC Microsoft SQL Server — полностью соответствует спецификации JDBC 4.1 и 4.2. Имена файлов, содержащихся в пакете 6.0 по совместимость версий Java. Например mssql jdbc-6.2.1.jre8.jar файл из пакета 6.2 рекомендуется для использования с Java 8. 

**Примечание:** проблемы с метаданными, кэширование улучшения был найден в RTW 6.2 JDBC, выпущенные на 29 июня 2017 г. Откат улучшения и новые JAR-файлов (версия 6.2.1) выпущенных 17 июля 2017 г. на [центра загрузки Майкрософт](https://go.microsoft.com/fwlink/?linkid=852460), [GitHub](https://github.com/Microsoft/mssql-jdbc/releases/tag/v6.2.1), и [Maven центра](http://search.maven.org/#search%7Cgav%7C1%7Cg%3A%22com.microsoft.sqlserver%22%20AND%20a%3A%22mssql-jdbc%22). Обновите проекты, чтобы использовать 6.2.1 выпуска JAR-файлов. Можно найти [заметки о выпуске](https://github.com/Microsoft/mssql-jdbc/releases/tag/v6.2.1) для получения дополнительных сведений.

**Azure Active Directory (AAD) Поддержка Linux**

Подключение приложений Linux для базы данных SQL Azure с использованием проверки подлинности AAD посредством имени пользователя и пароля и доступа маркеров методов.

**Федеральный обработки информации Standard (FIPS) включена JVM**

Драйвер JDBC теперь можно использовать в JVM, которые выполняются в режиме соответствия FIPS 140 в соответствии с федеральным стандартам и соответствия. 

**Улучшения проверки подлинности Kerberos** 

Драйвер JDBC теперь имеет поддержку: 
* Метод субъект и пароль для приложений, где конфигурация Kerberos не может быть измененного или не удалось получить новый маркер или keytab. Этот метод можно использовать для проверки подлинности SQL Server, который допускает только проверку подлинности Kerberos. 
* Проверка подлинности между областями, с помощью встроенной проверки подлинности Kerberos без явного указания имени участника-службы сервера. Драйвер теперь автоматически вычисляет область даже в том случае, если не указан.
* Ограниченное делегирование Kerberos, принимая олицетворение учетных данных пользователя в виде учетных данных GSS объекта источника данных. Эти учетные данные для олицетворенного затем используется для установления соединения Kerberos. 

**Добавлены значения времени ожидания**

Драйвер JDBC теперь поддерживает следующие настраиваемые значения времени ожидания можно изменить в зависимости от потребностей вашего приложения. 
* Время ожидания запроса для управления количеством секунд до истечения периода ожидания при выполнении запроса. 
* Время ожидания для сокета для задания количества миллисекунд для ожидания до истечения периода ожидания для сокета чтения или принять.

## <a name="updates-in-microsoft-jdbc-driver-60-for-sql-server"></a>Обновления в Microsoft JDBC Driver 6.0 для SQL Server

Microsoft JDBC Driver 6.0 для SQL Server — полностью соответствует спецификации JDBC 4.1 и 4.2. Имена файлов, содержащихся в пакете 6.0 по их соответствие с версией JDBC API. Например файл sqljdbc42.jar из пакета 6.0 — API JDBC 4.2 совместимым. Аналогичным образом sqljdbc41.jar файл совместима с API JDBC 4.1.

Чтобы убедиться, что у вас есть sqljdbc41.jar или sqljdbc42.jar вправо, запустите следующий код. Если выходные данные «версии драйвера: 6.0.7507.100», у вас есть пакет JDBC Driver 6.0.
```
Connection conn = DriverManager.getConnection("jdbc:sqlserver://<server>;user=<user>;password=<password>;");
System.out.println("Driver version: " + conn.getMetaData().getDriverVersion());
```  
 **Постоянное шифрование**  
  
 Поддержка недавно выпущенная функция постоянного шифрования в SQL Server 2016 это новая функция безопасности, что гарантирует, что конфиденциальные данные никогда не отображаются в виде обычного текста в экземпляре SQL Server. В основе работы функции постоянного шифрования лежит прозрачное шифрование данных в приложении, чтобы сервер SQL Server обрабатывал только зашифрованные данные, а не значения открытого текста. Даже в случае компрометации экземпляра SQL или хост-компьютера злоумышленник получит достаточно зашифрованный текст конфиденциальных данных. Дополнительные сведения см. [использование постоянного шифрования с драйвером JDBC](../../connect/jdbc/using-always-encrypted-with-the-jdbc-driver.md).  
  
 **Международного доменного имени (IDN)**  
  
 Поддержка международных доменных имен (IDN) в качестве имен серверов. Подробные сведения см. на использование международных доменных имен [международные функции драйвера JDBC](../../connect/jdbc/international-features-of-the-jdbc-driver.md) страницы.  
  
 **Параметризованный запрос**  
  
 Теперь поддерживается получение метаданных параметра с подготовленными инструкциями для сложных запросов, например подзапросов или соединений. Обратите внимание, что это улучшение доступно только при использовании SQL Server 2012 и более новых версий.  
  
 **Azure Active Directory (AAD)**  
  
 Проверка подлинности AAD — это механизм подключения к базе данных SQL Azure версии 12 с использованием удостоверения в Azure Active Directory. Проверка подлинности AAD используется для централизованного управления удостоверениями пользователей базы данных и в качестве альтернативы проверке подлинности SQL Server. Драйвер JDBC 6.0 позволяет указать учетные данные AAD в строке подключения JDBC для подключения к базе данных SQL Azure.  Подробные сведения см. свойство проверки подлинности на [задание свойств соединения](../../connect/jdbc/setting-the-connection-properties.md) страницы.  
  
 **Возвращающие табличные значения параметров**  
  
 Возвращающие табличное значение параметры обеспечивают простой способ упаковки строк данных из клиентского приложения к серверу SQL Server без необходимости несколько циклов приема-передачи или специальной логики на стороне сервера для обработки данных. Возвращающие табличные значения параметров можно использовать для инкапсуляции строк данных в клиентском приложении и отправки данных на сервер в одной параметризированной команды. Входящие строки данных хранятся в переменной таблицы, которое можно затем работать с помощью Transact-SQL. Дополнительные сведения см. [Using Table-Valued параметры](../../connect/jdbc/using-table-valued-parameters.md).  
  
 **Группы доступности AlwaysOn (AG)**  
  
 Драйвер теперь поддерживает прозрачное подключение к группам доступности AlwaysOn. Драйвер быстро обнаруживает текущую топологию AlwaysOn серверной инфраструктуры и прозрачно подключается к текущего активного сервера.
