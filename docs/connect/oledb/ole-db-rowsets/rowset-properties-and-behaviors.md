# <a name="rowset-properties-and-behaviors"></a>Свойства и поведение наборов строк
[!INCLUDE[appliesto-ss-asdb-asdw-pdw-md](../../../includes/appliesto-ss-asdb-asdw-pdw-md.md)]

  Это драйвер OLE DB для SQL Server свойств набора строк.  
  
|Идентификатор свойства|Описание|  
|-----------------|-----------------|  
|DBPROP_ABORTPRESERVE|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Этим свойством определяется поведение набора строк после аварийного завершения операции.<br /><br /> VARIANT_FALSE: Драйвер OLE DB для SQL Server делает наборы строк недействительными после аварийного завершения операции. Объект набора строк практически утратил свою функциональность. Он поддерживает только **IUnknown** операций и выпуска необработанных строк и обработчики методов доступа.<br /><br /> VARIANT_TRUE: Драйвер OLE DB для SQL Server поддерживает допустимый набор строк.|  
|DBPROP_ACCESSORDER|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: DBPROPVAL_AO_RANDOM<br /><br /> Описание: Порядок доступа. Порядок, в котором должен осуществляться доступ к столбцам в наборе строк.<br /><br /> DBPROPVAL_AO_RANDOM: Столбец может осуществляться в любом порядке.<br /><br /> DBPROPVAL_AO_SEQUENTIALSTORAGEOBJECTS: Столбцы привязаны как хранилища объектов может осуществляться только в последовательном порядке, по порядковому номеру столбца.<br /><br /> DBPROPVAL_AO_SEQUENTIAL: Все столбцы могут быть доступны в последовательном порядке, по порядковому номеру столбца.|  
|DBPROP_APPENDONLY|Это свойство набора строк не реализуется драйвером OLE DB для SQL Server. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_BLOCKINGSTORAGEOBJECTS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Драйвер OLE DB для SQL Server хранилища объектов блока с помощью других методов набора строк.|  
|DBPROP_BOOKMARKS DBPROP_LITERALBOOKMARKS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает закладки для определения строки набора строк при DBPROP_BOOKMARKS или DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> Присвоение любому из этих свойств значения VARIANT_TRUE не включает позиционирование набора строк по закладке. Чтобы создать набор строк, поддерживающий позиционирование по закладке, присвойте свойству DBPROP_IRowsetLocate или DBPROP_IRowsetScroll значение VARIANT_TRUE.<br /><br /> Драйвер OLE DB для SQL Server использует [!INCLUDE[msCoName](../../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] курсора для поддержки набора строк, который содержит закладки. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../oledb/ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Примечание: Задание этих свойств конфликтует с другими драйвер OLE DB для SQL Server определение курсора свойств приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_BOOKMARKS и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_BOOKMARKSKIPPED|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server вернет значение DB_E_BADBOOKMARK, если потребитель указал недопустимую закладку при позиционировании или поиске набора строк с закладками.|  
|DBPROP_BOOKMARKTYPE|R Чтение и запись: только для чтения<br /><br /> По умолчанию: DBPROPVAL_BMK_NUMERIC<br /><br /> Описание: Драйвер OLE DB для SQL Server реализует только числовые закладки. Драйвер OLE DB для SQL Server закладки является 32-разрядное целое число без знака, имеющее тип DBTYPE_UI4.|  
|DBPROP_CACHEDEFERRED|Это свойство набора строк не реализуется драйвером OLE DB для SQL Server. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_CANFETCHBACKWARDS DBPROP_CANSCROLLBACKWARDS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает обратную выборку и прокрутку в непоследовательных наборах строк. Драйвер OLE DB для SQL Server создает набор строк с поддержкой курсора, если DBPROP_CANFETCHBACKWARDS и DBPROP_CANSCROLLBACKWARDS имеют значение VARIANT_TRUE. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../oledb/ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_CANHOLDROWS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: По умолчанию драйвер OLE DB для SQL Server возвращает DB_E_ROWSNOTRELEASED при попытке потребителя получить больше строк из набора отложенных изменений, в котором существуют в настоящее время в наборе строк. Это поведение может быть изменено.<br /><br /> Присвоение свойствам DBPROP_CANHOLDROWS и DBPROP_IRowsetChange значения VARIANT_TRUE подразумевает набор строк с закладками. Если оба свойства имеют значение VARIANT_TRUE, **IRowsetLocate** интерфейс доступен на наборе строк и DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> Драйвер OLE DB для SQL Server наборы строк, содержащих закладки поддерживаются [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] курсоров.|  
|DBPROP_CHANGEINSERTEDROWS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Это свойство может устанавливаться только значение VARIANT_TRUE, если набор строк использует курсор, управляемый набором ключей.|  
|DBPROP_COLUMNRESTRICT|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server в значение VARIANT_TRUE при задается объектом-получателем нельзя добавить столбец в наборе строк. Другие столбцы в наборе строк можно изменять, а сами строки — удалять.<br /><br /> Если свойство имеет значение VARIANT_TRUE, то потребитель по элементу *dwFlags* член структуры DBCOLUMNINFO, чтобы определить, можно ли выполнять запись значение отдельного столбца. Для изменяемых столбцов *dwFlags* принимает значение DBCOLUMNFLAGS_WRITE.|  
|DBPROP_COMMANDTIMEOUT|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: 0<br /><br /> Описание: По умолчанию драйвер OLE DB для SQL Server время ожидания не истекает в **ICommand::Execute** метод.|  
|DBPROP_COMMITPRESERVE|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Этим свойством определяется поведение набора строк после операции фиксации.<br /><br /> VARIANT_TRUE: Драйвер OLE DB для SQL Server поддерживает допустимый набор строк.<br /><br /> VARIANT_FALSE: Драйвер OLE DB для SQL Server делает наборы строк недействительными после операции фиксации. Объект набора строк практически утратил свою функциональность. Он поддерживает только **IUnknown** операций и выпуска необработанных строк и обработчики методов доступа.|  
|DBPROP_DEFERRED|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Если задано значение VARIANT_TRUE драйвер OLE DB для SQL Server пытается использовать серверный курсор для набора строк. **Текст**, **ntext**, и **изображения** столбцы не возвращаются с сервера, пока они доступны для приложения.|  
|DBPROP_DELAYSTORAGEOBJECTS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает режим немедленного обновления для хранения объектов.<br /><br /> Изменения, примененные к данным в объекте последовательного потока, немедленно передаются [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)]. Фиксация изменений производится в соответствии с режимом транзакций набора строк.|  
|DBPROP_HIDDENCOLUMNS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> **Описание:** скрытые число столбцов<br /><br /> Если DBPROP_UNIQUEROWS имеет значение VARIANT_TRUE, то свойство DBPROP_HIDDENCOLUMNS возвращает число дополнительных «скрытых» столбцов, добавленных поставщиком для уникальной идентификации строк в наборе. Эти столбцы возвращаются **IColumnsInfo::GetColumnInfo** и **IColumnsRowset::GetColumnsRowset**. Тем не менее, они не включаются в число строк, возвращенных *pcColumns* аргумента, возвращаемого **IColumnsInfo::GetColumnInfo**.<br /><br /> Чтобы определить общее число столбцов, представленных в *prgInfo* структуры, возвращенный **IColumnsInfo::GetColumnInfo**, включая скрытые столбцы, потребитель должен добавить значение DBPROP_ HIDDENCOLUMNS количество столбцов, возвращаемых из **IColumnsInfo::GetColumnInfo** в *pcColumns*. Если свойство DBPROP_UNIQUEROWS имеет значение VARIANT_FALSE, то свойство DBPROP_HIDDENCOLUMNS равно нулю.|  
|DBPROP_IAccessor DBPROP_IColumnsInfo DBPROP_IConvertType DBPROP_IRowset DBPROP_IRowsetInfo|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает эти интерфейсы для всех наборов строк.|  
|DBPROP_IColumnsRowset|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает **IColumnsRowset** интерфейса.|  
|DBPROP_IConnectionPointContainer|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: IConnectionPointContainer. Если значение равно VARIANT_TRUE, то набор строк поддерживает заданный интерфейс. Если значение равно VARIANT_FALSE, то набор строк не поддерживает заданный интерфейс. Поставщики, поддерживающие тот или иной интерфейс, должны поддерживать связанное с этим интерфейсом свойство, которое возвращает значение VARIANT_TRUE. Эти свойства служат главным образом для запроса интерфейсов с помощью метода ICommandProperties::SetProperties.|  
|DBPROP_IMultipleResults|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает **IMultipleResults** интерфейса.|  
|DBPROP_IRowsetChange DBPROP_IRowsetUpdate|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает **IRowsetChange** и **IRowsetUpdate** интерфейсов.<br /><br /> Набор строк, созданный с DBPROP_IRowsetChange, равным VARIANT_TRUE, отражает режимы немедленного обновления.<br /><br /> Если DBPROP_IRowsetUpdate имеет значение VARIANT_TRUE, то DBPROP_IRowsetChange также имеет значение VARIANT_TRUE. Набор строк характеризуется поведением отложенного обновления.<br /><br /> Драйвер OLE DB для SQL Server использует [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] курсора для поддержки набора строк **IRowsetChange** или **IRowsetUpdate**. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../oledb/ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_IRowsetIdentity|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Драйвер OLE DB для SQL Server поддерживает **IRowsetIdentity** интерфейса. Если набор строк поддерживает этот интерфейс, то любые два дескриптора строк, относящихся к одной базовой строке, всегда представляют одни и те же данные и состояние. Потребители могут вызвать метод **IRowsetIdentity:: IsSameRow** метод для сравнения двух дескрипторы строк, чтобы увидеть, если они ссылаются на один и тот же экземпляр строки.|  
|DBPROP_IRowsetLocate DBPROP_IRowsetScroll|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server может предоставлять **IRowsetLocate** и **IRowsetScroll** интерфейсов.<br /><br /> Если DBPROP_IRowsetLocate имеет значение VARIANT_TRUE, CANFETCHBACKWARDS и DBPROP_CANSCROLLBACKWARDS также имеют значение VARIANT_TRUE.<br /><br /> Если DBPROP_IRowsetScroll имеет значение VARIANT_TRUE, то DBPROP_IRowsetLocate также имеет значение VARIANT_TRUE, поэтому оба интерфейса доступны в наборе строк.<br /><br /> Закладки необходимы для обоих интерфейсов. Драйвер OLE DB для SQL Server устанавливает DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS значение VARIANT_TRUE, если потребитель запрашивает один из этих интерфейсов.<br /><br /> Драйвер OLE DB для SQL Server использует [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] курсоры для поддержки **IRowsetLocate** и **IRowsetScroll**. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../oledb/ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Настройка этих свойств конфликтует с другими драйвер OLE DB для SQL Server определение курсора свойств приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_IRowsetScroll и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_IRowsetResynch|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server предоставляет **IRowsetResynch** интерфейса по требованию. Драйвер OLE DB для SQL Server может поддерживать этот интерфейс в любом наборе строк.|  
|DBPROP_ISupportErrorInfo|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Драйвер OLE DB для SQL Server предоставляет **ISupportErrorInfo** интерфейс наборов строк.|  
|DBPROP_ILockBytes|Этот интерфейс не реализуется драйвером OLE DB для SQL Server. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_ISequentialStream|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Драйвер OLE DB для SQL Server предоставляет **ISequentialStream** интерфейс для поддержки больших данных переменной длины, хранящихся в [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)].|  
|DBPROP_IStorage|Этот интерфейс не реализуется драйвером OLE DB для SQL Server. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_IStream|Этот интерфейс не реализуется драйвером OLE DB для SQL Server. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_IMMOBILEROWS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Свойство имеет значение VARIANT_TRUE только для [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] управляемые набором ключей курсоры; для всех остальных курсоров оно имеет значение VARIANT_FALSE.<br /><br /> VARIANT_TRUE: Набор строк не будет переупорядочивать вставленные или обновленные строки. Для **IRowsetChange::InsertRow**, строки появляются в конец набора строк. Для **IRowsetChange::SetData**, если набор строк не упорядочен, положение обновленных строк не изменяется. Если набор строк упорядочен и **IRowsetChange::SetData** переместить столбец, который используется для упорядочивания набора строк, строки не изменения. Если набор строк построен на наборе ключевых столбцов (обычно это столбцы, свойство DBPROP_OTHERUPDATEDELETE которых имеет значение VARIANT_TRUE, а свойство DBPROP_OTHERINSERT — значение VARIANT_FALSE), то изменение значения ключевого столбца, как правило, эквивалентно удалению текущей строки вставке новой. Поэтому строка может переместиться или даже исчезнуть из набора строк, если свойство DBPROP_OWNINSERT имеет значение VARIANT_FALSE, даже со значением свойства DBPROP_IMMOBILEROWS, равным VARIANT_TRUE.<br /><br /> VARIANT_FALSE: Если набор строк упорядочен, вставленные строки находятся в наборе строк правильном порядке. Если набор строк не является упорядоченным, то вставленные строки появляются в конце. Если **IRowsetChange::SetData** изменяет столбец, который используется для набора строк, строка перемещается заказа. Если набор строк не упорядочен, то положение строки не меняется.|  
|DBPROP_LITERALIDENTITY|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: Данное свойство всегда имеет значение VARIANT_TRUE.|  
|DBPROP_LOCKMODE|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: DBPROPVAL_LM_NONE<br /><br /> Описание: Уровень блокировки, выполняемый набором строк (DBPROPVAL_LM_NONE, DBPROPVAL_LM_SINGLEROW).<br /><br /> Примечание: При использовании изоляции моментального снимка в транзакции, если набор строк открывается с помощью набора ключей или динамического серверного курсора, а режим блокировки установлен на DBPROPVAL_LM_SINGLEROW, произойдет ошибка при выборке строки, если кто-то другой обновил эту строку, так как транзакция был запущен. Для других типов курсоров и режимов блокировки, если другой пользователь обновил строку со времени начала транзакции, то ошибка не произойдет до тех пор, пока пользователь не попытается обновить эту строку.
