# Создание мобильного отчета с вкладками с помощью детализации | Мобильные отчеты служб Reporting Services
Вы можете научиться создавать мобильных отчет [!INCLUDE[ssRSnoversion_md](../../includes/ssrsnoversion-md.md)], который выглядит и работает как отчет с вкладками, с помощью детализации и параметров.

Например, в этом отчете датчики в верхней части играют роль вкладок. При выборе датчика транспортировки (Transportation) остальная часть диаграммы фильтруется по данным о транспортировке.

![06-Mobile-Report-Web-Viewer-Transportation](../../reporting-services/mobile-reports/media/06-mobile-report-web-viewer-transportation.png)

В действительности это набор из пяти отдельных отчетов, в каждом из которых есть свой параметр для фильтрации отчета в соответствии с датчиком, выбранным в верхней части отчета. 

В этом примере сначала создаются все пять отчетов, а затем для каждого из них в детализацию к другим отчетам вносятся четыре других датчика. 

Далее приводится структура действий на высоком уровне.

1. Создайте отчет с именем "Продажи" и с пятью следующими датчиками.
     - Продажи
     - Транспортировка
     - Топливо
     - Память
     - Прочие расходы
2. Сделайте четыре копии этого отчета со следующими именами. 
     - Транспортировка
     - Топливо
     - Память
     - Прочие расходы
3.  В отчете "Продажи" установите каждый из четырех датчиков (кроме датчика продаж) в качестве детализации в соответствующие отчеты.

4. В том же отчете "Продажи" установите свойство Accent датчика продаж, чтобы он отличался от остальной части отчета и лучше выделялся.

5.  Теперь в отчете "Транспортировка" задайте датчик продаж как детализацию в отчет о продажах и другие датчики как детализацию в соответствующие отчеты.

6. Оставаясь в отчете "Транспортировка", установите свойство Accent датчика транспортировки, чтобы он отличался от остальной части отчета.

5. Повторите это для отчетов по топливу, складу и прочим расходам.
