# <a name="dashboards-in-includename-sosincludesname-sos-shortmd"></a>Панели мониторинга в [!INCLUDE[name-sos](../includes/name-sos-short.md)]

Чтобы просмотреть панель мониторинга, щелкните правой кнопкой мыши сервер или базу данных и выбрать **управление**.

![Пример панели мониторинга](media/dashboards/sample-dashboard.png)

**Свойства сервера** содержит свойства сервера, включая версию, выпуск, имя компьютера и версии ОС.

**Задачи** содержит commons задач, таких как восстановление и новый запрос.

**Поиск баз данных** без труда найти существующие базы данных, хранящихся на сервере, включая таблицы.

**Состояние архивации** без труда найти состояние резервного копирования для существующих баз данных.

## <a name="insight-detail"></a>Представление сведений
Всплывающее окно сведений о Insight предоставляет более подробную информацию о связанных insight мини-приложения. 
- Мини-приложение Insight визуализацию в краткие сводки представления числа, строки, диаграммы и т.д. 
- Всплывающее окно сведений о Insight сведения «перейти», со списком глубокому анализу данных для каждого элемента, перечисленных в мини-приложения высокого уровня Insight. 
  - Содержимое всплывающего меню сведения определяются с помощью отдельный запрос SQL для запроса основного графического элемента. 

Нет необходимости набора для запроса сведений о Insight, но стандартный макет.
- В верхней части представления всегда — это представление «Сводка» 2 столбца. Какие столбцы использовать определяются «label» и «value» свойства конфигурации JSON
- После нажатия кнопки, для строки в сводной таблице, нижней половине панели будут отображаться полный набор сведения о столбцах для этой строки.

## <a name="insight-actions"></a>Анализ действий
С помощью мини-приложение анализ и представление сведений может легко прийти плана действий, чтобы устранить проблему или управлять ими. Например будет рассматривать выполнение инструкции DBCC CHECKDB, читать журналы ошибок или восстановить базу данных, когда база данных находится в процессе восстановления, ожидающие. Или может быть любое из действий, которые необходимо выполнить.

С помощью [!INCLUDE[name-sos](../includes/name-sos-short.md)]элемента конфигурации Insight действия, вы можете сопоставить встроенные действия, такие как восстановление, или переносите собственные действия, определенные с помощью скрипта sql.

> Конфигурации пользовательских действий, с помощью скрипта sql находится в стадии разработки, и он еще не доступны в закрытой предварительной версии сборки.

## <a name="sample-insight-action-definition"></a>Пример определения действия Insight

```"actions"{}``` Определяет действие insight. Действие может быть определено по определенной области такие как ```"server"```, ```"database"``` и т. д и [!INCLUDE[name-sos](../includes/name-sos-short.md)] передает сведения о контексте текущего подключения к действию. 

Например, при запуске действия восстановления для базы данных WideWorldImporters, ```"database": "${Database}"``` указывает определение для передачи ```Database``` значение столбца в результат запроса для операции восстановления. Затем операция восстановления запускается в том, для базы данных. ```"types"``` — это массив json и несколько действий, которые можно добавить в массиве. По сути, он становится контекстного меню в диалоговом окне сведений Insight этого пользователя нажмите кнопку и выполнять действие. 

> [!INCLUDE[name-sos](../includes/name-sos-short.md)] Предварительная версия 0.17.1 включил «backup», «восстановление», «создать запрос» и «новая-база данных» как типы действий.

```json
"details": {
    "queryFile": "./sql/database_state_detail.sql",
    "label": {...},
    "value": "state",
    "actions": {
        "database": "${Database}",
        "types": ["restore"]
    }
}
```
