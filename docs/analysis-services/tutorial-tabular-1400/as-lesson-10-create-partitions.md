# <a name="create-partitions"></a>Создание секций

[!INCLUDE[ssas-appliesto-sql2017-later-aas](../../includes/ssas-appliesto-sql2017-later-aas.md)]

На этом занятии создается секций, чтобы разделить таблицу FactInternetSales на более мелкие логические части, которые могут обрабатываться (обновляться) независимо от других секций. По умолчанию каждая таблица, включить в модель имеет одну секцию, которая включает в себя столбцы и строки таблицы. Для таблицы FactInternetSales нам необходимо разделить данные по годам; одну секцию для каждой таблицы пять лет. После этого каждую секцию можно обработать независимо. Дополнительные сведения см. в разделе [секций](../tabular-models/partitions-ssas-tabular.md). 
  
Предполагаемое время выполнения этого занятия: **15 минут**  
  
## <a name="prerequisites"></a>предварительные требования  

В этой статье является частью учебника по табличному моделированию, который необходимо изучать по порядку. Перед выполнением задач этого занятия, необходимо завершить предыдущее занятие: [занятии 9: создание иерархий](../tutorial-tabular-1400/as-lesson-9-create-hierarchies.md).  
  
## <a name="create-partitions"></a>Создание секций  
  
#### <a name="to-create-partitions-in-the-factinternetsales-table"></a>Чтобы создать секции в таблицу FactInternetSales  
  
1.  В табличной модели обозревателе разверните **таблиц**, щелкните правой кнопкой мыши **FactInternetSales** > **секций**.  
  
2.  Щелкните в диспетчере секций **копирования**и затем измените имя на **FactInternetSales2010**.
  
    Поскольку необходимо, чтобы секция включала только строки в течение определенного периода, за год 2010, необходимо изменить выражение запроса.
  
4.  Нажмите кнопку **разработки** откройте редактор запросов, а затем нажмите кнопку **FactInternetSales2010** запроса.

5.  В режиме предварительного просмотра, нажмите кнопку со стрелкой вниз в **OrderDate** заголовок столбца, а затем щелкните **фильтров даты и времени** > **между**.

    ![как lesson10--редактора запросов](../tutorial-tabular-1400/media/as-lesson10-query-editor.png)

6.  В диалоговом окне фильтров строк в **отображает строки, где: OrderDate**, оставьте **после или равно**и введите в поле date **1/1/2010**. Оставить **и** оператор выбран, выберите **перед**, введите в поле date **1 1 сентября**и нажмите кнопку **ОК**.

    ![как lesson10-фильтр строк](../tutorial-tabular-1400/media/as-lesson10-filter-rows.png)
    
    Обратите внимание в редакторе запросов, в ПРИМЕНЕННЫХ ДЕЙСТВИЙ появиться еще один шаг, с именем отфильтрованные строки. Этот фильтр является выбор только даты заказов из 2010.

8.  нажмите кнопку **Импортировать**;

    В диспетчере секций Обратите внимание, что теперь выражение запроса содержит дополнительное предложение фильтрации строк.

    ![как запрос lesson10](../tutorial-tabular-1400/media/as-lesson10-query.png)
  
    Эта инструкция указывает, что эта секция должна включать только данные в этих строках, в которых OrderDate относится 2010 календарного года, как указано в предложении отфильтрованные строки.  
  
  
#### <a name="to-create-a-partition-for-the-2011-year"></a>Создание секции для 2011 года  
  
1.  В списке секций выберите **FactInternetSales2010** секции был создан и нажмите кнопку **копирования**.  Сменить имя раздела для **FactInternetSales2011**. 

    Необходимо использовать редактор запросов для создания нового предложения отфильтрованные строки. Так как вы создали копии запроса для 2010, необходимо выполнить достаточно внести небольшие изменения в запросе для 2011.
  
2.  В **выражение запроса**, в определенном порядке для этой секции включала только строки для 2011 года, замените лет в предложении фильтрации строк с **2011** и **2012**соответственно, например:  
  
    ```  
    let
        Source = #"SQL/localhost;AdventureWorksDW2014",
        dbo_FactInternetSales = Source{[Schema="dbo",Item="FactInternetSales"]}[Data],
        #"Removed Columns" = Table.RemoveColumns(dbo_FactInternetSales,{"OrderDateKey", "DueDateKey", "ShipDateKey"}),
        #"Filtered Rows" = Table.SelectRows(#"Removed Columns", each [OrderDate] >= #datetime(2011, 1, 1, 0, 0, 0) and [OrderDate] < #datetime(2012, 1, 1, 0, 0, 0))
    in
        #"Filtered Rows"
   
    ```  
  
#### <a name="to-create-partitions-for-2012-2013-and-2014"></a>Для создания секций для 2012, 2013 и 2014.  
  
- Выполните указанные ранее шаги, создания разделов для 2012, 2013 и 2014, изменение лет в предложении фильтрации строк, чтобы включать только те строки за этот год. 
  

## <a name="delete-the-factinternetsales-partition"></a>Удалить раздел FactInternetSales

Теперь, когда у вас есть секции для каждого года, можно удалить раздел FactInternetSales; Предотвращение перекрытия при выборе всех процесс при обработке секций.

#### <a name="to-delete-the-factinternetsales-partition"></a>Для удаления раздела FactInternetSales

-  Нажмите кнопку **FactInternetSales** секции, а затем нажмите кнопку **удалить**.



## <a name="process-partitions"></a>Обработка секций  

Обратите внимание, в диспетчере секций **последней обработки** столбец для каждого нового секции, которую вы создали показывает эти разделы никогда не будут обработаны. При создании секций следует выполнять операцию обработки секций или обработки таблиц, чтобы обновить данные в эти разделы.  
  
#### <a name="to-process-the-factinternetsales-partitions"></a>Обработка секций FactInternetSales  
  
1.  Нажмите кнопку **ОК** закрыть диспетчер секций.  
  
2.  Нажмите кнопку **FactInternetSales** , а затем нажмите кнопку **модели** меню > **процесс** > **Обработка секций**.  
  
3.  В диалоговом окне Обработка секций проверьте **режим** задано значение **обработка по умолчанию**.  
  
4.  Установите флажок в столбце **Обработка** для каждой из пяти созданных секций и нажмите кнопку **OK**.  

    ![как lesson10-Обработка секций](../tutorial-tabular-1400/media/as-lesson10-process-partitions.png)
  
    Если появится предложение ввести учетные данные олицетворения, введите имя пользователя Windows и пароль, которые указаны в занятии 2.  
  
    Откроется диалоговое окно **Обработка данных** со сведениями обработки для каждой секции. Обратите внимание, что для каждой секции передается различное количество строк. Каждая секция включает только те строки для года, указанного в предложении WHERE в инструкции SQL. По завершении обработки закройте диалоговое окно "Обработка данных".  
  
    ![как lesson10-процесса — завершение](../tutorial-tabular-1400/media/as-lesson10-process-complete.png)
