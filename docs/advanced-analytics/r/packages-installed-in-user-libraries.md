# <a name="packages-installed-in-user-libraries"></a>Пакеты, установленные в библиотеках пользователей

Если пользователю нужен новый пакет R и у него нет прав администратора, пакеты невозможно будет установить в расположение по умолчанию. Они установятся в закрытую библиотеку пользователя. 

Чтобы ссылаться на пакет в коде в типичной среде разработки R, пользователю потребуется добавить расположение в переменную среды R `libPath` или добавить ссылку на полный путь к пакету следующим образом:  
  
~~~~
library("c:/Users/<username>/R/win-library/packagename")  
~~~~

## <a name="problems-with-packages-in-user-libraries"></a>Проблемы с пакетами в библиотеках пользователей

Однако в [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] это обходное решение **не** поддерживается. Пакеты следует устанавливать в библиотеку по умолчанию. В противном случае при попытке вызова пакета может появиться следующая ошибка:

*Произошла ошибка в библиотеке (xxx). Отсутствует пакет с именем xxx*
 

Таким образом, при переносе решений R для запуска в [!INCLUDE[ssNoVersion_md](../../includes/ssnoversion-md.md)] важно сделать следующее:
+ установите все необходимые пакеты в библиотеку по умолчанию;
+ измените код, чтобы обеспечить загрузку пакетов из библиотеки по умолчанию, а не из нерегламентированных каталогов или библиотек пользователей;
+ проверьте код, чтобы убедиться в отсутствии вызовов к удаленным пакетам;
+ измените любой код, который попытается установить пакеты динамически.
 
Если пакет установлен в библиотеку по умолчанию, среда выполнения R загрузит его из этой библиотеки, даже если в коде R указана другая библиотека.
