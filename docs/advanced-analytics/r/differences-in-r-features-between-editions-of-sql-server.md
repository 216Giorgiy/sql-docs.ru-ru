# <a name="differences-in-machine-learning-features-between-editions-of-sql-server"></a>Различия в возможностях обучения машины между выпусками SQL Server
 
 Поддержка для машинного обучения доступна в следующих выпусках SQL Server 2016 и 2017 г. SQL Server:

## <a name="summary-of-differences"></a>Сводка различий

-   **Enterprise Edition**
    
     2017 г. SQL Server включает в себя службы обучения машины (в-базы данных. SQL Server 2016 включает службы R Services. Эта функция поддерживает аналитика в базе данных в SQL Server, включая использование SQL Server в качестве контекста вычислений.
     
     2017 г. SQL Server включает в себя Microsoft машины обучения Server (изолированный). SQL Server 2016 включает Microsoft R Server (изолированный). Эта функция поддерживает ввода в эксплуатацию машинного обучения, которая не требует использования SQL Server в качестве контекста вычислений.

     Не ограничены от этих функций в выпуске Enterprise Edition, который предоставляет оптимизация производительности и масштабируемости с помощью параллелизации и потоковой передачи. Этот выпуск также увеличивает использование поддержка платформы для потоковой передачи и параллельного выполнения.
     
     Аналитика в базе данных с помощью SQL Server поддерживает управление ресурсами внешних скриптов, чтобы настроить использование ресурсов сервера.
     
     Более новые выпуски Microsoft R Server включают улучшенную версию подсистему ввода в эксплуатацию, которая поддерживает развертывание быстрого и безопасные и совместное использование решений R. Дополнительные сведения см. в разделе [mrsdeploy](https://docs.microsoft.com/r-server/r-reference/mrsdeploy/mrsdeploy-package).

-   **Developer Edition**

     Выпуск Developer Edition имеет те же возможности, что и Enterprise Edition, однако его нельзя использовать в рабочей среде.  
  
-   **Standard Edition**

     Все возможности анализа в базе данных, включенных в Enterprise Edition, за исключением управление ресурсами. Производительность и масштабируемость также ограничен: данных, который может быть обработан должны помещаться в памяти сервера, а обработка ограничивается вычислений один поток, даже при использовании **RevoScaleR** функции.
  
-   **Express и веб-выпуски**
  
     Только Express Edition with Advanced Services включает машинного обучения функции. Ограничения производительности такие же, как в выпуске Standard Edition. Выпуск Web edition не предназначен для выполнения задач, таких как создание моделей машинного обучения; Тем не менее можно использовать функцию ПРОГНОЗИРОВАНИЯ для выполнения оценки в другом месте с помощью обученной модели.

-   **База данных SQL Azure**
  
     Функции обучения компьютера, например R и сценариев Python в настоящее время не поддерживается в базе данных SQL Azure.
     
     Дополнительные сведения и объявления о когда этот компонент будет доступен, см. в блоге SQL Server: [Python в 2017 г. SQL Server: усиленной в базе данных машинного обучения](https://blogs.technet.microsoft.com/dataplatforminsider/2017/04/19/python-in-sql-server-2017-enhanced-in-database-machine-learning/)


### <a name="languages-supported-in-all-editions"></a>Языки, поддерживаемые в все выпуски

На следующих языках обучения машины поддерживаются для всех выпусков:

+ SQL Server 2017 г.: R и Python
+ SQL Server 2016: Только для чтения

Microsoft R Open входит в состав всех выпусков.

Microsoft R Client может работать со всеми выпусками.

## <a name="machine-learning-in-enterprise-edition"></a>Машинного обучения в выпуске Enterprise Edition

Производительность компьютера обучению в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] ожидается обычно преодолеть реализации с помощью обычных R, заданного в том же оборудовании. В том, что в SQL Server, решения R может выполняться с использованием ресурсов сервера, иногда распространяться на несколько процессов, использующих **RevoScaleR** функции. 

Производительность не имеет были оцениваться решения, Python, как компонент находится в разработке, но некоторые преимущества ожидаются вступили в силу.

Пользователи также могут ожидать значительные отличия в производительности и масштабируемости для того же решения машинного обучения при запуске в Visual Studio Enterprise Edition. Standard Edition. Причины, поддержка параллельной обработки, повышенная потоков, доступных для машинного обучения и потоковой передачи (или фрагментации), что позволяет функции RevoScaleR для обработки данных больше, чем может поместиться в памяти. 

Тем не менее может снизиться производительность даже на аналогичном оборудовании, от многих факторов вне кода R или Python. Эти факторы включают конкурирующих нагрузку на ресурсы сервера, тип плана запроса, создания, изменения схемы на необходимость обновления статистики или создать новый план запроса, фрагментации и многое другое. Возможно, что хранимая процедура, содержащий код R или Python может запуска в секундах, в одной рабочей нагрузки, но принимают минут, если есть другие службы.  Таким образом рекомендуется отслеживать производительность серверов, включая сетевого взаимодействия для контекстах удаленных вычислений при измерении производительности обучения машины несколько аспектов.

Также рекомендуется настроить [регулятора ресурсов](../../relational-databases/resource-governor/resource-governor.md) (доступно в выпуске Enterprise Edition) для настройки способа задания внешнего скрипта приоритеты и обрабатываются в условиях рабочих нагрузок с интенсивной на сервере. Определение функции-классификаторы указать источник внешних скриптов заданий и определения приоритета некоторых рабочих нагрузок, ограничить объем памяти, используемой SQL-запросы и управлять числом параллельных процессов, используемые на основе рабочей нагрузки.

## <a name="machine-learning-in-developer-edition"></a>Машинного обучения в выпуск Developer Edition

Выпуск Developer Edition предоставляет ту же производительность, что и Enterprise Edition. Тем не менее Developer Edition не поддерживается для рабочей среды.

## <a name="machine-learning-in-standard-edition"></a>Машинного обучения в выпуске Standard Edition

При одинаковой конфигурации оборудования даже выпуск Standard Edition должен обеспечивать некоторое повышение производительности по сравнению со стандартными пакетами R.

Выпуск Standard не поддерживает регулятора ресурсов. Управление ресурсами с помощью является лучшим способом, чтобы настроить ресурсы сервера для поддержки различных рабочих нагрузок, таких как модели обучения и оценки.

По сравнению с выпусками Enterprise и Developer выпуск Standard Edition также предоставляет ограниченную производительность и масштабируемость. Все **RevoScaleR** функциями и пакетами входят в состав выпуска Standard Edition, но ограничен число процессов, он может использовать службу, которая запускает и управляет R-скриптов. Кроме того, данные, обрабатываемые с помощью скрипта, должны помещаться в памяти.  Те же ограничения применяются для решений, использующих **revoscalepy**.

## <a name="machine-learning-in-express-edition-with-advanced-services"></a>Машинного обучения в выпуске Express Edition with Advanced Services

К Express Edition применяются те же ограничения, что и к Standard Edition.

## <a name="machine-learning-in-web-edition"></a>Машинного обучения в Web Edition

Выпуск Web edition не поддерживает выполнение скриптов R или Python. Тем не менее, можно использовать функцию ПРОГНОЗИРОВАНИЯ для выполнения [оценки собственного](../sql-native-scoring.md) на модели, обучение которых производилось на другой экземпляр SQL Server или R и затем сохраненные в требуемом формате двоичного.
