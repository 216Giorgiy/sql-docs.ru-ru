# <a name="in-database-python-analytics-for-sql-developers"></a>В базе данных аналитики Python для разработчиков SQL

Цель данного руководства — предоставить программисты SQL практический опыт создания машинного обучения решения в SQL Server. В этом пошаговом руководстве вы узнаете, как включить Python в приложение путем добавления кода Python в хранимых процедурах.

> [!NOTE]
> Предпочитаете R? В разделе [этого учебника](sqldev-in-database-r-for-sql-developers.md), который обеспечивает аналогичное решение, но использует R и выполнять eb в SQL Server 2016 или 2017 г. SQL Server.

## <a name="overview"></a>Обзор

Процесс построения законченного решения обычно состоит получения и очистки данных, изучения данных и формирования характеристик, обучения и настройки модели и, наконец, развертывания модели в рабочей среде. Разработка и тестирование фактический код лучше всего выполняется в среде выделенный разработки, такие как эти средства Python:

+ PyCharm популярные интегрированной среды разработки
+ Spyder, который входит в состав [2017 г. Visual Studio](https://blogs.msdn.microsoft.com/visualstudio/2017/05/12/a-lap-around-python-in-visual-studio-2017/) при установке [рабочей нагрузки обработки и анализа данных](https://blogs.msdn.microsoft.com/visualstudio/2016/11/18/data-science-workloads-in-visual-studio-2017-rc/)
+ [Расширения Python для Visual Studio](https://docs.microsoft.com/visualstudio/python/python-in-visual-studio).

После создания и тестирования решения в Интегрированной среде разработки, вы можете развернуть код Python [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] с помощью [!INCLUDE[tsql](../../includes/tsql-md.md)] хранимых процедур в знакомой среде [!INCLUDE[ssManStudio](../../includes/ssmanstudio-md.md)].

В этом пошаговом руководстве мы предположим, что вы предоставили кода Python, необходимые для решения и сосредоточусь на построение и развертывание решения с помощью SQL Server.

- [Шаг 1. Скачивание образца данных](sqldev-py1-download-the-sample-data.md)

  Загрузите образец набора данных и все файлы сценариев на локальном компьютере.

- [Шаг 2. Импорт данных в SQL Server с помощью PowerShell](sqldev-py2-import-data-to-sql-server-using-powershell.md)

  Выполните сценарий PowerShell, который создает базу данных и таблицу на указанном экземпляре и загружает данные в таблицу образца.

- [Шаг 3. Анализ и визуализация данных](sqldev-py3-explore-and-visualize-the-data.md)

  Выполните просмотр основных данных и визуализации, путем вызова Python из [!INCLUDE[tsql](../../includes/tsql-md.md)] хранимых процедур.

- [Шаг 4. Создание характеристик данных с помощью T-SQL](sqldev-py5-train-and-save-a-model-using-t-sql.md)

  Сформируйте характеристики данных с помощью пользовательских функций SQL.
  
- [Шаг 5. Обучение и сохранение модели с помощью T-SQL](sqldev-py5-train-and-save-a-model-using-t-sql.md)

   Постройте и сохранить модель машинного обучения, с помощью Python в хранимых процедурах.
  
-  [Шаг 6. Ввод модели в эксплуатацию](sqldev-py6-operationalize-the-model.md)

  После сохранения модели в базу данных вызовите модели для прогнозирования с помощью [!INCLUDE[tsql](../../includes/tsql-md.md)].

> [!NOTE]
> Мы рекомендуем не использовать [!INCLUDE[ssManStudioFull](../../includes/ssmanstudiofull-md.md)] для записи или тестирование кода Python. Если код, который необходимо внедрить в хранимой процедуре все проблемы, данные, возвращенные хранимой процедуры обычно недостаточно понять причину ошибки.


### <a name="scenario"></a>Сценарий

В этом пошаговом руководстве используется известный набор данных по работе такси в Нью-Йорке. Чтобы быстро и легко в данном пошаговом руководстве, выборки данных. Используя эти данные, вы создадите модель двоичной классификации, которая прогнозирует ли определенный trip шансов получить совет или нет, на основе столбцов, таких как время дня, расстояние и место сбора.

### <a name="requirements"></a>Требования

Это пошаговое руководство предназначено для пользователей, которые уже знакомы с операциями с базами данных, такими как создание баз данных и таблиц, импорт данных в таблицы и создание запросов SQL.

Предоставляется весь код Python. Опытный программист SQL может пройти это руководство, используя [!INCLUDE[tsql](../../includes/tsql-md.md)] в [!INCLUDE[ssManStudioFull](../../includes/ssmanstudiofull-md.md)] или выполняя предоставленные скрипты PowerShell.

Прежде чем приступать к выполнению пошагового руководства, необходимо завершить следующие подготовительные действия.

- Установка экземпляра SQL Server 2017 г. с машины обучения службы и включить Python (требуется CTP-версии 2.0 или более поздней версии).
- Учетная запись, используемая для этого пошагового руководства, должна иметь разрешения на создание баз данных и других объектов, отправку данных, выбор данных и выполнение хранимых процедур.
