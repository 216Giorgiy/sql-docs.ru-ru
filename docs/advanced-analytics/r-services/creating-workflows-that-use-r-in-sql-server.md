# Создание в SQL Server потоков, использующих R
  Реляционная база данных — это оптимизированные технология доставку масштабируемых решений для обработки транзакций, хранения и запросы к данным. Тем не менее традиционно R решения обычно полагались на импорт данных из различных источников, обычно в формате CSV для последующего изучения данных и моделирования. Этот метод не только неэффективен, но и небезопасен.  
  
 С помощью [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] предоставляет несколько преимуществ:  
  
-   Безопасность данных. Степень R переводится в базу данных, ближе к источнику данных. Позволяет избежать перемещения данных к бесполезному расходу ресурсов или небезопасной.  
  
-   Скорость передачи данных. Базы данных оптимизированы для операций с множествами. Кроме того последние нововведения в базах данных таких как таблиц в памяти и хранения в один столбец данных дополнительные повысить обработки и анализа данных, сделав агрегаты lightning fast.  
  
-   Интеграция. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] является центральной точкой операций для многих других задач управления данных и приложений, использующих enterprise. Использование данных в базе данных уже обеспечивает данных согласованный и установки последних обновлений. Вместо обработки данных на языке R, можно положиться на конвейеры данных предприятия включая [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] и фабрики данных Azure. Результаты или анализ отчетов можно легко через Power BI или [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)].  
  
 Используя правильное сочетание SQL и R для разнообразных задач по обработке и анализу данных, исследователи данных и разработчики систем могут повысить эффективность работы. Эта статья поможет понять, как вы можете сочетать R с другими корпоративными решениями для преобразования данных, анализа и составления отчетов.  
  
##  <a name="bkmk_ssis"></a> Создание эффективных рабочих процессов, охватывающих R и базы данных  
 Рабочие процессы для обработки и анализа данных имеют высокий уровень цикличности и содержат множество преобразований данных, в том числе масштабирование, агрегирование, вычисление вероятностей, переименование и слияние атрибутов. Ученые данные привыкли для выполнения многих из этих задач в R, Python или другом языке. Однако выполнение таких процессов на данных предприятия требует прозрачную интеграцию с ETL средств и процессов.  
  
 Поскольку [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] позволяет выполнять сложные операции в R через Transact-SQL и хранимых процедур, задач, связанных с R можно интегрировать с существующими процессами ETL без минимальный повторной разработки. А не выполнять цепочки задач ntensive памяти в R, подготовки данных можно оптимизировать с помощью наиболее эффективного средства, включая [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] и [!INCLUDE[tsql](../../includes/tsql-md.md)].  
  
 Например, можно объединить R с [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] в следующих сценариях:  
  
-   Используйте [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] задачи для создания необходимых объектов в базе данных SQL  
  
-   использование условного ветвления, чтобы переключать контекст вычислений для задач с применением R;  
  
-   выполнение задач с применением R, которые создают свои собственные данные;  
  
-   вызов из [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] R-сценария, сохраненного в текстовой переменной.

##  <a name="bkmk_ssrs"></a> Создание визуализаций, охватывающих R и корпоративные средства создания отчетов  
 С помощью языка R можно создавать диаграммы и интересные визуализации, но он недостаточно хорошо интегрирован с внешними источниками данных. Это означает, что каждую диаграмму и график следует создавать отдельно. Также совместный доступ может оказаться сложной.  
  
 С помощью [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)], можно выполнять сложные операции в R через [!INCLUDE[tsql](../../includes/tsql-md.md)] хранимых процедур, которые легко могут быть использованы с помощью разнообразных средств создания отчетов, включая enterprise [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)] и Power BI.  
  
-   Визуализация графические объекты, возвращаемые из сценария r.   
    использование [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)]  
  
-   Используйте таблицу в Power BI
