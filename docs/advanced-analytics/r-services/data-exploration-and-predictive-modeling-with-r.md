# Изучение данных и прогнозное моделирование с помощью R
  Специалисты по анализу данных часто используют R для изучения данных и построения прогнозных моделей. Обычно это итеративный процесс проб и ошибок, повторяющихся до получения хорошей прогнозной модели. Как опытный специалист, вы можете подключиться к базе данных [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] , передать данные на локальную рабочую станцию с помощью пакета RODBC, изучить их и создать прогнозную модель с использованием стандартных пакетов R.  
  
 Однако такой подход имеет недостатки. Перемещение данных может быть медленным, неэффективным или небезопасным, кроме того, сам язык R имеет ограничения по производительности и масштабируемости. Эти недостатки становятся более очевидными, когда требуется переместить и проанализировать большие объемы данных или использовать наборы данных, которые не помещаются в доступную на компьютере память.  
  
 Можно преодолеть эти трудности с помощью новых пакетов масштабируемых и включены функции R с [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)]. Пакет **RevoScaleR** содержит реализации для некоторых наиболее популярных функций R, переработанных для обеспечения параллелизма и масштабирования. Кроме того, пакет RevoScaleR обеспечивает поддержку для изменения *контекста выполнения*. Это означает, что как для всего решения, так и для отдельной функции можно указать, что вычисления должны выполняться с использованием ресурсов того компьютера, где размещен экземпляр [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] , а не локальной рабочей станции. У такого подхода есть несколько преимуществ: предотвращение ненужного перемещения данных и больший объем вычислительных ресурсов на сервере.  
  
 В этом разделе приведены предназначенные для специалистов по анализу данных рекомендации по использованию [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] и выполнению задач, относящихся к разработке и тестированию решений R.  
  
##  <a name="bkmk_RDevTools"></a> Средства разработки R  
 R клиента Microsoft дает полную среду разработки и тестирования моделей прогнозирования в обработке данных. Клиент R включает:  
  
-  **[!INCLUDE[rsql_rro-noversion](../../includes/rsql-rro-noversion-md.md)]:** Распространения R среды выполнения и набор пакетов, например библиотеку ядра Intel math, повысить производительность стандартные операции R.  
  
-   **RevoScaleR:** пакет R, позволяющий отправить вычислений экземпляр [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. [!INCLUDE[rsql_rre-noversion](../../includes/rsql-rre-noversion-md.md)]. Он также включает набор общих функций R, которые были переработаны для обеспечения повышения производительности и масштабируемости. Такие усовершенствованные функции имеют префикс **rx** . Он также включает поставщиков данных для различных источников; Эти функции начинаются с префикса **Rx**.  
  
-   **Бесплатные инструменты разработки:** можно использовать любой редактор кода на основе Windows, которая поддерживает R, такие как [!INCLUDE[rsql_rtvs](../../includes/rsql-rtvs-md.md)] или RStudio. Загрузка [!INCLUDE[rsql_rro-noversion](../../includes/rsql-rro-noversion-md.md)] также включает общие средства командной строки для R, например RGui.exe.  
  
##  <a name="bkmk_packages"></a> Пакеты и среда R  
 Поддерживаемая в [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] среда R состоит из среды выполнения, языка с открытым исходным кодом и графического модуля, который поддерживают и расширяют несколько пакетов. Язык позволяет использовать множество расширений, реализуемых с помощью пакетов.  
  
 Существует несколько источников дополнительных пакетов R, которые можно использовать с [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)] :  
  
  
-   Общие цели R пакеты из общих хранилищ. Можно получить популярные пакеты R с открытым исходным кодом из открытых репозиториев, таких как CRAN, где хранится более 6000 пакетов.  
  
     Доступны дополнительные пакеты для поддержки прогнозной аналитики в определенных сферах, таких как финансы, геномика и т. д.  
  
     На платформе Windows пакеты R предоставляются в виде ZIP-файлов и можно загрузить и установить лицензии GPL.  
  
     Дополнительные сведения об установке пакетов независимых производителей для использования с [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)], в разделе [установить дополнительные пакеты R, на сервере SQL Server](../../advanced-analytics/r-services/install-additional-r-packages-on-sql-server.md)  
  
-   Дополнительные пакеты и библиотек, предоставляемых [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)].   
  
      **RevoScaleR** пакет включает анализ больших объемов данных высокой производительности, улучшенные версии функций, которые поддерживают распространенные задачи обработки и анализа данных, учеников, оптимизированный для упрощенного алгоритма Байеса, линейной регрессии, модели временных рядов и нейронных сетей и сложных математических библиотек.  
  
     Пакет **RevoPemaR** позволяет разрабатывать в R собственные параллельные алгоритмы внешней памяти.  
  
     Дополнительные сведения об этих пакетов и их использовании см. в разделе [прогнозирующего моделирования и просмотр данных & #40; Учебник: Службы SQL Server R & #41;](../../advanced-analytics/r-services/sql-server-r-services.md).  
  
## Использование источников данных и контекстов вычислений  
 При использовании пакета RevoScaleR для подключения к [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)], существуют некоторые важные новые функции для использования в коде R:  
  
-   [RxSqlServerData](RxSqlServerData.md) — это функция, предоставленных в пакете RevoScaleR для поддержки данных улучшенные возможности подключения к [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].  
  
     Используйте ее в коде R для определения *источника данных*. Объект источника данных указывает сервер и таблицы, где находятся данные, и управляет чтением и записью данных в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].  
  
-    [RxInSqlServer](rxInSqlServer.md) функция может использоваться для указания *контекста вычисления*.  Другими словами, можно указать, где следует выполнять код R: на локальном компьютере или на компьютере, на котором размещена [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] экземпляра.  
  
     Если значение контекста вычисления, влияет на только вычислений, которые поддерживают удаленное выполнение контекста операции R, предоставленный пакетом RevoScaleR и функции, связанные с. Как правило, R решения, основанные на стандартных пакетов CRAN невозможна в контексте удаленного вычислений, хотя они могут выполняться [!INCLUDE[ssNoVersion_md](../../includes/ssnoversion-md.md)] компьютере, если запущены T-SQL. Тем не менее, можно использовать `rxExec` функции для вызова отдельных функций R и выполнить их в удаленный [!INCLUDE[ssNoVersion_md](../../includes/ssnoversion-md.md)].

## Развертывание кода R в рабочей среде  
 Важной частью процесса обработки и анализа данных является обмен результатами с другими специалистами или использование прогнозных моделей для улучшения бизнес-показателей или процессов. В [!INCLUDE[rsql_productname](../../includes/rsql-productname-md.md)]можно легко переместить в рабочую среду готовую модель или скрипт R.  
  
 Дополнительные сведения о перемещение кода для запуска в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)], в разделе [операционализации код R](../../advanced-analytics/r-services/operationalizing-your-r-code.md).  
  
 Обычно процесс развертывания начинается с очистки скрипта от ненужного в рабочей среде кода. При перемещении вычислений ближе к данным может найти способы более эффективно перемещение, обобщения и представления данных, чем делать все, что на R.  
  
 Рекомендуется в обработке данных обратитесь к разработчику базы данных о способах повышения производительности, особенно в том случае, если решение не очистки данных, проектирование, функция может быть более эффективным в SQL. Изменения процессов ETL может потребоваться, чтобы убедиться, что рабочие процессы для создания или оценки модели не дают сбой, и что входных данных доступен в нужном формате.
