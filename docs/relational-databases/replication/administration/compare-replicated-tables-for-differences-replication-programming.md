# сравнить реплицируемые таблицы на предмет различий (программирование репликации)
  Проверка статей используется для определения того, совпадают ли опубликованные данные в статьях таблицы на издателе с данными  на подписчике. В результате могут быть выявлены расхождения. Дополнительные сведения см. в разделе [проверки репликации данных](../../../relational-databases/replication/validate-replicated-data.md). Однако проверка дает только сведения о том, есть расхождение или нет. Никаких подробностей о расхождении данных в исходной таблице и целевой таблице не сообщается. Программа командной строки **tablediff** возвращает подробные сведения о расхождениях в двух таблицах и даже может создать скрипт [!INCLUDE[tsql](../../../includes/tsql-md.md)] для приведения подписки в соответствие с данными на издателе.  
  
> [!NOTE]  
>   **Tablediff** поддерживается только для [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] серверов.  
  
### Поиск различий в реплицированных таблицах с помощью средства tablediff  
  
1.  Запустите средство [tablediff Utility](../../../tools/tablediff-utility.md)из командной строки любого сервера в топологии репликации. Укажите значения следующих параметров.  
  
    -   **-sourceserver** — имя сервера, на котором данные заведомо правильные, обычно это издатель.  
  
    -   **-База данных-источник** — имя базы данных, содержащей нужные данные.  
  
    -   **-sourcetable** — имя исходной таблицы для сравниваемой статьи.  
  
    -   (Необязательно) **-sourceschema** -владелец схемы исходной таблицы, если не схему по умолчанию.  
  
    -   (Необязательно) **-sourceuser** и **- sourcepassword** при использовании проверки подлинности SQL Server для подключения к издателю.  
  
        > [!IMPORTANT]  
        >  По возможности используйте аутентификацию Windows. При необходимости использования проверки подлинности [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] предлагайте пользователям вводить учетные данные во время выполнения. В случае необходимости хранения учетных данных в файле скрипта этот файл следует защищать во избежание несанкционированного доступа.  
  
    -   **-destinationserver** — имя сервера, на котором сравнивается данные, обычно подписчик.  
  
    -   **-destinationdatabase** — имя сравниваемой базы данных.  
  
    -   **-destinationtable** — имя сравниваемой таблицы.  
  
    -   (Необязательно) **-destinationschema** -владелец схемы целевой таблицы, если не схему по умолчанию.  
  
    -   (Необязательно) **-destinationuser** и **- destinationpassword** при использовании [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] проверки подлинности для подключения к подписчику.  
  
        > [!IMPORTANT]  
        >  По возможности используйте аутентификацию Windows. При необходимости использования проверки подлинности [!INCLUDE[ssNoVersion](../../../includes/ssnoversion-md.md)] предлагайте пользователям вводить учетные данные во время выполнения. В случае необходимости хранения учетных данных в файле скрипта этот файл следует защищать во избежание несанкционированного доступа.  
  
    -   (Необязательно) Используйте **- c** сравнения уровня столбца.  
  
    -   (Необязательно) Используйте **- q** для быстрого, строке сравнения счетчик - и схемой.  
  
    -   (Необязательно) Укажите имя файла и путь для **-o** для вывода результатов в файл.  
  
    -   (Необязательно) Укажите таблицу в базе данных подписки, в который необходимо вставить результаты для **-et**. Если таблица уже существует, укажите **-dt** Чтобы удалить таблицу.  
  
    -   (Необязательно) Используйте **-f** для создания [!INCLUDE[tsql](../../../includes/tsql-md.md)] файла для исправления данных на подписчике, чтобы он соответствовал данных на издателе. Используйте **-df** для указания номера для [!INCLUDE[tsql](../../../includes/tsql-md.md)] инструкций в каждом файле.  
  
    -   (Необязательно) Используйте **-rc** и **-ri** Чтобы указать число попыток повтора операции и интервал повторных попыток.  
  
    -   (Необязательно) Используйте **-strict** для обеспечения строгого сравнения схем исходной и целевой таблиц.
