# Имя входа для обновляемых подписок
  Для немедленного обновления, если вы выбрали **репликации** на **обновляемых подписок** странице этого мастера необходимо указать учетную запись с подписчиком, при которой устанавливаются соединения с издателем. 
  
 Подключения используются триггерами, которые запускаются на подписчике и распространяют изменения на издатель. Эта учетная запись необходима, даже если вы выбрали **Ставить изменения в очередь и фиксировать при первой возможности** на **обновляемых подписок** страницы. По умолчанию мастера создания подписки настраивает обновление посредством очередей с возможностью переключения на немедленное обновление при необходимости.  
  
> **ВАЖНО!** Учетная запись, указанная для подключения только должно быть предоставлено разрешение для вставки, обновления и удаления данных в представлениях репликация создает в базе данных публикации. Он не должны предоставляться никакие другие разрешения. Предоставьте разрешения на представления в базе данных публикации, с именем в форме **syncobj_***\< Шестнадцатеричное_число>* для учетной записи, настроенной на каждом подписчике.  
  
 Для этого типа соединения имеются следующие три параметра.  
  
-   Уже определенный связанный или удаленный сервер.  
  
-   Связанный сервер, создаваемый репликацией; соединение устанавливается с использованием учетных данных, заданных в данном мастере.  
  
-   Связанный сервер, создаваемый репликацией; соединение устанавливается с использованием учетных данных пользователя, который вносит изменения на подписчике.  
  
 Первые два параметра можно задать в данном мастере. Последний вариант можно указать только с помощью [sp_link_publication &#40; Transact-SQL &#41;](../../relational-databases/system-stored-procedures/sp-link-publication-transact-sql.md); Укажите значение **1** для параметра **@security_mode**.  
  
## Параметры  
 **Создание связанного сервера, который подключается с использованием следующей проверки подлинности имени входа SQL Server:**  
 Репликация создает связанный сервер с учетными данными, указанными в **входа** и **пароль** полей.  
  
 **Имя входа**  
 Введите [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] запись, обладающую только разрешениями, описанными в этом разделе.  
  
 **Пароль**  
 Введите надежный пароль для имени входа, указанной в **входа**.  
    
 **Использовать уже указанный связанный или удаленный сервер**  
 Для данного параметра необходим уже определенный связанный или удаленный сервер. Дополнительные сведения см. в разделе [связанные серверы &#40; компонент Database Engine &#41;](../../relational-databases/linked-servers/linked-servers-database-engine.md) и [удаленных серверов](../../database-engine/configure-windows/remote-servers.md). Убедитесь в том, что имя входа, используемое для связанного сервера или удаленного сервера, имеет надежный пароль и обладает только разрешениями, описанными в данном разделе.
