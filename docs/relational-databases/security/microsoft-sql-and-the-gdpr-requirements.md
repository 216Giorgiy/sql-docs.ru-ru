# <a name="guide-to-enhancing-privacy-and-addressing-gdpr-requirements-with-the-microsoft-sql-platform"></a>Руководство по повышению конфиденциальности и адресации GDPR требования с помощью платформы Microsoft SQL


## <a name="summary"></a>Сводка
На 25 мая 2018 закон европейских конфиденциальности происходит из-за вступят в силу, задает новую панель глобального прав конфиденциальности, безопасности и соответствия требованиям. Общие стабилизации защиты данных или GDPR, существенно о защите и включение прав конфиденциальности отдельных пользователей и устанавливает требования strict конфиденциальности глобальный управляемый и защищен, при этом соблюдаются выбора отдельных регулирующей личных данных. 

Клиентов Microsoft SQL, которые подлежат GDPR ли управление на основе облака или локальных баз данных или в оба потребуется убедиться, что подходящие данные в их системах баз данных является соответствующее обработано и защищенными в соответствии с GDPR принципы. Это означает, что многие клиенты потребуется для просмотра или изменения их управление базой данных и процедуры, обработки данных особенно сосредоточиться на безопасность обработки данных как настоящим GDPR.

Технологии Microsoft SQL предоставляют множество возможностей встроенных средств безопасности, которые могут помочь снизить риск для данных и улучшения защиты и управляемости данных на уровне базы данных и более поздних. В этом документе рассматриваются эти возможности и являются общими некоторые подходы корпорацией Майкрософт, с помощью Microsoft SQL для достижения цели GDPR конфиденциальности данных.
