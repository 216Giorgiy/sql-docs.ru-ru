## <a name="whats-new-with-polybase-in-sql-server-2019"></a>Новые возможности PolyBase в SQL Server 2019 

PolyBase в SQL Server 2019 теперь может считывать данные из большего числа источников данных. Данные из этих внешних источников данных могут храниться как внешние таблицы в SQL Server. PolyBase также поддерживает вычисление с переносом нагрузки на БД в этих внешних источниках данных, исключая универсальные типы ODBC. 

### <a name="compatible-data-sources"></a>Совместимые источники данных

- SQL Server
- Oracle;
- Teradata
- MongoDB
- **Совместимые** универсальные типы ODBC

  > [!NOTE]
  >
  >PolyBase допускает подключение к внешним источникам данных с помощью сторонних драйверов ODBC. Эти драйверы не предоставляются вместе с PolyBase, но могут работать правильно. Дополнительные сведения см. в [руководстве](polybase-configure-odbc-generic.md) по универсальной конфигурации ODBC в PolyBase.  

## <a name="polybase-vs-linked-servers"></a>PolyBase и связанные серверы

|PolyBase | Связанные серверы|
|--------------------------|--------------------------|  
|Объект в области базы данных|Объект в области экземпляра| 
|Использует драйверы ODBC|Использует поставщики OLEDB| 
| Поддерживает только операции чтения. В будущем поддержка будет расширена.| Поддерживает только операции чтения. В будущем поддержка будет расширена.| 
|Запросы поддерживают горизонтальное масштабирование и перенос нагрузки на БД|Запросы являются однопоточными и поддерживают перенос нагрузки на БД|

|Только обычная проверка подлинности. Улучшения в SQL Server 2019|Обычная и встроенная проверка подлинности|
