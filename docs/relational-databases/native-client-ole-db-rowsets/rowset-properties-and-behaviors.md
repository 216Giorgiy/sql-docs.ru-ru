# <a name="rowset-properties-and-behaviors"></a>Свойства и поведение наборов строк
[!INCLUDE[appliesto-ss-asdb-asdw-pdw-md](../../includes/appliesto-ss-asdb-asdw-pdw-md.md)]
[!INCLUDE[SNAC_Deprecated](../../includes/snac-deprecated.md)]

  Это [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства наборов строк поставщика OLE DB для собственного клиента.  
  
|Идентификатор свойства|Описание|  
|-----------------|-----------------|  
|DBPROP_ABORTPRESERVE|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Этим свойством определяется поведение набора строк после аварийного завершения операции.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента делает наборы строк недействительными после аварийного завершения операции. Объект набора строк практически утратил свою функциональность. Он поддерживает только **IUnknown** операций и выпуска необработанных строк и обработчики методов доступа.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] допустимый набор строк поддерживает поставщик OLE DB для собственного клиента.|  
|DBPROP_ACCESSORDER|R Чтение и запись: чтение и запись<br /><br /> Default: DBPROPVAL_AO_RANDOM<br /><br /> Описание: Порядок доступа. Порядок, в котором должен осуществляться доступ к столбцам в наборе строк.<br /><br /> DBPROPVAL_AO_RANDOM: Столбец может осуществляться в любом порядке.<br /><br /> DBPROPVAL_AO_SEQUENTIALSTORAGEOBJECTS: Столбцы привязаны как хранилища объектов может осуществляться только в последовательном порядке, по порядковому номеру столбца.<br /><br /> DBPROPVAL_AO_SEQUENTIAL: Все столбцы могут быть доступны в последовательном порядке, по порядковому номеру столбца.|  
|DBPROP_APPENDONLY|Это свойство набора строк не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_BLOCKINGSTORAGEOBJECTS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] OLE DB для собственного клиента поставщик хранилища объектов блока с помощью других методов набора строк.|  
|DBPROP_BOOKMARKS DBPROP_LITERALBOOKMARKS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента поддерживает закладки для определения строки набора строк, если DBPROP_BOOKMARKS или DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> Присвоение любому из этих свойств значения VARIANT_TRUE не включает позиционирование набора строк по закладке. Чтобы создать набор строк, поддерживающий позиционирование по закладке, присвойте свойству DBPROP_IRowsetLocate или DBPROP_IRowsetScroll значение VARIANT_TRUE.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщика OLE DB для собственного клиента [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсора для поддержки набора строк, который содержит закладки. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Примечание: Задание этих свойств, которые конфликтуют с другими [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства определение курсора поставщика OLE DB для собственного клиента приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_BOOKMARKS и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_BOOKMARKSKIPPED|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента вернет значение DB_E_BADBOOKMARK, если потребитель указал недопустимую закладку при позиционировании или поиске набора строк с закладками.|  
|DBPROP_BOOKMARKTYPE|R Чтение и запись: только для чтения<br /><br /> По умолчанию: DBPROPVAL_BMK_NUMERIC<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента реализует только числовые закладки. Объект [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] закладкой поставщика OLE DB для собственного клиента является 32-разрядное целое число без знака, имеющее тип DBTYPE_UI4.|  
|DBPROP_CACHEDEFERRED|Это свойство набора строк не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_CANFETCHBACKWARDS DBPROP_CANSCROLLBACKWARDS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента поддерживает обратную выборку и прокрутку в непоследовательных наборах строк. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента создает набор строк с поддержкой курсора, если DBPROP_CANFETCHBACKWARDS и DBPROP_CANSCROLLBACKWARDS имеют значение VARIANT_TRUE. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_CANHOLDROWS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: по умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента возвращает DB_E_ROWSNOTRELEASED при попытке потребителя получить больше строк из набора отложенных изменений, в котором существуют в настоящее время в наборе строк. Это поведение может быть изменено.<br /><br /> Присвоение свойствам DBPROP_CANHOLDROWS и DBPROP_IRowsetChange значения VARIANT_TRUE подразумевает набор строк с закладками. Если оба свойства имеют значение VARIANT_TRUE, **IRowsetLocate** интерфейс доступен на наборе строк и DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Собственные наборы строк поставщика OLE DB для клиента, содержащие закладки поддерживаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсоров.|  
|DBPROP_CHANGEINSERTEDROWS|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Это свойство может устанавливаться только значение VARIANT_TRUE, если набор строк использует курсор, управляемый набором ключей.|  
|DBPROP_COLUMNRESTRICT|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента присваивает свойству значение VARIANT_TRUE при потребителем нельзя добавить столбец в наборе строк. Другие столбцы в наборе строк можно изменять, а сами строки — удалять.<br /><br /> Если свойство имеет значение VARIANT_TRUE, то потребитель по элементу *dwFlags* член структуры DBCOLUMNINFO, чтобы определить, можно ли выполнять запись значение отдельного столбца. Для изменяемых столбцов *dwFlags* принимает значение DBCOLUMNFLAGS_WRITE.|  
|DBPROP_COMMANDTIMEOUT|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: 0<br /><br /> Описание: по умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента время ожидания не истекает в **ICommand::Execute** метод.|  
|DBPROP_COMMITPRESERVE|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: Этим свойством определяется поведение набора строк после операции фиксации.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] допустимый набор строк поддерживает поставщик OLE DB для собственного клиента.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента делает наборы строк недействительными после фиксации. Объект набора строк практически утратил свою функциональность. Он поддерживает только **IUnknown** операций и выпуска необработанных строк и обработчики методов доступа.|  
|DBPROP_DEFERRED|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: При значении VARIANT_TRUE [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента пытается использовать серверный курсор для набора строк. **Текст**, **ntext**, и **изображения** столбцы не возвращаются с сервера, пока они доступны для приложения.|  
|DBPROP_DELAYSTORAGEOBJECTS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента поддерживает режим немедленного обновления для хранения объектов.<br /><br /> Изменения, примененные к данным в объекте последовательного потока, немедленно передаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. Фиксация изменений производится в соответствии с режимом транзакций набора строк.|  
|DBPROP_HIDDENCOLUMNS|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> **Описание:** скрытые число столбцов<br /><br /> Если DBPROP_UNIQUEROWS имеет значение VARIANT_TRUE, то свойство DBPROP_HIDDENCOLUMNS возвращает число дополнительных «скрытых» столбцов, добавленных поставщиком для уникальной идентификации строк в наборе. Эти столбцы возвращаются **IColumnsInfo::GetColumnInfo** и **IColumnsRowset::GetColumnsRowset**. Тем не менее, они не включаются в число строк, возвращенных *pcColumns* аргумента, возвращаемого **IColumnsInfo::GetColumnInfo**.<br /><br /> Чтобы определить общее число столбцов, представленных в *prgInfo* структуры, возвращенный **IColumnsInfo::GetColumnInfo**, включая скрытые столбцы, потребитель должен добавить значение DBPROP_ HIDDENCOLUMNS количество столбцов, возвращаемых из **IColumnsInfo::GetColumnInfo** в *pcColumns*. Если свойство DBPROP_UNIQUEROWS имеет значение VARIANT_FALSE, то свойство DBPROP_HIDDENCOLUMNS равно нулю.|  
|DBPROP_IAccessor DBPROP_IColumnsInfo DBPROP_IConvertType DBPROP_IRowset DBPROP_IRowsetInfo|R Чтение и запись: только для чтения<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента поддерживает эти интерфейсы для всех наборов строк.|  
|DBPROP_IColumnsRowset|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поддерживает поставщик OLE DB для собственного клиента **IColumnsRowset** интерфейса.|  
|DBPROP_IConnectionPointContainer|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: IConnectionPointContainer. Если значение равно VARIANT_TRUE, то набор строк поддерживает заданный интерфейс. Если значение равно VARIANT_FALSE, то набор строк не поддерживает заданный интерфейс. Поставщики, поддерживающие тот или иной интерфейс, должны поддерживать связанное с этим интерфейсом свойство, которое возвращает значение VARIANT_TRUE. Эти свойства служат главным образом для запроса интерфейсов с помощью метода ICommandProperties::SetProperties.|  
|DBPROP_IMultipleResults|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поддерживает поставщик OLE DB для собственного клиента **IMultipleResults** интерфейса.|  
|DBPROP_IRowsetChange DBPROP_IRowsetUpdate|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поддерживает поставщик OLE DB для собственного клиента **IRowsetChange** и **IRowsetUpdate** интерфейсов.<br /><br /> Набор строк, созданный с DBPROP_IRowsetChange, равным VARIANT_TRUE, отражает режимы немедленного обновления.<br /><br /> Если DBPROP_IRowsetUpdate имеет значение VARIANT_TRUE, то DBPROP_IRowsetChange также имеет значение VARIANT_TRUE. Набор строк характеризуется поведением отложенного обновления.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщика OLE DB для собственного клиента [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсора для поддержки набора строк **IRowsetChange** или **IRowsetUpdate**. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_IRowsetIdentity|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поддерживает поставщик OLE DB для собственного клиента **IRowsetIdentity** интерфейса. Если набор строк поддерживает этот интерфейс, то любые два дескриптора строк, относящихся к одной базовой строке, всегда представляют одни и те же данные и состояние. Потребители могут вызвать метод **IRowsetIdentity:: IsSameRow** метод для сравнения двух дескрипторы строк, чтобы увидеть, если они ссылаются на один и тот же экземпляр строки.|  
|DBPROP_IRowsetLocate DBPROP_IRowsetScroll|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента могут предоставлять **IRowsetLocate** и **IRowsetScroll** интерфейсов.<br /><br /> Если DBPROP_IRowsetLocate имеет значение VARIANT_TRUE, CANFETCHBACKWARDS и DBPROP_CANSCROLLBACKWARDS также имеют значение VARIANT_TRUE.<br /><br /> Если DBPROP_IRowsetScroll имеет значение VARIANT_TRUE, то DBPROP_IRowsetLocate также имеет значение VARIANT_TRUE, поэтому оба интерфейса доступны в наборе строк.<br /><br /> Закладки необходимы для обоих интерфейсов. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента присваивает DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS значение VARIANT_TRUE, если потребитель запрашивает один из этих интерфейсов.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщика OLE DB для собственного клиента [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсоры для поддержки **IRowsetLocate** и **IRowsetScroll**. Дополнительные сведения см. в разделе [наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Настройка этих свойств, которые конфликтуют с другими [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства определение курсора поставщика OLE DB для собственного клиента приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_IRowsetScroll и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_IRowsetResynch|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента предоставляет **IRowsetResynch** интерфейса по требованию. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента могут предоставлять доступ к интерфейсу в любом наборе строк.|  
|DBPROP_ISupportErrorInfo|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента предоставляет **ISupportErrorInfo** интерфейс наборов строк.|  
|DBPROP_ILockBytes|Этот интерфейс не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_ISequentialStream|R Чтение и запись: чтение и запись<br /><br /> По умолчанию: значение VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента предоставляет **ISequentialStream** интерфейс для поддержки больших данных переменной длины, хранящихся в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].|  
|DBPROP_IStorage|Этот интерфейс не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_IStream|Этот интерфейс не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать свойство возникает ошибка.|
