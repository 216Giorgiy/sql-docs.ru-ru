# <a name="dashboards-in-includename-sosincludesname-sos-shortmd"></a>Панели мониторинга в [!INCLUDE[name-sos](../includes/name-sos-short.md)]

Чтобы просмотреть панель мониторинга, щелкните правой кнопкой мыши сервер или базу данных и выбрать **управление**.

![В примере информационной панели](media/dashboards/sample-dashboard.png)

**Свойства сервера** содержит свойства сервера, включая версию, выпуск, имя компьютера и версия операционной системы.

**Задачи** содержит commons задач, таких как восстановление и новый запрос.

**Поиск баз данных** легко находить существующих баз данных, сохраненных на сервере, включая таблицы.

**Состояние архивации** легко находить состояния резервного копирования для существующих баз данных.

## <a name="insight-detail"></a>Подробные сведения
Всплывающее окно сведений анализ предоставляет более подробными сведениями о связанных insight мини-приложения. 
- Анализ мини-приложение отображает представление сводки по быстро count, линии, диаграммы и т. д. 
- Всплывающее окно сведений Insight сведения «детализировать», вывод более глубокого понимания данных для каждого из перечисленных в высокоуровневое представление мини-приложении. 
  - Содержимое всплывающее окно сведений определяется отдельный запрос SQL на запрос основного мини-приложения. 

Нет необходимости набора для запроса представления сведений, но макет является стандартным.
- В верхней части представления, всегда является столбец 2 представление «Сводка». Какие столбцы использовать определяются «label» и «value» свойства конфигурации JSON
- После нажатия кнопки для строки в сводной таблице, нижней половине всплывающего окна будут отображаться полный набор сведения о столбце для этой строки.

## <a name="insight-actions"></a>Анализ действий
Мини-приложение анализ и понимание сведений можно легко прийти плана действий, чтобы устранить проблему, и управлять ими. Например будет рассматривать выполнение инструкции DBCC CHECKDB, читать журналы ошибок или восстановления базы данных, когда база данных находится в состоянии восстановления, ожидания. Или он может иметь любое из действий, которые необходимо выполнить.

С помощью [!INCLUDE[name-sos](../includes/name-sos-short.md)]элемента конфигурации представление действия, вы можете сопоставить встроенные действия, как восстановить или подключить собственные действия, определенные с помощью сценария sql.

> Конфигурации пользовательских действий, с помощью скрипта sql находится в стадии разработки, и оно еще не доступны в личной предварительной версии сборки.

## <a name="sample-insight-action-definition"></a>Пример определения действия подробные сведения

```"actions"{}``` Определяет действие подробные сведения. Действия могут быть определены по определенной области, например ```"server"```, ```"database"``` и т. д и [!INCLUDE[name-sos](../includes/name-sos-short.md)] передает сведения о текущем соединении контекст действия. 

Например, при запуске действия восстановления для базы данных WideWorldImporters, ```"database": "${Database}"``` указывает определения для передачи ```Database``` значение столбца в результатах запроса, для операции восстановления. Затем запускается операции восстановления базы данных. ```"types"``` — это массив json и несколько действий, которые могут быть перечислены в массиве. По сути, оно становится контекстного меню в диалоговом окне сведений анализ этого пользователя щелкните и выполнение действия. 

> [!INCLUDE[name-sos](../includes/name-sos-short.md)] Предварительный просмотр 0.17.1 включил «backup», «восстанавливается», «создать запрос» и «новый-базы данных» как типы действий.

```json
"details": {
    "queryFile": "./sql/database_state_detail.sql",
    "label": {...},
    "value": "state",
    "actions": {
        "database": "${Database}",
        "types": ["restore"]
    }
}
```
