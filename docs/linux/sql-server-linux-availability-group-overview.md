# <a name="availability-groups-for-sql-server-on-linux"></a>Группы доступности для SQL Server в Linux

[!INCLUDE[tsql-appliesto-sslinux-only](../includes/tsql-appliesto-sslinux-only.md)]

Группы доступности AlwaysOn в SQL Server является высокого уровня доступности (HA), решение для горизонтального масштабирования и аварийного восстановления (DR). Он предоставляет высокого уровня ДОСТУПНОСТИ для группы хранения с прямым подключением базы данных. Он поддерживает несколько баз данных-получателей для интеграции высокого уровня ДОСТУПНОСТИ и аварийного восстановления, сбой автоматического обнаружения, быстро прозрачную отработку отказа и балансировки нагрузки для чтения. Это широкий набор возможностей позволяет добиться оптимальной доступности соглашения об уровне обслуживания для рабочих нагрузок.

Группы доступности SQL Server впервые появились в SQL Server 2012 и улучшены с каждым выпуском. Эта функция доступна в Linux. Для размещения рабочих нагрузок SQL Server с помощью непрерывности строгие бизнес-требований, запустите группы доступности на всех поддерживаемых [распределения ОС Linux](sql-server-linux-release-notes.md). Кроме того все возможности, которые позволяют группы доступности гибким, интеграции и эффективное решение аварийного восстановления высокой ДОСТУПНОСТИ в Linux, также доступны. К ним относятся следующие объекты. 

- **Отработка отказа базы данных несколькими** группа доступности поддерживает среду отработки отказа для набора пользовательских баз данных, известных как базы данных доступности.
- **Быстрое обнаружение сбоя и переход на другой ресурс** как ресурс в кластере высокой доступности группы доступности используются преимущества аналитики встроенных кластера для обнаружения немедленно отработки отказа и действие отработки отказа.
- **Прозрачная отработка отказа, используя виртуальные IP-адреса** позволяет клиенту использовать одной строке подключения для источника в случае отработки отказа. Требует интеграции с помощью диспетчера кластеров.
- **Несколько баз данных-получателей синхронной и асинхронной** группа доступности поддерживает до восьми вторичных реплик. С синхронными репликами первичная реплика ожидает возможности выполнения фиксации транзакции, первичная реплика ожидает транзакции записываются на диск на журнал транзакций. Первичная реплика не ждет получения записи для асинхронного синхронных реплик.  
- **Переход на другой ресурс вручную или автоматически** отработка отказа синхронной вторичную реплику может инициироваться автоматически по кластерам, или по требованию администратор базы данных.
- **Активные вторичные реплики для чтения и резервного копирования рабочих нагрузок** можно настроить один или несколько вторичных реплик для поддержки доступа только для чтения базы данных-получатели и разрешить создание резервных копий баз данных-получателей.
- **Автоматическое заполнение** SQL Server автоматически создает вторичные реплики для каждой базы данных в группе доступности.
- **Маршрутизация только для чтения** SQL Server направляет входящие подключения к прослушивателю группы доступности на вторичную реплику, которая настроена на разрешение рабочих нагрузок только для чтения. 
- **Мониторинг и отработка отказа триггер работоспособности уровня базы данных** расширенный мониторинг на уровне базы данных и диагностики. 
- **Конфигураций аварийного восстановления** с распределенные группы доступности или настройка группы доступности с несколькими подсетями. 
- **Возможности масштабирования чтения** в SQL Server 2017 г. можно создать группу доступности с или без высокой ДОСТУПНОСТИ для масштабирования операций только для чтения. 


Дополнительные сведения о группах доступности SQL Server см. в разделе [группы обеспечения доступности AlwaysOn в SQL Server](http://msdn.microsoft.com/library/hh510230.aspx).

## <a name="availability-group-terminology"></a>Терминология группы доступности

Группа доступности поддерживает среду отработки отказа для дискретного набора пользовательских баз данных - известных как базы данных доступности — которые выполняют отработку отказа совместно. Группа доступности поддерживает один набор первичных баз данных чтения записи и от одной до четырех наборов соответствующих вторичных баз данных. Кроме того, базы данных-получатели можно сделать доступными только для чтения или для некоторых операций резервного копирования. Группа доступности определяет набор из двух или более партнеров отработки отказа, известных как реплики доступности. Реплики доступности являются компонентами группы доступности. Дополнительные сведения см. [Обзор из групп доступности AlwaysOn (SQL Server)](http://msdn.microsoft.com/library/ff877884.aspx).

Ниже описаны основные части решения группы доступности SQL Server:

 группа доступности  
 Контейнер для набора баз данных, *базы данных доступности*, которые совместно отрабатывают отказ.  
  
 база данных доступности  
 База данных, принадлежащая к группе доступности. Для каждой базы данных доступности группа доступности поддерживает одну копию для чтения и записи ( *первичная база данных*) и до восьми копий только для чтения (*вторичные базы данных*).  
  
 первичная база данных  
 Копия базы данных доступности для чтения и записи.  
  
 база данных-получатель  
 Копия базы данных доступности только для чтения.  
  
 реплика доступности  
 Экземпляр группы доступности, на которой размещается на конкретном экземпляре SQL Server и поддерживает локальную копию каждой базы данных доступности, которая принадлежит группе доступности. Существует два типа реплик доступности: одна *первичная реплика* и до восьми *вторичных реплик*.  
  
 первичная реплика  
 Реплика доступности делает базы данных-источники доступными для соединений чтения и записи с клиентов, а также отправляет записи журнала транзакций для каждой базы данных-источника каждой вторичной реплике.  
  
 вторичная реплика  
 Реплика доступности, которая поддерживает вторичную копию каждой базы данных доступности и служит потенциальным назначением отработки отказа для группы доступности. При необходимости вторичная реплика может поддерживать доступ только для чтения к базам данных-получателям и создание резервных копий баз данных-получателей.  
  
 Прослушиватель группы доступности  
 Имя сервера, к которому могут подключаться клиенты, чтобы получить доступ к базе данных из первичной или вторичной реплики группы доступности. Прослушиватели группы доступности направляют входящие соединения на первичную реплику или на доступную только для чтения вторичную реплику.  


## <a name="new-in-sql-server-2017-for-availability-groups"></a>Новые возможности SQL Server 2017 г. для групп доступности

2017 г. SQL Server предоставляет новые возможности для групп доступности.

**CLUSTER_TYPE** использования с `CREATE AVAILABILITY GROUP`. Определяет тип диспетчера кластера сервера, который управляет группа доступности. Может принимать одно из следующих типов:

   - **WSFC** Winows server отказоустойчивого кластера. В Windows это значение по умолчанию для CLUSTER_TYPE.
   - **ВНЕШНИЕ** диспетчер кластеров, не отказоустойчивого кластера Windows server — например, в Linux с Pacemaker.
   - **Нет** без диспетчера кластеров. Используется для группы доступности чтения шкалы.

Дополнительные сведения об этих параметрах см. в разделе [CREATE AVAILABILITY GROUP](http://msdn.microsoft.com/library/ff878399.aspx) или [ALTER AVAILABILITY GROUP](http://msdn.microsoft.com/library/ff878601.aspx).

**Фиксирует гарантии на синхронных вторичных реплик.**

Use `required_synchronized_secondaries_to_commit`with `CREATE AVAILABILITY GROUP` or `ALTER AVAILABILITY GROUP`. Когда `required_synchronized_secondaries_to_commit` задано значение больше 0, транзакции в основной реплике базы данных будет ожидать до момента фиксации транзакции на указанное число **синхронного сервера-получателя** журналов транзакций базы данных реплики. Если недостаточно синхронные вторичные реплики не находятся в сети, все подключения к первичной реплике будут отклоняться до возобновления связи с достаточно вторичными репликами.

**Группы доступности чтения шкалы**

Создание группы доступности без кластера для поддержки рабочих нагрузок чтения шкалы. В разделе [группы доступности чтения шкалы](../database-engine/availability-groups/windows/read-scale-availability-groups.md).
