# <a name="conversion-settings-mysqltosql"></a>Параметры преобразования (MySQLToSQL)
**«Настройки»** вкладка позволяет пользователю задавать параметры уровня узла. Вкладке будут доступны на следующих узлах метабазы:  
  
-   Узел базы данных  
  
-   Категория функции  
  
-   Узел функции  
  
-   Категории таблицы  
  
-   Узел таблицы  
  
## <a name="specifications"></a>Спецификации:  
**Параметры** вкладке имеется два параметра пользователя viz.:  
  
1.  Функции преобразования  
  
2.  Преобразование таблицы  
  
Эти параметры будут доступны на основе типа узла метабазы. Например преобразование функции связанных параметр будет недоступен в узле таблицы  
  
> [!NOTE]  
> -   Изменения, внесенные пользователем в рабочей области проекта будет сохранен как файл отдельный предпочтения.  
> -   Расширение этого файла будет **ccprefs**.  
  
1.  **Настройка функции преобразования:**  
  
    1.  На этой вкладке содержатся **«Принудительное преобразование функции»** параметр. Этот параметр может иметь одно из следующих четырех значений:  
  
        -   Преобразовать в соответствии с параметрами проекта [наследуется]  
  
        -   Всегда преобразовывать функции  
  
        -   Всегда преобразовывать в процедуру  
  
        -   Преобразовать в соответствии с параметрами проекта  
  
    2.  В зависимости от параметров функции будут преобразованы на функцию или хранимую процедуру.  
  
    3.  Параметры пользователя сохраняются в файле каскадные параметры после нажатия кнопки **применить** кнопки.  
  
2.  **Настройка преобразования таблицы:**  
  
    1.  На этой вкладке содержатся **«Отключение ROWID поколения дополнительный столбец»** параметр. Этот параметр может иметь одно из следующих четырех значений:  
  
        -   Преобразовать в соответствии с параметрами проекта [наследуется]  
  
        -   Да  
  
        -   Нет  
  
        -   Преобразовать в соответствии с параметрами проекта  
  
    2.  Если **«Да»**, этот параметр запрещает создание создания дополнительный столбец ROWID в целевых таблицах.  
  
    3.  Параметры пользователя сохраняются в файле каскадные параметры после нажатия кнопки **применить** кнопки.
