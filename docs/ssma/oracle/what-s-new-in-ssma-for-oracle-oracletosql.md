# <a name="whats-new-in-ssma-for-oracle-oracletosql"></a>Новые возможности SSMA для Oracle (OracleToSQL)
В этом разделе перечислены SSMA для Oracle изменения в каждом выпуске.  

## <a name="ssma-v77"></a>SSMA v7.7
V7.7 выпуск SSMA для Oracle содержит следующие изменения:
- SSMA для Oracle была дополнена целевых исправлений, которые улучшают метрики качества и преобразования.
- На основе популярных спроса, 32-разрядной версии SSMA для Oracle — обратно. По сравнению с предыдущей реализации (до v7.4), существует два пакета установщика, но они не могут быть установлены параллельно. В результате необходимо выбрать наиболее подходящий на основе компонентов подключения к версии, что у вас есть. Всегда предпочтительнее использовать 64-разрядной версии, если это возможно.
- Поддержка 2017 г. SQL Server официальный Oracle расширения пакета также поддерживается в системе Linux (новый параметр удаленной установки). Обратите внимание, что пакет расширения функциональности только при установке в Linux, как тест-инженера и возможности миграции данных на сервере не поддерживается 
- SSMA для Oracle позволяет переносить материализованные представления в виде обычных таблиц (настраивается через параметры **параметры проекта** -> **синхронизации**  ->  **Обнаружение резервное копирование таблиц для материализации представлений**).

> [!IMPORTANT]
> SSMA v7.4 и более поздние версии .net 4.5.2 является необходимым условием установки.

## <a name="ssma-v76"></a>SSMA v7.6
V7.6 выпуск SSMA для Oracle была расширена с целевой исправления, повышения качества и преобразование метрик и поддержка 2017 г. SQL Server (Предварительная версия). Поддержка 2017 г. SQL Server в Windows и Linux в общедоступной предварительной версии и не должен использоваться для миграции в рабочей среде.

> [!IMPORTANT]
> SSMA v7.4 и более поздние версии .net 4.5.2 является необходимым условием установки и 32-разрядной версии средства больше не поддерживаются.

## <a name="ssma-v75"></a>SSMA v7.5
V7.5 выпуск SSMA для Oracle содержит следующие изменения:
- Дополнено несколько улучшений, чтобы обеспечить большую доступность для людей с ограниченными возможностями.
- Обновлен для повышения качества и преобразование метрику с целевой исправления, такие как Улучшенная обработка типов данных даты и float, во время переноса данных, на основе отзывов клиентов.

> [!IMPORTANT]
> Версия .net 4.5.2 является необходимым условием для установки SSMA v7.5. Кроме того начиная с v7.4, 32-разрядной версии SSMA прекращается.

## <a name="ssma-v74"></a>SSMA v7.4
V7.4 выпуск SSMA для Oracle содержит следующие изменения:

- SSMA для Oracle теперь поддерживает хранилище данных SQL Azure в качестве целевой платформы для миграции.

    ![Окно нового проекта.](../media/new-project.png)
  - Поддерживает параметры хранения хранилища данных, как показано на следующем рисунке:

    ![Параметры хранилища для хранилища данных](../media/storage-options_red.png)
  - Поддерживает параметры распределения данных, как показано на следующем рисунке:

    ![Распределение данных для хранилища данных](../media/data-distribution_red.png)

- **Время ожидания запроса** теперь доступен при обнаружении объекта схемы в исходной и целевой.

    ![параметр времени ожидания запроса](../media/query-timeout_red.png)

- Метрика качества и преобразование усовершенствованы целевой исправления, на основе отзывов клиентов.

> [!IMPORTANT]
> Версия .net 4.5.2 является необходимым условием для установки SSMA v7.4. Кроме того начиная с v7.4, 32-разрядной версии SSMA прекращается.

## <a name="ssma-v73"></a>SSMA v7.3
V7.3 выпуск SSMA для Oracle содержит следующие изменения:
- Улучшенное качество и преобразование метрику с целевой исправления на основе отзывов клиентов.
- SSMA расширения среды .NET framework, представляемый через следующие элементы:
  - Экспорт функций в проект SQL Server Data Tools (SSDT).
    -   Теперь можно экспортировать скриптов схемы с SSMA для проекта SSDT. Скрипты схемы можно использовать для внесения изменений дополнительные схемы и развертывания базы данных.
![Сохранить как SSDT проект-команда](../media/export-schema-scripts_red.png)
  - Библиотеки, которые могут быть использованы SSMA для выполнения пользовательских преобразований.
    - Теперь можно создать код, который может обработать синтаксис пользовательского преобразования и преобразования, которые не были обработаны ранее SSMA.
      - Инструкции о том, как создать пользовательский преобразователь доступны в этой записи блога [расширение SQL Server Migration Assistant по возможности конвертации](https://blogs.msdn.microsoft.com/datamigration/2017/02/21/2185/).
      - Пример проекта для преобразования можно загрузить это [блога](https://blogs.msdn.microsoft.com/datamigration/ssmafororacleconversionsample/).


## <a name="ssma-v72"></a>SSMA v7.2
V7.2 выпуск SSMA для Oracle содержит следующие изменения:
- Улучшенное качество и преобразование метрику с целевой исправления на основе отзывов клиентов.
- Улучшения телеметрии для предоставления более точек данных для устранения проблем клиентов и повысить скорость преобразования элемента SSMA.

## <a name="ssma-v71"></a>SSMA v7.1
V7.1 выпуск SSMA для Oracle содержит следующие изменения:
- 2017 г. SQL Server в Windows и Linux CTP-версия 1 больше не поддерживаемую целевую платформу для миграции. Эта функция в technical preview и позволяет перемещение схему и данные в целевых серверов SQL.
- SSMA теперь поддерживает автоматическое обновление, чтобы загрузить последнюю версию SSMA как только он доступен.
- Теперь может устанавливаться SSMA двоичные файлы доставляются через файлы пакета установщика Windows (.msi).

## <a name="may-2016"></a>Май 2016 г.  
Май 2016 г. выпуск SSMA для Oracle содержит следующие изменения:  

-   Добавлена поддержка SQL Server 2016.
-   Добавлено преобразование архивные таблицы Oracle переключение обратно в темпоральных таблиц SQL Server.
-   Добавленный преобразования политики VPD Oracle, преобразование в объекты политика SQL Server (Безопасность на уровне строк для Oracle).
-   Снижение времени начальной загрузки для Oracle.
-   Улучшенные средства синтаксического анализа и арбитр конфликтов.
-   Удалить проверку установщика для .net 2.0.
-   Обновленный пакет расширения зависимость от .net 3.5 в .net 4.0.
-   Исправлена «сохранить проект» и команды «Открыть проект» для консоли SSMA.
-   Команда фиксированный «securepassword» для консоли SSMA.
-   Исправлена подсчет объектов для начальной загрузки.
-   Исправлено преобразование символьных типов данных для Oracle.
-   Исправлена ошибка в глобальных параметрах.
  
## <a name="march-2016"></a>Март 2016 г.  
Март 2016 г. Предварительный выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавлена поддержка для миграции в SQL Server 2016.  
-   Добавлена поддержка миграции Oracle безопасность на уровне строк (с некоторыми ограничениями).  
-   Добавлена поддержка для переноса таблиц Oracle в памяти для хранилища столбцов SQL Server.

-   Исправлена запросов схем медленных Oracle (RFC 5076207).  
-   Фиксированный подключиться к Azure из консоли.

## <a name="july-2014"></a>Июль 2014 г.  
Июль 2014 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавлена поддержка для базы данных SQL Azure.  
-   Расширение пакета функции переданы схему поддерживает базу данных SQL Azure.  
-   Добавлена поддержка Oracle материализованные представления.  
-   Добавлена поддержка памяти SQL Server 2014 для оптимизированных таблиц.  
-   Повышение производительности включены проверены для баз данных с более чем 10k объектов.  
-   Усовершенствования пользовательского интерфейса для работы с большим количеством объектов.  
-   Добавить выделение схем «хорошо известный» LOB.  
-   Включить увеличение скорости преобразования.  
-   Добавлена поддержка для отображения объекта подсчитывает в пользовательском Интерфейсе.  
-   Размер ограниченной отчета на более 25%.
-   Улучшенные сообщения об ошибках для непроанализированного конструкций.

-   Добавлена поддержка MS SQL Server 2014.  
-   Добавлена поддержка оптимизации Oracle 12 и запроса.  
-   Исправленные ошибки относительно преобразование в Azure.

## <a name="january-2012"></a>Январь 2012 г.  
Январь 2012 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавлена поддержка для входных параметров RowType и типом записи по умолчанию NULL.  
  
## <a name="july-2011"></a>Июль 2011 г.  
Июль 2011 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавлена поддержка для преобразования последовательности Oracle [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] генератора последовательности «Denali».  
-   Усовершенствованные отчетов во время переноса данных об ошибках.  
-   Улучшенное преобразование инструкцию, используя зарезервированные слова.  
-   Улучшенная неявное преобразование значения даты в функции.  
  
## <a name="april-2011"></a>Апреля 2011 г.  
Апреля 2011 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Консолидированные продукта «SSMA для Oracle», который поддерживает [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2005, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 и [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
-   Добавлена поддержка для подключения и переход на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
-   Модуль переноса расширенные данные клиента с поддержкой параллельного переноса данных.  
-   Большая производительность миграции с простой и массового войти в модели восстановления.  
-   Добавлена поддержка для обеспечения обратной совместимости проекты, созданные в предыдущих версиях SSMA (версии 4.0 и версия 4.2).  
-   Добавлена возможность установить SSMA для Oracle версии 5.0 продукта на одном компьютере (SxS) с более ранними версиями SSMA (версии 4.0 и версия 4.2).  
-   Добавлена поддержка для отчетов, определяемые пользователем типы (включая подтип, VARRAY, ВЛОЖЕННУЮ ТАБЛИЦУ, объект таблицы и представление объектов) и их применения в блоках PL/SQL специальные вывода сообщений об ошибках.  

## <a name="july-2010"></a>Июль 2010 г.  
Июль 2010 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавлена поддержка для миграции в SQL Server 2008 R2.  
-   Добавить новое приложение консоли SSMA для выполнения командной строки.  
-   Добавлена поддержка для переноса данных с помощью серверных и клиентских данных миграции ядер.  
-   Добавлена поддержка инструкции «SELECT настраиваемый» в переноса данных.  
-   Добавлена поддержка миграции из Oracle 11g R2.  
  
## <a name="june-2008"></a>Июнь 2008 г.  
Июнь 2008 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Усовершенствования для отчета оценки, включая дополнительную информацию для синонимов, необработанный источник объектов синтаксическому разбору, панелей и удаление эмблема SQL Server и сохранение структуры.  
-   Усовершенствования в преобразовании объекта:  
    -   Пакеты DBMS_LOB, преобразование DBMS_SQL добавлен.  
    -   Присоединяет пересмотра преобразования.  
    -   Изменение коллекции и записи преобразования, теперь преобразование записей в простых случаях освобождены с помощью отдельные переменные для каждого поля.  
    -   Усовершенствования реализации записей и коллекций.  
    -   Ранжирующие функции агрегирования добавлен.  
    -   Добавлено предложение ROLLUP или CUBE.  
    -   Улучшения для NEXTVAL/CURVAL.  
    -   Группировка в предложении SET, группирующих наборов и группировки идентификатор были добавлены столбцы.  
    -   СЛИЯНИЕ добавлены инструкции.  
    -   Поддержка преобразования записей и коллекций, как типы данных CLR, добавленные и новые типы даты и времени.  
-   Добавлены новые возможности для тест-инженер. Таблиц теперь можно протестировать как объектами с помощью тестировщик, порядок вызова нескольких объектов тестируемым в тестовом случае может быть изменен, пользователь может проверить процедуры и функции с помощью записи и коллекций, как параметры и возвращаемые значения и зависимости, которые анализатор был добавлен для проверки только используемые таблицы.  
  
## <a name="august-2007"></a>Августа 2007 г.  
Августа 2007 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Добавить новый компонент инженер-ИСПЫТАТЕЛЬ позволяет создавать, управлять и выполнению тестовых случаев, чтобы проверить преобразованный код SQL.  
-   Добавлена поддержка для преобразования подтипы Oracle, коллекций и локальные модули были добавлены к преобразователь SQL.  
-   Добавлена новая функция синхронизации позволяет синхронизировать конкретных объектов с базой данных SQL Server.  
-   Добавлены новые параметры преобразования.  
  
## <a name="april-2007"></a>Апреля 2007 г.  
Выпуск SSMA для Oracle апреля 2007 г. был первоначального выпуска.
