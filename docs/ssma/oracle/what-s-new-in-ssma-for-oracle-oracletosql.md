# <a name="what39s-new-in-ssma--for-oracle-oracletosql"></a>Какой &#39; новые возможности SSMA для Oracle (OracleToSQL)
В этом разделе перечислены SSMA для Oracle изменения в каждом выпуске.  

## <a name="may-2016"></a>Май 2016 г.  
Май 2016 г. выпуск SSMA для Oracle содержит следующие изменения:  

1.  Добавлена поддержка SQL Server 2016
2.  Добавлено преобразование архивные таблицы Oracle переключение обратно в темпоральных таблиц SQL Server
3.  Добавленный преобразования политики VPD Oracle, преобразование в объекты политика SQL Server (Безопасность на уровне строк для Oracle)
4.  Снижение времени начальной загрузки для Oracle
5.  Улучшенные средства синтаксического анализа и арбитр конфликтов
6.  Удалить проверку установщика для .net 2.0
7.  Обновленный пакет расширения зависимость от .net 3.5 .net 4.0
8.  Исправлена «сохранить проект» и команды «Открыть проект» для консоли SSMA
9.  Команда фиксированный «securepassword» для консоли SSMA
10. Исправлена подсчет объектов для начальной загрузки
11. Исправлена, преобразование символьных типов данных для Oracle
12. Исправлена ошибка в глобальных параметрах
  
## <a name="march-2016"></a>Март 2016 г.  
Март 2016 г. Предварительный выпуск SSMA для Oracle содержит следующие изменения:  
  
1.  Поддерживает миграцию на SQL Server 2016  
  
2.  Поддержка миграции Oracle безопасность на уровне строк (с некоторыми ограничениями)  
  
3.  Поддержка миграции Oracle в памяти таблиц в хранилище столбцов SQL Server  
  
## <a name="january-2016"></a>Январь 2016 г.  
Январь 2014 г обслуживания выпуск SSMA для Oracle содержит следующие изменения:  
  
1.  Добавлена поддержка для кластеризованных индексов  
  
2.  Фиксированные запросы схем медленных Oracle (RFC 5076207)  
  
3.  Фиксированный подключиться к Azure из консоли  
  
4.  Пункт меню журнала добавлено представление SSMA (RFC 5706203)  
  
5.  Добавлены телеметрии  
  
## <a name="july-2014"></a>Июль 2014 г.  
Июль 2014 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
1.  Поддержка базы данных Azure SQL  
  
2.  Функциональные возможности пакета расширения перемещен в схему для поддержки базы данных SQL Azure  
  
3.  Поддержка Oracle материализованные представления  
  
4.  Поддержка памяти SQL Server 2014 для оптимизированных таблиц  
  
5.  Повышение производительности для проверки базы данных с более чем 10k объектов  
  
6.  Улучшения пользовательского интерфейса для обработки большого числа объектов  
  
7.  Выделение «хорошо известный» LOB схем  
  
8.  Увеличение скорости преобразования  
  
9. Отображение счетчиков объектов в пользовательском Интерфейсе  
  
10. Уменьшение размера отчета на более 25%  
  
11. Улучшенные сообщения об ошибках для непроанализированного конструкций.  
  
## <a name="april-2014"></a>Апрель 2014 г.  
Апрель 2014 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
1.  Добавлена поддержка MS SQL Server 2014.  
  
2.  Добавлена поддержка оптимизации Oracle 12 и запроса.  
  
3.  Исправленные ошибки относительно преобразование в Azure.  
  
4.  Исправленные ошибки относительно страницы невидимой отчетов в IE 10.  
  
## <a name="january-2012"></a>Январь 2012 г.  
Январь 2012 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Входные параметры для поддержки RowType и типом записи по умолчанию NULL.  
  
## <a name="july-2011"></a>Июль 2011 г.  
Июль 2011 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Поддержка для преобразования последовательности Oracle [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] генератора последовательности «Denali».  
  
-   Усовершенствованные отчетов во время переноса данных об ошибках.  
  
-   Улучшенное преобразование инструкцию, используя зарезервированные слова.  
  
-   Улучшенная неявное преобразование значения даты в функции.  
  
## <a name="april-2011"></a>Апреля 2011 г.  
Апреля 2011 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Консолидированные продукта «SSMA для Oracle», который поддерживает [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2005, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 и [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
  
-   Поддержка подключения и перенос на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
  
-   Теперь клиент стороны миграции подсистема обработки данных, поддержка параллельного переноса данных.  
  
-   Большая производительность миграции с простой и массового войти в модели восстановления.  
  
-   Обратная совместимость проекты, созданные в предыдущих версиях SSMA (версии 4.0 и версия 4.2).  
  
-   SSMA для Oracle версии 5.0 продукта могут быть установлены параллельно (SxS) с более ранними версиями SSMA (версии 4.0 и версия 4.2).  
  
-   Поддержка определяемых пользователем типов (включая подтип, VARRAY, ВЛОЖЕННУЮ ТАБЛИЦУ, объект таблицы и представление объектов) и их применения в блоках PL/SQL специальные вывода сообщений об ошибках.  
  
## <a name="july-2010"></a>Июль 2010 г.  
Июль 2010 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Поддержка миграции в SQL Server 2008 R2  
  
-   Новое приложение консоли SSMA для выполнения командной строки  
  
-   Поддержка переноса данных с помощью обработчиков миграции данных стороны клиента и стороне сервера  
  
-   Поддержка инструкция «SELECT настраиваемый» в переноса данных  
  
-   Поддержка миграции из Oracle 11g R2  
  
## <a name="june-2008"></a>Июнь 2008 г.  
Июнь 2008 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Усовершенствования в отчете оценки были выполнены. Он включает дополнительные сведения для синонимов, необработанный источник объектов синтаксическому разбору, панелей и удаление эмблема SQL Server и сохранение структуры.  
  
-   Усовершенствования в преобразовании объекта:  
  
    -   Пакеты DBMS_LOB, преобразование DBMS_SQL добавлен.  
  
    -   Присоединяет пересмотра преобразования.  
  
    -   Изменение коллекции и записи преобразования, теперь преобразование записей в простых случаях освобождены с помощью отдельные переменные для каждого поля.  
  
    -   Усовершенствования реализации записей и коллекций.  
  
    -   Ранжирующие функции агрегирования добавлен.  
  
    -   Добавлено предложение ROLLUP или CUBE.  
  
    -   Улучшения для NEXTVAL/CURVAL.  
  
    -   Группировка в предложении SET, группирующих наборов и группировки идентификатор были добавлены столбцы.  
  
    -   СЛИЯНИЕ добавлены инструкции.  
  
    -   Поддержка преобразования записей и коллекций, как типы данных CLR, добавленные и новые типы даты и времени.  
  
-   Были добавлены новые возможности для тест-инженер. Таблиц теперь можно протестировать как объектами с помощью тестировщик, порядок вызова нескольких объектов тестируемым в тестовом случае может быть изменен, пользователь может проверить процедуры и функции с помощью записи и коллекций, как параметры и возвращаемые значения и зависимости, которые анализатор был добавлен для проверки только используемые таблицы.  
  
## <a name="august-2007"></a>Августа 2007 г.  
Августа 2007 г. выпуск SSMA для Oracle содержит следующие изменения:  
  
-   Новый тест-ИНЖЕНЕР компонент позволяет создавать, управлять и выполнению тестовых случаев, чтобы проверить преобразованный код SQL.  
  
-   Преобразование подтипы Oracle, коллекций и локальные модули будут добавлены преобразователь SQL.  
  
-   Новая функция синхронизации позволяет синхронизировать конкретных объектов с базой данных SQL Server.  
  
-   Добавлены новые параметры преобразования.  
  
## <a name="april-2007"></a>Апреля 2007 г.  
Выпуск SSMA для Oracle апреля 2007 г. был первоначального выпуска.
