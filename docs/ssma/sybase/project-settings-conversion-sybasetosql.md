# <a name="project-settings-conversion-sybasetosql"></a>Параметры проекта (преобразование) (SybaseToSQL)
На странице преобразования **параметры проекта** диалоговое окно содержит настройки, установленные как SSMA преобразует синтаксис Sybase адаптивной Server Enterprise (ASE) для [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или синтаксис SQL Azure.  
  
Область преобразования доступна в **параметры проекта** и **параметры проекта по умолчанию** диалоговые окна:  
  
-   Если вы хотите указать параметры для всех проектов SSMA на **средства** последовательно выберите пункты **параметры проекта по умолчанию**, нажмите кнопку **Общие** в нижней части левой панели, а затем нажмите кнопку **преобразование**.  
  
-   Чтобы указать параметры для текущего проекта на **средства** последовательно выберите пункты **параметры проекта**, нажмите кнопку **Общие** в нижней части левой панели, а затем нажмите кнопку **преобразование**.  
  
## <a name="miscellaneous-options"></a>Различные параметры  
**@@ERROR**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure и ASE используют различные коды ошибок.  
  
Используйте этот параметр, чтобы указать тип сообщения (предупреждение или ошибка), когда он встречает ссылку на панели вывода или список ошибок отображается SSMA **@@ERROR**  в ASE кода.  
  
-   При выборе **преобразовать и пометить с предупреждением**, SSMA будет преобразовать операторы и отметьте их атрибутом комментарии предупреждения.  
  
-   При выборе **метки с ошибкой**, SSMA пропустит преобразования и пометить инструкций с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** преобразовать и пометить с предупреждением  
  
**Полный режим:** метки с ошибкой  
  
**Преобразование оператор LIKE**  
Указывает, следует ли преобразовать как операнды для Sybase ASE поведение сопоставления. Важно, что Sybase удаляет конечные пробелы в операторе like шаблону. Достаточно выполнить приведение правого выражения в тип данных фиксированной длины с максимальной точностью.  
  
-   Выберите **простое преобразование** для преобразования выражений без все исправления.  
  
-   Для инструкции select поведение ASE **приведение к фиксированной длины.**  
  
При выборе режима преобразования в поле режим SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic**: простое преобразование  
  
**Полный режим**: приведение к фиксированной длины  
  
**ПРЕОБРАЗОВАНИЯ ИЛИ ПРИВЕДЕНИЯ ПУСТЫХ СТРОК В ЧИСЛОВЫЕ ТИПЫ**  
Указывает способ обработки пустым строк в выражениях CONVERT или CAST с числового типа в качестве аргумента типа данных. Для этого параметра доступны следующие параметры:  
  
-   Выберите **простое преобразование** для преобразования выражений без все исправления.  
  
-   Если **пустой строке, ни одного цифровой** выбран, то строковый параметр {s} будет заменен вариантов ltrim(rtrim({s})) при» «ЗАТЕМ 0 else {s} ВЫРАЖЕНИЕ ЭЛЕМЕНТА  
  
При выборе режима преобразования в поле режим SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic**: простое преобразование  
  
**Полный режим**: пустой строке, ни одного числовой  
  
**Объединение NULL**  
Этот параметр указывает, как преобразовать объединение строк со значением NULL. Для конкретно этот параметр можно задать следующие параметры.  
  
-   **Перенос с помощью функции ISNULL:** Если этот параметр имеет значение, каждый неконстантное «string_expression» в объединение будет вставлено с ISNULL(string_expression) и пустые значения будут заменены на пустую строку.  
  
-   **Сохранить текущий синтаксис**  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** переносить с помощью функции ISNULL  
  
**Преобразование пустых строк**  
Этот параметр указывает, как преобразовать пустые строки. Для конкретно этот параметр можно задать следующие параметры.  
  
-   **Замените все строковые выражения пространства**  
  
-   **Замените пустые строковые константы места**  
  
-   Для использования [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ поведение SQL Azure, выберите **сохранить текущего синтаксиса**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** замените все строковые выражения пространства  
  
**ПРЕОБРАЗОВАНИЕ и ПРИВЕДЕНИЕ преобразование двоичных строк**  
Преобразование двоичных значений в числа могут возвращать различные значения на разных платформах. Например, на x86 процессоров, CONVERT (целое число, 0x00000100) возвращает 65536 в ASE и 256 в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]. ASE также возвращает разные значения в зависимости от порядка байтов.  
  
Используйте этот параметр для управления, как преобразовать SSMA преобразует и выражения CASE, которые содержат двоичные значения:  
  
-   Выберите **простое преобразование** для преобразования выражений без предупреждения и исправления. Используйте этот параметр, если известно, что сервер ASE имеет порядка байтов, которая не требует каких-либо изменений двоичное значение.  
  
-   Выберите **преобразования и исправьте** иметь SSMA преобразования и исправьте выражения для использования на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]. Можно изменить порядок байтов в символьных констант. Все остальные двоичные значения (например, двоичные переменные и столбцы) будут помечены с ошибками. Это значение используется в том случае, если известно, что сервер ASE имеет порядка байтов, который требует внесения изменений в двоичные значения.  
  
-   Выберите **преобразовать и пометить с предупреждением** иметь SSMA преобразования и исправьте выражения и пометить все преобразования выражений с комментариями предупреждение.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию:** преобразовать и пометить с предупреждением  
  
**Режим оптимистичного:** простое преобразование  
  
**Полный режим:** преобразования и исправления  
  
**Динамический SQL**  
Используйте этот параметр, чтобы указать тип сообщения (предупреждение или ошибка) SSMA выводится в области вывода или список ошибок при обнаружении в коде ASE динамического кода SQL.  
  
-   При выборе **преобразовать и пометить с предупреждением**, SSMA будет преобразовать динамического SQL и пометить инструкций с комментариями предупреждение.  
  
-   При выборе **метки с ошибкой**, SSMA пропустит преобразования и пометить инструкций с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** преобразовать и пометить с предупреждением  
  
**Полный режим:** метки с ошибкой  
  
**Преобразование проверки равенства**  
В [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, если параметр SET ANSI_NULLS имеет значение on, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, возвращает UNKNOWN, если сравнение на равенство содержит значение null. Если SET ANSI_NULLS равно, сравнения на равенство, содержащие значения null возвращает true, если сравниваемые столбца и выражения или два выражения являются как null. Путем проверки на равенство по умолчанию (ANSINULL ВЫКЛЮЧЕН) Sybase ASE сравнения ведут себя как [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure с ANSI_NULLS OFF.  
  
-   При выборе **простое преобразование**, преобразует код ASE SSMA [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ синтаксис SQL Azure без дополнительных проверок значений null. Используйте этот параметр, если SET ANSI_NULLS равно в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, или если вы захотите изменить сравнения на равенство для каждого случая.  
  
-   При выборе **значения NULL рассмотрим**, SSMA добавит проверяет наличие значений null при помощи предложения IS NULL и IS NOT NULL.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** простое преобразование  
  
**Полный режим:** попробуйте NULL значения  
  
**Строки формата**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure не поддерживает *format_string* аргумент в инструкции PRINT и RAISERROR. *Format_string* переменной поддерживается размещение подстановочные параметры непосредственно в строке, а затем заменить параметры во время выполнения. Вместо этого [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] требует полной строки с помощью строкового литерала, или строка, построенных с помощью переменной. Дополнительные сведения см. в разделе «PRINT ([!INCLUDE[tsql](../../includes/tsql_md.md)])» раздела [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] электронной документации.  
  
Когда встречается SSMA *format_string* аргумент, его можно либо построения строкового литерала с помощью переменных или создать новую переменную и создания строки с помощью этой переменной.  
  
-   Чтобы использовать строковый литерал для функции PRINT и RAISERROR, установите **создать новую строку**.  
  
    В этом режиме Если инструкции PRINT или RAISERROR не используются заполнители и локальные переменные, инструкция не изменяется. Двойные символы процента (%), заменяются на % один символ процента в строковых литералах печати.  
  
    Если использует инструкцию PRINT или RAISERROR, заполнители и один или несколько локальных переменных, как показано в следующем примере:  
  
    ```  
    PRINT 'Total: %1!%%', @percent  
    ```  
    SSMA преобразует его в следующий синтаксис:  
  
    ```  
    PRINT 'Total: '+ CAST(@percent AS varchar(max)) + '%'  
    ```  
    Если *format_string* является переменной, например, в следующей инструкции:  
  
    ```  
    PRINT @fmt, @arg1, @arg2  
    ```  
    SSMA не может выполнить преобразование простую строку и необходимо создать новую переменную.  
  
    ```  
    DECLARE @print_format_1 varchar(max)  
    SET @print_format_1  =   
        REPLACE (@fmt, '%%', '%')  
    SET @print_format_1  =   
        REPLACE (@print_format_1, '%1!',   
        CAST (@arg1 AS varchar(max)))  
    SET @print_format_1  =   
        REPLACE (@print_format_1, '%2!',   
        CAST (@arg2 AS varchar(max)))  
    PRINT @print_format_1  
    ```  
    Если используется **создать новую строку** режиме SSMA предполагается, что [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] параметр CONCAT_NULL_YIELDS_NULL имеет значение OFF. Таким образом SSMA не проверяет наличие нулевыми аргументами.  
  
-   Чтобы SSMA построить новую переменную для каждой инструкции PRINT и RAISERROR, а затем использовать эту переменную для строкового значения, установите **создать новую переменную**.  
  
    В этом режиме, если инструкции PRINT или RAISERROR не используются заполнители и локальные переменные SSMA заменяет символов двойных все процента (%) с одного символа процента в соответствии с [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ синтаксис SQL Azure.  
  
    Если использует инструкцию PRINT или RAISERROR, заполнители и один или несколько локальных переменных, как показано в следующем примере:  
  
    ```  
    PRINT 'Total: %1!%%', @percent  
    ```  
    SSMA преобразует его в следующий синтаксис:  
  
    ```  
    DECLARE @print_format_1 varchar(max)  
    SET @print_format_1 = 'Total: %1!%'  
    SET @print_format_1  =   
        REPLACE (@print_format_1, '%1!',   
        ISNULL(CAST (@percent AS VARCHAR(max)), ''))  
    PRINT @print_format_1  
    ```  
    Если *format_string* является переменной, например, в следующей инструкции:  
  
    ```  
    PRINT @fmt, @arg1, @arg2  
    ```  
    SSMA создает новую переменную следующим образом, проверка на значения null в каждый аргумент:  
  
    ```  
    DECLARE @print_format_1 varchar(max)  
    SET @print_format_1  =   
        REPLACE (@fmt, '%%', '%')  
    SET @print_format_1  =   
        REPLACE (@print_format_1, '%1!',   
        ISNULL(CAST (@arg1 AS varchar(max)),''))  
    SET @print_format_1  =   
        REPLACE (@print_format_1, '%2!',   
        ISNULL(CAST (@arg2 AS varchar(max)),''))  
    PRINT @print_format_1  
    ```  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** создать новую строку  
  
**Полный режим:** Создание новой переменной  
  
**Вставить явное значение в столбец отметки времени**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure не поддерживает вставку явные значения в столбец временной метки.  
  
-   Чтобы исключить столбцы отметки времени из инструкций INSERT, установите **исключить столбец**.  
  
-   Чтобы напечатать сообщение об ошибке, каждый раз, столбец типа timestamp в инструкции INSERT, выберите **метки с ошибкой**. В этом режиме инструкции INSERT не будут преобразованы и будут помечены ошибки комментарии.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** исключить столбец  
  
**Полный режим:** метки с ошибкой  
  
**Хранить временные объекты, определяемые в процедурах**  
Этот параметр указывает, если определения временные объекты, которые отображаются в процедурах должны храниться в метаданных источника во время преобразования.  
  
-   Выберите **Да** для хранения в метаданных.  
  
-   Выберите **нет** Если объекты не должны храниться.  
  
**Режим по умолчанию или оптимистичного:** Да  
  
**Полный режим:** нет  
  
**Преобразование таблицы прокси-сервера**  
Указывает, если таблицы прокси ASE преобразуются в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ таблиц SQL Azure или они не преобразуются, и код помечен как ошибка комментариями.  
  
-   Выберите **преобразования** для преобразования таблицы прокси-сервера в обычных таблицах.  
  
-   Выберите **метки с ошибкой** для указания кода таблицы прокси-сервера с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic/Full:** метки с ошибкой  
  
**Число сообщений инструкции RAISERROR**  
ASE пользователя сообщения хранятся в каждой базе данных. [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]сообщения для пользователя централизованно хранятся и доступны через **sys.messages** представления каталога. Кроме ASE пользовательских сообщений начинаются с 20000, но [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] сообщения об ошибках начинаются с 50001.  
  
Этот параметр указывает число, которое необходимо добавить к номеру сообщения ASE пользователя для преобразования его в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] сообщение для пользователя. Если ваш [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] имеет сообщений для пользователя в **sys.messages** представления каталога, может потребоваться изменить этот номер более высокое значение. Это преобразованное сообщение номера не конфликтуют с существующие номера сообщений.  
  
Следует отметить следующее.  
  
-   ASE сообщения в диапазоне 17000 19999 из системной таблицы sysmessages и не преобразуются.  
  
-   Если номер ошибки, на которые ссылается инструкция RAISERROR является константой, SSMA добавит номер базового сообщения с константой, чтобы определить новый номер сообщения для пользователя.  
  
-   Если номер ошибки, на который имеется ссылка является переменной или выражения, SSMA создаст промежуточного локальной переменной.  
  
-   В режиме оптимистической SSMA предполагается, что [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] параметр CONCAT_NULL_YIELDS_NULL отключен и делает не выполняет проверку нулевыми аргументами.  
  
-   В режиме полной SSMA ищет нулевыми аргументами.  
  
-   Инструкция RAISERROR с ERRORDATA *списка* не преобразуется.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или оптимистичный/Full:** 30001  
  
**Системные объекты**  
Используйте этот параметр, чтобы указать тип сообщения (предупреждение или ошибка) SSMA выводится в области вывода или список ошибок при обнаружении использование ASE системные объекты.  
  
-   При выборе **преобразовать и пометить с предупреждением**, SSMA преобразует ссылки на системные объекты и пометит инструкций с комментариями предупреждение.  
  
-   При выборе **метки с ошибкой**, SSMA не станет преобразовывать ссылки на объекты системы и пометит инструкций с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** преобразовать и пометить с предупреждением  
  
**Полный режим:** метки с ошибкой  
  
**Неразрешенные идентификаторов**  
Используйте этот параметр, чтобы указать тип сообщения (предупреждение или ошибка), SSMA отображается в области вывода или список ошибок, если его не удается разрешить идентификатор.  
  
-   При выборе **преобразовать и пометить с предупреждением**, SSMA предпринимается попытка преобразовать ссылок на идентификаторы неразрешенные и пометит инструкций с комментариями предупреждение.  
  
-   При выборе **метки с ошибкой**, SSMA не станет преобразовывать ссылок на идентификаторы неразрешенные и пометит инструкций с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** преобразовать и пометить с предупреждением  
  
**Полный режим:** метки с ошибкой  
  
## <a name="system-function-options"></a>Параметры функции системы  
**Функция CHARINDEX**  
В ASE CHARINDEX возвращает NULL только в том случае, если все входные выражения имеют значение NULL. [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, возвратят значение NULL, если любой входное выражение имеет значение NULL.  
  
-   Чтобы использовать поведение ASE, выберите **заменить функцию**. С помощью вызова CHARINDEX_VARCHAR или CHARINDEX_NVARCHAR определяемой пользователем функции, на основе типа параметров, переданных (созданы в базе данных пользователя в списке имя схемы «s2ss») для эмуляции режима Sybase ASE подставляется все вызовы функции CHARINDEX.  
  
-   Для использования [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ поведение SQL Azure, выберите **сохранить текущего синтаксиса**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** Replace, функция  
  
**Функция DATALENGTH**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure и ASE различаются значения, возвращенного функция DATALENGTH, когда значение является пробелом. В этом случае [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ возвращает значение 0, SQL Azure и ASE возвращает 1.  
  
-   Чтобы использовать поведение ASE, выберите **заменить функцию**. Выражение CASE для эмуляции режима Sybase ASE, заменяются все вызовы функция DATALENGTH.  
  
-   Чтобы использовать значение по умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] / поведение SQL Azure, выберите **сохранить текущего синтаксиса**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** Replace, функция  
  
**INDEX_COL, функция**  
Поддерживает необязательный ASE *user_id* аргумент функции INDEX_COL; Однако [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure не поддерживает этот аргумент. Если вы используете *user_id* , эта функция не может преобразовать аргумент [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ синтаксис SQL Azure.  
  
-   Чтобы использовать поведение ASE, выберите **функция Convert**. Если код содержит *user_id* аргумент SSMA будет выведено сообщение об ошибке.  
  
-   Чтобы отобразить сообщение об ошибке, каждый раз при обнаружении INDEX_COL, выберите **метки с ошибкой**. SSMA не станет преобразовывать ссылки на функцию и пометит инструкцию с комментариями ошибки.  
  
**Режим по умолчанию или Optimistic/Full:** метки с ошибкой  
  
**Функция INDEX_COLORDER**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure не поддерживает функцию INDEX_COLORDER системы.  
  
-   Чтобы использовать поведение ASE, выберите **функция Convert**. Все вызовы функции INDEX_COLORDER замещается вызов определяемую пользователем функцию с именем INDEX_COLORDER (созданы в базе данных пользователя в списке имя схемы «s2ss»), который имитирует поведение Sybase ASE.  
  
-   Чтобы напечатать сообщение об ошибке, каждый раз при обнаружении, INDEX_COLORDER, выберите **метки с ошибкой**. SSMA не станет преобразовывать ссылки на функцию и пометит инструкцию с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic/Full:** метки с ошибкой  
  
**Функции LEFT и RIGHT**  
Слева и справа функций в Sybase ведут себя иначе отрицательную длину параметра.  
  
-   Чтобы использовать поведение ASE, выберите **функцией Replace**. Параметр длины будет заменен выражение CASE, в которой было бы возвращено значение null для отрицательное значение.  
  
-   Чтобы использовать поведение SQL Server, выберите **сохранить текущего синтаксиса**  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** Replace, функция  
  
> [!NOTE]  
> Если параметр длины является литералом и не сложное выражение, значение длины всегда заменяется значение null, независимо от параметра проекта.  
  
**Функция NEXT_IDENTITY**  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure не поддерживает функцию NEXT_IDENTITY системы.  
  
-   Чтобы использовать поведение ASE, выберите **функция преобразования**. Все вызовы функции NEXT_IDENTITY замещается выражения (IDENT_CURRENT(parameter Value) + IDENT_INCR(parameter Value) которые эмулируют поведение Sybase ASE.  
  
-   Чтобы напечатать сообщение об ошибке, каждый раз при обнаружении, NEXT_IDENTITY, выберите **метки с ошибкой**. SSMA не станет преобразовывать ссылки на функцию и пометит инструкцию с комментариями ошибки.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic/Full:** метки с ошибкой  
  
**Функция PATINDEX**  
Указывает, следует ли преобразовать функции PATINDEX для Sybase ASE поведение сопоставления. Важно, что Sybase удаляет конечные пробелы в шаблон поиска. Достаточно выполнить приведение выражение значения с фиксированной длиной данных введите с наибольшей точностью и применять функцию rtrim шаблон поиска.  
  
-   Для инструкции select поведение ASE **использовать**.  
  
-   Чтобы использовать значение по умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ поведение SQL Azure, выберите **используют**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** не используйте  
  
**Полный режим:** использования  
  
**REPLICATE, функция**  
Функция REPLICATE повторяет указанное число раз. В ASE при указании повторить строки нулевой раз результат имеет значение null. В [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, результатом является пустая строка.  
  
-   Чтобы использовать поведение ASE, выберите **заменить функцию**. С помощью вызова REPLICATE_VARCHAR или REPLICATE_NVARCHAR определяемой пользователем функции, на основе типа параметров, переданных (созданы в базе данных пользователя в списке имя схемы «s2ss») для эмуляции режима Sybase ASE подставляется все вызовы функции REPLICATE.  
  
-   Чтобы использовать значение по умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ поведение SQL Azure, выберите **функцией Replace**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic режим/Full:** Replace, функция  
  
**Функция TRIM (LTRIM, RTRIM)**  
Этот параметр указывает, следует ли заменить вызовы функции Trim (LTRIM, RTRIM) синтаксис Sybase ASE эквивалентных функций или для сохранения текущего синтаксиса. Для этого конкретного параметра присутствуют следующие параметры:  
  
-   **REPLACE-функция**  
  
-   **Сохранить текущий синтаксис**  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic режим/Full:** Replace, функция  
  
**SUBSTRING, функция**  
В ASE, функция `SUBSTRING(expression, start, length)` возвращает значение NULL, если задано значение start больше, чем количество символов в выражении или если длина равна нулю. В [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]/ SQL Azure, эквивалентное выражение возвращает пустую строку.  
  
-   Чтобы использовать поведение ASE, выберите **заменить функцию**. С помощью вызова SUBSTRING_VARCHAR или SUBSTRING_NVARCHAR или SUBSTRING_VARBINARY определяемой пользователем функции, на основе типа параметров, переданных (созданы в базе данных пользователя в списке имя схемы «s2ss») для эмуляции режима Sybase ASE подставляется все вызовы функции SUBSTRING.  
  
-   Для использования [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] / поведение SQL Azure, выберите **сохранить текущего синтаксиса**.  
  
При выборе режима преобразования в **режим** , SSMA применяется следующий параметр:  
  
**Режим по умолчанию или Optimistic:** сохранить текущего синтаксиса  
  
**Полный режим:** Replace, функция
