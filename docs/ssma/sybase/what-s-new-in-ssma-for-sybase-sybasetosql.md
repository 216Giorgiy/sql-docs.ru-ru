# <a name="what39s-new-in-ssma-for-sap-ase-sybasetosql"></a>Какой &#39; новые возможности SSMA для SAP ASE (SybaseToSQL)
В этом разделе перечислены SSMA для SAP ASE (прежнее название — SSMA для СУБД Sybase) изменения в каждом выпуске. 

## <a name="ssma-v75"></a>SSMA v7.5
V7.5 выпуск SSMA для SAP ASE содержит следующие изменения:
-   Дополнено несколько улучшений, чтобы обеспечить большую доступность для людей с ограниченными возможностями.
-   Обновить для предоставления поддержки для СОЗДАНИЯ или ЗАМЕНЫ синтаксиса.

> [!IMPORTANT]
> Версия .net 4.5.2 является необходимым условием для установки SSMA v7.5. Кроме того начиная с v7.4, 32-разрядной версии SSMA прекращается.  

## <a name="ssma-v74"></a>SSMA v7.4
V7.4 выпуск SSMA для СУБД Sybase содержит следующие изменения:
- **Время ожидания запроса** теперь доступен при обнаружении объекта схемы в исходной и целевой.
![параметр времени ожидания запроса](../media/query-timeout_red.png)
- Метрика качества и преобразование усовершенствованы целевой исправления, на основе отзывов клиентов.

> [!IMPORTANT]
> Версия .net 4.5.2 является необходимым условием для установки SSMA v7.4. Кроме того начиная с v7.4, 32-разрядной версии SSMA прекращается.  

## <a name="ssma-v73"></a>SSMA v7.3
V7.3 выпуск SSMA для СУБД Sybase содержит следующие изменения:
- Улучшенное качество и преобразование метрику с целевой исправления на основе отзывов клиентов.
- SSMA расширения среды .NET framework, представляемый через следующие элементы:
  - Экспорт функций в проект SQL Server Data Tools (SSDT).
    -   Теперь можно экспортировать скриптов схемы с SSMA для проекта SSDT. Скрипты схемы можно использовать для внесения изменений дополнительные схемы и развертывания базы данных.
![Сохранить как SSDT проект-команда](../media/export-schema-scripts_red.png)
  - Библиотеки, которые могут быть использованы SSMA для выполнения пользовательских преобразований.
    - Теперь можно создать код, который может обработать синтаксис пользовательского преобразования и преобразования, которые не были обработаны ранее SSMA.
      - Инструкции о том, как создать пользовательский преобразователь доступны в этой записи блога [расширение SQL Server Migration Assistant по возможности конвертации](https://blogs.msdn.microsoft.com/datamigration/2017/02/21/2185/).
      - Пример проекта для преобразования можно загрузить это [блога](https://blogs.msdn.microsoft.com/datamigration/ssmafororacleconversionsample/).

## <a name="ssma-v72"></a>SSMA v7.2
V7.2 выпуск SSMA для СУБД Sybase содержит следующие изменения:
- Улучшенное качество и преобразование метрику с целевой исправления на основе отзывов клиентов.
- Улучшения телеметрии для предоставления более точек данных для устранения проблем клиентов и повысить скорость преобразования элемента SSMA.

## <a name="ssma-v71"></a>SSMA v7.1
V7.1 выпуск SSMA для Access содержит следующие изменения:
- 2017 г. SQL Server в Windows и Linux CTP-версия 1 больше не поддерживаемую целевую платформу для миграции. Эта функция в technical preview и поддерживает перемещение схему и данные в целевых серверов SQL.
- SSMA теперь поддерживает автоматическое обновление, чтобы загрузить последнюю версию SSMA как только он доступен.
- Теперь может устанавливаться SSMA двоичные файлы доставляются через файлы пакета установщика Windows (.msi).

## <a name="may-2016"></a>Май 2016 г.  
Май 2016 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  

-  Добавлена поддержка SQL Server 2016.
-  Удалить проверку установщика для .net 2.0.
-  Обновленный пакет расширения зависимость от .net 3.5 в .net 4.0.
-  Исправлена «сохранить проект» и команды «Открыть проект» для консоли SSMA.
-  Команда фиксированный «securepassword» для консоли SSMA.
-  Исправлена подсчет объектов для начальной загрузки.
-  Исправлена ошибка в глобальных параметрах.

## <a name="march-2016"></a>Март 2016 г.  
Март 2016 г. Предварительный выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-  Добавлена поддержка для миграции в SQL Server 2016.  
  
## <a name="january-2016"></a>Январь 2016 г.  
Январь 2016 г. обслуживание выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-  Добавлено представление пункт меню журнала SSMA (RFC 5706203).  
-  Добавлены телеметрии. 
  
## <a name="july-2014"></a>Июль 2014 г.  
Июль 2014 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-  Улучшенное преобразование кода базы данных SQL Azure.  
-  Расширение функциональности пакета перемещен в схему для поддержки базы данных SQL Azure.  
-  Для баз данных с более чем 10 КБ добавлены производительность проверки объектов.  
-  Усовершенствования пользовательского интерфейса для работы с большим количеством объектов.  
-  Добавлена возможность выделения «хорошо известный» схемы LOB (то есть они могут быть пропущены во время преобразования).  
-  Увеличение скорости добавленный преобразования.  
-  Добавлена возможность отображения счетчиков объектов в пользовательском Интерфейсе. 
-  Размер ограниченной отчета на более 25%.  
-  Улучшенные сообщения об ошибках для непроанализированного конструкций.  
  
## <a name="april-2014"></a>Апрель 2014 г.  
Апрель 2014 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлена поддержка MS SQL Server 2014.  
-   Исправленные ошибки относительно преобразование в Azure.  
-   Исправленные ошибки относительно страницы невидимой отчетов в IE 10.  
  
## <a name="january-2012"></a>Январь 2012 г.  
Январь 2012 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлена поддержка для преобразования триггер отката.  
-   Указанное исправление для преобразования@ROWCOUNT и @@ERROR в одной и той же инструкции SET.  
  
## <a name="july-2011"></a>Июль 2011 г.  
Июль 2011 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Усовершенствованные отчетов во время переноса данных об ошибках.  
  
## <a name="april-2011"></a>Апреля 2011 г.  
Апреля 2011 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Объединенный продукта «SSMA для Sybase», которая поддерживает [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2005, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali» и Azure SQL.  
-   Добавлена поддержка для подключения и переход на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
-   Добавлен новый компонент для преобразования и переноса баз данных Sybase в Azure SQL.  
-   Модуль переноса расширенные данные клиента с поддержкой параллельного переноса данных.  
-   Большая производительность миграции с простой и массового войти в модели восстановления.  
-   Добавлена возможность правильно преобразовать и переноса баз данных с учетом регистра Sybase в SQL Server, с учетом регистра.  
-   Добавлена поддержка для преобразования инструкций соединения Sybase ASE не совместимые с ANSI для инструкций соединения SQL Server ANSI была расширена на удаление и изменение инструкций.  
-   Предоставляемые возможности дополнительные подключения для подключения к серверам Sybase ASE, с помощью поставщика ODBC для Sybase ASE и поставщиков Sybase ASE ADO.Net.  
-   Удалена зависимость от отдельных базу данных с именем **SysDB**, которая содержит функции эмуляции Sybase (устанавливается как часть пакета расширения).  
-   Добавлена возможность установить SSMA для пакета расширения Sybase на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] кластеров.  
-   Добавлена обратная совместимость проекты, созданные в предыдущих версиях SSMA (версии 4.0 и версия 4.2).  
-   Добавлена возможность установить SSMA для Sybase версии 5.0 продукта side-by-side (SxS) с более ранними версиями SSMA (версии 4.0 и версия 4.2).  
  
## <a name="july-2010"></a>Июль 2010 г.  
Июль 2010 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлена поддержка для миграции в SQL Server 2008 R2.  
-   Добавить новое приложение консоли SSMA для выполнения командной строки.  
-   Добавлена поддержка для переноса данных с помощью серверных и клиентских данных миграции ядер.  
-   Добавлена поддержка инструкции «SELECT настраиваемый» в переноса данных.  
-   Добавлена поддержка миграции из Sybase ASE 15.0.3 и 15,5.  
  
## <a name="june-2008"></a>Июнь 2008 г.  
Июнь 2008 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлены SSMA тест-инженер, который автоматически проверяет преобразования объекта базы данных и переноса данных, внесенных SSMA. После завершения всех шагов миграции SSMA используйте тест-инженер SSMA для проверки, преобразованные объекты работать так же, как и все данные были перемещены должным образом.  
-   Добавленный преобразования версии SQL. Пользователь теперь может указать Временная таблица (и другого объекта) объявления для каждой процедуры источника для использования в преобразовании.  
-   Усовершенствования в преобразовании объекта:  
    -   Присоединяет пересмотра преобразования.  
    -   Статистические функции и на нестатистические данные без необходимости или группу предложениями.  
    -   Функция УДОСТОВЕРЕНИЙ с помощью инструкции SELECT INTO.  
    -   Кластеризованного ограничения и индексы в данных только для блокировки.  
    -   Временные таблицы, созданные SELECT INTO.  
    -   Ограничения и индексы для временных таблиц.  
    -   Новый [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] поддерживаются 2008 типы даты и времени.  
    -   Поддержка возможности подключения и типы данных Sybase 15.0.  
  
## <a name="may-2007"></a>Мая 2007 г.  
Мая 2007 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлена возможность быстрее загружать содержимое базы данных при сохранении проекта.  
-   Добавлена поддержка пользователь ввел комментарии в SQL Server в формате режима SQL.  
-   Усовершенствования в преобразования объекта.  
  
Файл справки не был обновлен для этого выпуска. Дополнительные сведения см. заметки к документации далее в этом разделе.  
  
## <a name="november-2006"></a>Ноября 2006 г.  
Ноября 2006 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлены новые глобальные параметры:  
    -   Можно отключить отображение номеров строк в окнах редактора.  
    -   Можно настроить SSMA для запроса, чтобы заменить повторяющиеся объекты или всегда или никогда не заменять повторяющиеся объекты во время преобразования схемы.  
-   Добавлен новый параметр преобразования, который позволяет настроить обработку SSMA следующих ситуаций:  
    -   Оператор CAST или CONVERT, содержащее двоичную строку.  
    -   Проверяет наличие значений null в выражениях равенства.  
    -   Таблицы, прокси-сервера.  
    -   Номера ошибок пользователя сообщения для инструкции RAISERROR.  
    -   Инструкции UPDATE, которые содержат неразрешенные идентификаторы.  
-   Добавлен новый вариант миграции, который позволяет указать порядок обработки дат, которые находятся за пределами SSMA [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] диапазон дат.  
-   Добавлен **в формате SQL** на **SQL** вкладки, которая форматирует кода для повышения удобства чтения.  
-   Исправления ошибок, которые включают следующее:  
    -   SSMA теперь преобразует БЛОКИРОВКИ таблицы *таблицы* IN {SHARED | ЭКСКЛЮЗИВНОЕ} инструкции режиме, добавив подсказку TABLOCK или TABLOCKX последующий запрос SELECT для таблицы.  
    -   Необходимые приведения добавляются при использовании двоичных типов в символьных выражений.  
    -   Память и производительность приложений.  
  
## <a name="july-2006"></a>Июль 2006 г.
