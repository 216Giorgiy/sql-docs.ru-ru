# <a name="what39s-new-in-ssma--for-sybase-sybasetosql"></a>Какой &#39; новые возможности SSMA для СУБД Sybase (SybaseToSQL)
В этом разделе перечислены SSMA для Sybase изменения в каждом выпуске.  

## <a name="may-2016"></a>Май 2016 г.  
Май 2016 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  

1.  Добавлена поддержка SQL Server 2016
2.  Удалить проверку установщика для .net 2.0
3.  Обновленный пакет расширения зависимость от .net 3.5 .net 4.0
4.  Исправлена «сохранить проект» и команды «Открыть проект» для консоли SSMA
5.  Команда фиксированный «securepassword» для консоли SSMA
6.  Исправлена подсчет объектов для начальной загрузки
7.  Исправлена ошибка в глобальных параметрах

## <a name="march-2016"></a>Март 2016 г.  
Март 2016 г. Предварительный выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
1.  Поддерживает миграцию на SQL Server 2016  
  
## <a name="january-2016"></a>Январь 2016 г.  
Январь 2016 г. обслуживание выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
1.  Пункт меню журнала добавлено представление SSMA (RFC 5706203)  
  
2.  Добавлены телеметрии  
  
## <a name="july-2014"></a>Июль 2014 г.  
Июль 2014 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
1.  Улучшенное преобразование кода базы данных SQL Azure  
  
2.  Функциональные возможности пакета расширения перемещен в схему для поддержки базы данных SQL Azure  
  
3.  Повышение производительности для проверки базы данных с более чем 10k объектов  
  
4.  Улучшения пользовательского интерфейса для обработки большого числа объектов  
  
5.  Выделение «хорошо известный» схем LOB (то есть они могут быть пропущены во время преобразования)  
  
6.  Увеличение скорости преобразования  
  
7.  Отображение счетчиков объектов в пользовательском Интерфейсе  
  
8.  Уменьшение размера отчета на более 25%  
  
9. Улучшенные сообщения об ошибках для непроанализированного конструкций.  
  
## <a name="april-2014"></a>Апрель 2014 г.  
Апрель 2014 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Добавлена поддержка MS SQL Server 2014.  
  
-   Исправленные ошибки относительно преобразование в Azure  
  
-   Исправленные ошибки относительно страницы невидимой отчетов в IE 10.  
  
## <a name="january-2012"></a>Январь 2012 г.  
Январь 2012 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Поддержка преобразования триггер отката.  
  
-   Указанное исправление для преобразования@ROWCOUNT и @@ERROR в одной и той же инструкции SET.  
  
## <a name="july-2011"></a>Июль 2011 г.  
Июль 2011 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Усовершенствованные отчетов во время переноса данных об ошибках.  
  
## <a name="april-2011"></a>Апреля 2011 г.  
Апреля 2011 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Объединенный продукта «SSMA для Sybase», которая поддерживает [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2005, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali» и SQL Azure.  
  
-   Поддержка подключения и перенос на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] «Denali».  
  
-   Новой функции для преобразования и переноса баз данных Sybase в SQL Azure.  
  
-   Теперь клиент стороны миграции подсистема обработки данных, поддержка параллельного переноса данных.  
  
-   Большая производительность миграции с простой и массового войти в модели восстановления.  
  
-   С учетом регистра баз данных Sybase можно правильно преобразовать и перенесенных в SQL Server, с учетом регистра.  
  
-   Поддержка преобразований операторов соединения Sybase ASE не совместимые с ANSI, SQL Server ANSI инструкций соединения была расширена на удаление и изменение инструкций.  
  
-   Варианты дополнительные подключения для подключения к серверам Sybase ASE, с помощью поставщика ODBC для Sybase ASE и поставщиков Sybase ASE ADO.Net.  
  
-   Вызывается для удаления зависимости от отдельной базы данных **SysDB** , содержащее функции эмуляции Sybase (устанавливается как часть пакета расширения).  
  
-   SSMA для пакета расширения Sybase, теперь можно устанавливать на [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] кластеров.  
  
-   Обратная совместимость проекты, созданные в предыдущих версиях SSMA (версии 4.0 и версия 4.2).  
  
-   SSMA для СУБД Sybase версии 5.0 продукта могут быть установлены параллельно (SxS) с более ранними версиями SSMA (версии 4.0 и версия 4.2).  
  
## <a name="july-2010"></a>Июль 2010 г.  
Июль 2010 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Поддержка миграции в SQL Server 2008 R2  
  
-   Новое приложение консоли SSMA для выполнения командной строки  
  
-   Поддержка переноса данных с помощью обработчиков миграции данных стороны клиента и стороне сервера  
  
-   Поддержка инструкция «SELECT настраиваемый» в переноса данных  
  
-   Поддержка миграции из Sybase ASE 15.0.3 и 15,5  
  
## <a name="june-2008"></a>Июнь 2008 г.  
Июнь 2008 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Инженер-испытатель SSMA добавлен, он автоматически выполняют проверку преобразования объекта базы данных и переноса данных, внесенных SSMA. После завершения всех шагов миграции SSMA используйте тест-инженер SSMA для проверки, преобразованные объекты работать так же, как и все данные были перемещены должным образом.  
  
-   Добавлено преобразование версии SQL. Пользователь теперь может указать временные таблицы (и другие объекты) объявления для каждой процедуры источника для использования в преобразовании.  
  
-   Усовершенствования в преобразовании объекта:  
  
    -   Присоединяет пересмотра преобразования.  
  
    -   Статистические функции и на нестатистические данные без необходимости или группу предложениями.  
  
    -   Функция УДОСТОВЕРЕНИЙ с помощью инструкции SELECT INTO.  
  
    -   Кластеризованного ограничения и индексы в данных только для блокировки.  
  
    -   Временные таблицы, созданные SELECT INTO.  
  
    -   Ограничения и индексы для временных таблиц.  
  
    -   Новый [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] поддерживаются 2008 типы даты и времени.  
  
    -   Поддержка возможности подключения и типы данных Sybase 15.0.  
  
## <a name="may-2007"></a>Мая 2007 г.  
Мая 2007 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   При сохранении проекта загрузку содержимого базы данных происходит быстрее.  
  
-   Поддержка комментарии, введенные пользователем режим SQL в формате SQL Server.  
  
-   Улучшения в преобразования объекта.  
  
Обратите внимание, что файл справки не был обновлен для этого выпуска. Дополнительные сведения см. заметки к документации далее в этом разделе.  
  
## <a name="november-2006"></a>Ноября 2006 г.  
Ноября 2006 г. выпуск SSMA для СУБД Sybase содержит следующие изменения:  
  
-   Новые глобальные параметры:  
  
    -   Можно отключить отображение номеров строк в окнах редактора.  
  
    -   Можно настроить SSMA для запроса, чтобы заменить повторяющиеся объекты или всегда или никогда не заменять повторяющиеся объекты во время преобразования схемы.  
  
-   Новые параметры преобразования позволяют настроить обработку SSMA следующих ситуаций:  
  
    -   Инструкции CAST или CONVERT, содержащий двоичная строка  
  
    -   Проверяет наличие значений null в выражениях равенства  
  
    -   Таблицы прокси-сервера  
  
    -   Номера ошибок пользователя сообщения для инструкции RAISERROR  
  
    -   Инструкции UPDATE, которые содержат неразрешенные идентификаторы  
  
-   Новый вариант миграции вы можете задать порядок обработки дат, которые находятся за пределами SSMA [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] диапазон дат.  
  
-   Объект **в формате SQL** на **SQL** вкладки, которая форматирует кода для повышения удобства чтения.  
  
-   Исправления ошибок, которые включают следующее:  
  
    -   SSMA теперь преобразует БЛОКИРОВКИ таблицы *таблицы* IN {SHARED | ЭКСКЛЮЗИВНОЕ} инструкции режиме, добавив подсказку TABLOCK или TABLOCKX последующий запрос SELECT для таблицы.  
  
    -   Необходимые приведения добавляются при использовании двоичных типов в символьных выражений.  
  
    -   Память и производительность приложений.  
  
## <a name="july-2006"></a>Июль 2006 г.  
Первый выпуск SSMA для СУБД Sybase был в июле 2006 г.
