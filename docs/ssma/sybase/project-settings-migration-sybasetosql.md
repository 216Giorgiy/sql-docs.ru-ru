# <a name="project-settings-migration-sybasetosql"></a>Параметры проекта (миграция) (SybaseToSQL)
На странице миграции **параметры проекта** диалоговое окно содержит настройки, установленные как SSMA осуществляет миграцию данных из Sybase Adaptive Server Enterprise (ASE) для [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].  
  
На панели миграции доступен в обеих **параметры проекта** и **параметры проекта по умолчанию** диалоговым окнам.  
  
-   Для задания параметров для всех проектов SSMA на **средства** меню, выберите **параметры проекта по умолчанию**, выберите тип проекта миграции, для которого требуются параметры просмотра или изменения из  **Целевой версии миграции** раскрывающемся списке щелкните **Общие** в нижней части левой панели, а затем нажмите кнопку **миграции**.  
  
-   Для задания параметров для текущего проекта на **средства** меню, выберите **параметры проекта**, нажмите кнопку **Общие** в нижней части левой панели и нажмите кнопку **Миграции**.  
  
## <a name="date-correction-options"></a>Параметры исправления даты  
  
|Термин|Определение|  
|--------|--------------|  
|**Замените неподдерживаемые даты**|Указывает, следует ли SSMA устранить, выпущенных до самой ранней даты [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] **datetime** Дата (01 января 1753 г.).<br /><br />Чтобы сохранить значения текущей даты, установите флажок **ничего не делать**. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] в столбце даты и времени не будет принимать дат ранее 01 января 1753 г. При использовании более старой даты, необходимо преобразовать значения даты и времени в символьные значения.<br /><br />Чтобы преобразовать дат ранее 01 января 1753 г. значение NULL, выберите **замените NULL**.<br /><br />Для замены дат ранее 01 января 1753 поддерживаемая Дата, выберите **замените ближайшего поддерживаемая Дата**.<br /><br />**Режим по умолчанию**: не выполнять никаких действий<br /><br />**Режим оптимистичного**: не выполнять никаких действий<br /><br />**Полный режим**: замените ближайшего поддерживаемая Дата|  
  
## <a name="migration-engine"></a>Модуль миграции  
  
|Термин|Определение|  
|--------|--------------|  
|**Модуль миграции**|Указывает компонент database engine, используемый во время переноса данных. Перенос данных на стороне клиента ссылается на клиента SSMA при извлечении данных из источника, массовой вставке данных в SQL Server. Перенос данных на стороне сервера относится к SSMA миграции подсистема обработки данных (программа массового копирования), выполняющихся на компьютере SQL Server как задание агента SQL Server, извлечение данных из источника и вставка непосредственно в SQL Server, что позволяет избежать дополнительной клиента прыжок (повышает производительность).<br /><br />**Режим по умолчанию**: Модуль переноса данных на стороне клиента<br /><br />**Режим оптимистичного**: Модуль переноса данных на стороне клиента<br /><br />**Полный режим**: Модуль переноса данных на стороне клиента|  
  
> [!IMPORTANT]  
> При **модуль переноса** параметру присваивается **модуль переноса данных на стороне сервера**, создать проект, установка параметра **подсистема обработки миграции данных на стороне сервера использовать 32-разрядных** отображается . Он указывает, используется ли 32- или 64-разрядная версия программы программы массового копирования (BCP) для переноса данных.  
  
## <a name="miscellaneous-options"></a>Различные параметры  
  
|Термин|Определение|  
|--------|--------------|  
|**Размер пакета**|Указывает пакет, размер, используемый во время переноса данных.<br /><br />**Режим по умолчанию**: 10000<br /><br />**Режим оптимистичного**: 10000<br /><br />**Полный режим**: 10000|  
|**Проверочные ограничения**|Указывает, должна ли SSMA проверять ограничения при вставке данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
|**Время ожидания миграции данных**|Указывает время ожидания, используемое во время переноса данных<br /><br />**Режим по умолчанию**: 15<br /><br />**Режим оптимистичного**: 15<br /><br />**Полный режим**: 15|  
|**Варианты миграции расширенные данные**|Показаны варианты миграции дополнительные данные для каждой таблицы на вкладке "отдельные данные".<br /><br />**Режим по умолчанию**: скрыть<br /><br />**Режим оптимистичного**: скрыть<br /><br />**Полный режим**: скрыть|  
|**Запускать триггеры**|Указывает, является ли SSMA запускать триггеры вставки при добавлении данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
|**Сохранять ИД**|Указывает, сохраняет ли SSMA Sybase значений идентификаторов, при добавлении данных в SQL Server. Значение False приводит значения идентификаторов, могут быть назначены целевым объектом.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**Сохранять значения NULL**|Указывает, сохраняет ли SSMA значения null в источнике данных, при добавлении данных в SQL Server, независимо от значения по умолчанию, указанные в SQL Server.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**В случае ошибки**|Остановка переноса данных при возникновении ошибки. Он имеет три варианта:<br /><br />**Остановить миграцию:** Останавливает операцию миграции данных<br /><br />**Перейти к следующей таблице:** останавливает перенос данных в текущей таблице и переходит к следующему<br /><br />**Перейти к следующему пакету:** останавливает перенос данных в текущий пакет и переходит к следующему<br /><br />**Режим по умолчанию**: перейти к следующему пакету<br /><br />**Режим оптимистичного**: перейти к следующему пакету<br /><br />**Полный режим**: перейти к следующему пакету|  
|**Round дробной части числа**|Указывает, следует ли обрезать дробных частей десятичных и числовых данных во время миграции для целочисленных типов или отобразить сообщение об ошибке, если дробная часть является нетривиальным<br /><br />**Режим по умолчанию**: нет<br /><br />**Режим оптимистичного**: нет<br /><br />**Полный режим**: нет|  
|**Порядок байтов Юникода Sybase**|Указывает тип обратный порядок байтов для строки Юникода Sybase. Можно задать следующие параметры для конкретного значения этого параметра:<br /><br />С прямым порядком байтов<br /><br />С прямым порядком байтов<br /><br />**Режим по умолчанию**: с прямым порядком байтов<br /><br />**Режим оптимистичного**: с прямым порядком байтов<br /><br />**Полный режим**: с прямым порядком байтов|  
|**Блокировка таблицы**|Указывает, блокирует ли SSMA таблиц, при добавлении данных к таблицам во время переноса данных. Получает блокировку массовых обновлений в течение операции массового копирования. Если значение равно False, блокировка устанавливается на уровне строк.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**Использование курсоров**|Данные извлекаются из базы данных-источника с помощью курсоров, если этот параметр имеет значение.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
  
## <a name="parallel-data-migration"></a>Перенос параллельных данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Режим миграции параллельных данных**|Указывает режим, используемый для вилки потоков для включения параллельных операций миграции. В режиме Auto SSMA выбирает число потоков (10 по умолчанию), разделенными для переноса данных. В пользовательский режим, пользователь может указать число потоков, разделенными для переноса данных (минимальное значение — 1 и максимальным — 100). В настоящее время только части данных миграции механизм клиента поддерживает параллельных операций миграции.<br /><br />**Режим по умолчанию**: автоматически<br /><br />**Режим оптимистичного**: автоматически<br /><br />**Полный режим**: автоматически|  
  
> [!IMPORTANT]  
> При **параллельном режиме миграции данных** параметру присваивается **Custom**, создать проект, установка параметра **число потоков** отображается. Он указывает количество потоков, используемых для переноса данных.
