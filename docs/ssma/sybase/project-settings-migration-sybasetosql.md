# <a name="project-settings-migration-sybasetosql"></a>Параметры проекта (миграция) (SybaseToSQL)
На странице миграции **параметры проекта** диалоговое окно содержит настройки, установленные как SSMA выполняет миграцию данных из Sybase адаптивной Server Enterprise (ASE) для [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)].  
  
Области миграции доступен в обеих **параметры проекта** и **параметры проекта по умолчанию** диалоговым окнам.  
  
-   Для задания параметров для всех проектов SSMA на **средства** последовательно выберите пункты **параметры проекта по умолчанию**, выберите тип проекта миграции, для которого требуются параметры быть просмотреть / изменить из **миграции целевой версии** раскрывающегося списка пункт **Общие** в нижней части левой панели, а затем нажмите кнопку **миграции**.  
  
-   Чтобы указать параметры для текущего проекта на **средства** последовательно выберите пункты **параметры проекта**, нажмите кнопку **Общие** в нижней части левой панели, а затем нажмите кнопку **миграции**.  
  
## <a name="date-correction-options"></a>Параметры исправления даты  
  
|Термин|Определение|  
|--------|--------------|  
|**Замените неподдерживаемый дат**|Указывает, следует ли SSMA правильную дат, предшествующих самой ранней [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] **datetime** даты (01 января 1753 г.).<br /><br />Чтобы сохранить значения на текущую дату, выберите **ничего не**. [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]в столбце datetime не будет принимать дат, предшествующих 01 января 1753 г. При использовании старых даты, необходимо преобразовать значения даты и времени в символьные значения.<br /><br />Чтобы преобразовать даты до 01 января 1753 г. значение NULL, выберите **замените NULL**.<br /><br />Чтобы заменить дат, предшествующих 01 января 1753 поддерживаемая Дата, выберите **замените ближайшего поддерживаемая Дата**.<br /><br />**Режим по умолчанию**: не предпринимать никаких действий<br /><br />**Режим оптимистичного**: не предпринимать никаких действий<br /><br />**Полный режим**: замените ближайшего поддерживаемая Дата|  
  
## <a name="migration-engine"></a>Модуль переноса  
  
|Термин|Определение|  
|--------|--------------|  
|**Модуль переноса**|Указывает компонент database engine, используемый во время переноса данных. Миграцию данных клиента ссылается SSMA клиента, получении данных из источника и массовой вставке данных в SQL Server. Миграцию данных сервер ссылается SSMA миграции подсистема обработки данных (программа массового копирования), выполняющихся на компьютере SQL Server как задание агента SQL Server, извлечение данных из источника и вставка непосредственно в SQL Server, что позволяет избежать дополнительной клиента прыжок (производительность).<br /><br />**Режим по умолчанию**: Модуль переноса данных стороны клиента<br /><br />**Режим оптимистичного**: Модуль переноса данных стороны клиента<br /><br />**Полный режим**: Модуль переноса данных стороны клиента|  
  
> [!IMPORTANT]  
> Когда **модуль переноса** включен режим **модуль переноса данных стороне сервера**, создать проект установки параметра **модуль переноса данных стороне сервера использовать 32-разрядных** отображается. Он указывает, используется ли 32-разрядной или 64-разрядная версия программы программы массового копирования (BCP) для переноса данных.  
  
## <a name="miscellaneous-options"></a>Различные параметры  
  
|Термин|Определение|  
|--------|--------------|  
|**Размер пакета**|Указывает пакет, размер, используемый во время переноса данных.<br /><br />**Режим по умолчанию**: 10000<br /><br />**Режим оптимистичного**: 10000<br /><br />**Полный режим**: 10000|  
|**Проверочные ограничения**|Указывает, должна ли SSMA проверять ограничения при вставке данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
|**Время ожидания миграции данных**|Указывает время ожидания, используемое во время переноса данных<br /><br />**Режим по умолчанию**: 15<br /><br />**Режим оптимистичного**: 15<br /><br />**Полный режим**: 15|  
|**Параметры миграции расширенные данные**|Показывает параметры миграции дополнительные данные для каждой таблицы в вкладке отдельных сведений.<br /><br />**Режим по умолчанию**: скрытие<br /><br />**Режим оптимистичного**: скрытие<br /><br />**Полный режим**: скрытие|  
|**Запускать триггеры**|Указывает, будет ли SSMA следует запускать триггеры вставки, при добавлении данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
|**Сохранять ИД**|Указывает, сохраняет ли SSMA Sybase значений идентификаторов при добавлении данных в SQL Server. Значение False приводит значений идентификаторов, которые назначены целевым объектом.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**Сохранять значения NULL**|Указывает, сохраняет ли SSMA значения null в источнике данных, при добавлении данных в SQL Server независимо от значения по умолчанию, заданные в SQL Server.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**В случае ошибки**|Остановка переноса данных при возникновении ошибки. Он имеет три параметра:<br /><br />**Остановить миграцию:** останавливает операции переноса данных<br /><br />**Перейти к следующей таблицы:** останавливает перенос данных в текущей таблице и переход к следующему<br /><br />**Перейти к следующему пакету:** останавливает переноса данных в текущий пакет и переходит к следующему<br /><br />**Режим по умолчанию**: перейти к следующему пакету<br /><br />**Режим оптимистичного**: перейти к следующему пакету<br /><br />**Полный режим**: перейти к следующему пакету|  
|**Round дробной части числа**|Указывает, следует ли обрезать дробных частей десятичных и числовых данных во время миграции к целочисленному типу или отобразить сообщение об ошибке, если дробная часть равна нетривиальной<br /><br />**Режим по умолчанию**: нет<br /><br />**Режим оптимистичного**: нет<br /><br />**Полный режим**: нет|  
|**Порядок байтов Юникод Sybase**|Тип порядком строк Юникода Sybase. Для конкретно этот параметр можно задать следующие параметры.<br /><br />Обратный порядок байтов<br /><br />Обратный порядок байтов<br /><br />**Режим по умолчанию**: обратный порядок байтов<br /><br />**Режим оптимистичного**: обратный порядок байтов<br /><br />**Полный режим**: обратный порядок байтов|  
|**Блокировка таблицы**|Указывает ли SSMA блокировки таблиц, при добавлении данных к таблицам во время переноса данных. Получает блокировку массовых обновлений в течение операции массового копирования. Если значение равно False, блокировка устанавливается на уровне строк.<br /><br />**Режим по умолчанию**: True<br /><br />**Режим оптимистичного**: True<br /><br />**Полный режим**: True|  
|**Использование курсоров**|Данные извлекаются из базы данных-источника с помощью курсоров, если этот параметр имеет значение.<br /><br />**Режим по умолчанию**: False<br /><br />**Режим оптимистичного**: False<br /><br />**Полный режим**: False|  
  
## <a name="parallel-data-migration"></a>Перенос параллельных данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Режим миграции параллельных данных**|Указывает режим, используемый для вилки потоков для включения параллельных данных миграции. В режиме Auto SSMA выбирает число потоков (по умолчанию — 10), разделенными для переноса данных. В режиме другой пользователь может указать число потоков, разделенными для переноса данных (минимальное значение равно 1 и максимальным — 100). В настоящее время только клиент стороны миграции подсистема обработки данных поддерживает параллельных данных миграции.<br /><br />**Режим по умолчанию**: Auto<br /><br />**Режим оптимистичного**: Auto<br /><br />**Полный режим**: Auto|  
  
> [!IMPORTANT]  
> При **параллельном режиме миграции данных** включен режим **настраиваемый**, создать проект установки параметра **число потоков** отображается. Он указывает количество потоков, используемых для переноса данных.
