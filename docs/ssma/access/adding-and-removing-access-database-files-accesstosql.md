# <a name="adding-and-removing-access-database-files-accesstosql"></a>Добавление и удаление базы данных Access (AccessToSQL)
Для переноса данных Access [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure, необходимо добавить один или несколько баз данных Access в проект SSMA. Эти базы данных должны быть Access 97 или более поздней версии. При наличии баз данных из более ранней версии Access, необходимо преобразовать базы данных до более новой версии. Это делается путем открытия и сохранения баз данных Microsoft Access 97 или более поздней версии, прежде чем добавлять их SSMA.  
  
## <a name="what-happens-when-you-add-access-database-files"></a>Что происходит при добавлении файлов баз данных Access?  
При добавлении базы данных Access в проект SSMA SSMA считывает метаданные базы данных и затем добавляет эти метаданные в файл проекта. Эти метаданные описание базы данных и ее объектов. SSMA использует метаданные, когда он преобразует объекты в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или синтаксис SQL Azure, и если его перемещения данных для [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure. Можно просмотреть эти метаданные в обозревателе метаданных доступа и просмотрите свойства для отдельных объектов базы данных.  
  
> [!NOTE]  
> В базе данных Access можно разбить на несколько файлов: серверную базу данных, который содержит таблицы и интерфейсные базы данных, содержащие запросы, формы, отчеты, макросы, модули и сочетания клавиш. Если вы хотите перенести разбиение базы данных в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure, добавление SSMA клиентской базы данных.  
  
## <a name="permissions-that-are-required-by-ssma"></a>Разрешения, которые требуются SSMA  
Для переноса базы данных Access для [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure, группы "пользователи" и пользователя Admin необходимо иметь разрешение на администрирование. Сведения о том, как выполнить миграцию баз данных с защитой рабочих групп см. в разделе [Подготовка базы данных Access для миграции](http://msdn.microsoft.com/en-us/9b80a9e0-08e7-4b4d-b5ec-cc998d3f5114)  
  
## <a name="selecting-databases-to-add"></a>Выбор баз данных для добавления  
Если вы хотите добавить одну или несколько баз данных в проект SSMA файлы находятся в одном месте известных, эти файлы можно добавить с помощью следующей процедуры.  
  
**Чтобы добавить отдельные файлы базы данных**  
  
1.  На **файл** меню, нажмите кнопку **добавление баз данных**.  
  
2.  В **откройте** диалогового окна поле, найдите папку, содержащую файл базы данных или файлов.  
  
3.  Выберите файлы, которые вы хотите добавить и нажмите кнопку **откройте**.  
  
## <a name="finding-databases-to-add"></a>Поиск базы данных для добавления  
Если требуется добавить несколько баз данных Access из разных папок в проект SSMA или требуется добавить один файл, однако для поиска файла, можно выполните следующие действия для одного из нескольких файлов и добавьте их в проект.  
  
**Чтобы найти и добавить базы данных**  
  
1.  На **файл** меню, нажмите кнопку **найти баз данных**.  
  
2.  В мастере найти базы данных введите имя диска, путь к файлу или UNC-путь, который требуется найти. Можно также щелкнуть **Обзор** для поиска диск или сетевой папки.  
  
3.  Нажмите кнопку **добавить** для добавления в список расположение.  
  
    Повторите два предыдущих шага для дополнительных местоположений поиска.  
  
4.  При необходимости добавьте условия поиска для уточнения списка баз данных, которые возвращаются.  
  
    > [!IMPORTANT]  
    > **Полностью или частично имя файла** текстовое поле не поддерживает подстановочные знаки.  
  
5.  Нажмите кнопку **сканирования**.  
  
    Появится страница сканирования. Это показывает, что базы данных, которые были найдены и ход выполнения поиска. Чтобы остановить поиск, нажмите кнопку **остановить**.  
  
6.  На странице «Выбор файлов» выберите базы данных, которые требуется добавить в проект.  
  
    Можно использовать **выделить все** и **очистить все** кнопок в верхней части списка, чтобы выбрать или очистить все базы данных. Удерживая нажатой клавишу CTRL, чтобы выбрать несколько баз данных или удерживайте клавишу SHIFT, вниз, чтобы выбрать диапазон баз данных.  
  
7.  Нажмите кнопку **Далее**.  
  
8.  На странице «Проверка» щелкните **Готово**.  
  
## <a name="browsing-access-metadata"></a>Просмотр метаданных доступа  
После добавления в проект базы данных Access, метаданные проекта отображается в обозревателе метаданных доступа. Вы можете просматривать иерархию баз данных и объектов базы данных в обозревателе.  
  
**Для просмотра метаданных**  
  
1.  В обозревателе метаданных доступа разверните **доступа к метабазе**, а затем разверните **баз данных**.  
  
2.  Разверните базу данных, которую требуется просмотреть, а затем разверните **запросы**.  
  
    Обратите внимание на список запросов. Если выбран запрос, **SQL** вкладку и **свойства** вкладка отображается в правой области.  
  
3.  Разверните **таблиц** и затем выберите таблицу.  
  
    Обратите внимание, что отображаются четыре вкладки: **таблицы**, **сопоставление типов**, **свойства**, и **данные**.  
  
4.  Разверните таблицу, затем **ключей**, а затем выберите ключ.  
  
    Ключевые свойства отображаются в правой области.  
  
5.  Разверните **индексы**, а затем выберите индекса.  
  
    В правой области отображаются свойства индекса.  
  
## <a name="refreshing-databases"></a>Обновление баз данных  
При изменении базы данных Access после добавления файла, его можно обновить метаданные из базы данных Access.  
  
**Чтобы обновить метаданные доступа**  
  
-   В обозревателе метаданных доступ, щелкните правой кнопкой мыши базу данных и выберите **обновление из базы данных**.  
  
## <a name="removing-databases"></a>Удаление баз данных  
В базе данных Access можно удалить из проекта, выполните следующие действия.  
  
**Чтобы удалить базу данных из проекта.**  
  
1.  В обозревателе метаданных доступа разверните **доступа к метабазе**, а затем разверните **баз данных**.  
  
2.  Щелкните правой кнопкой мыши базу данных, а затем выберите **удалить базу данных**.
