# <a name="converting-access-database-objects-accesstosql"></a>Преобразование объектов базы данных Access (AccessToSQL)
После добавления базы данных Access и подключены к [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure, SSMA отображает метаданные для доступа и [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или объекты базы данных SQL Azure. Теперь выберите объекты базы данных Access и затем преобразуйте схем в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или схем SQL Azure.  
  
## <a name="the-conversion-process"></a>Процесс преобразования  
Преобразование объектов базы данных принимает определения объектов из получить доступ к метаданным, преобразует их в эквивалент [!INCLUDE[tsql](../../includes/tsql_md.md)] синтаксис, а затем загружает эти данные в проект. Затем можно просмотреть [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] объектов SQL Azure и их свойства с помощью или [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или обозреватель метаданных SQL Azure.  
  
> [!IMPORTANT]  
> Преобразование объектов не создает объекты в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure. Только преобразует определения объектов и сохраняет их в проект SSMA.  
  
Во время преобразования SSMA выводит состояние в области вывода и ошибки, предупреждения и информационные сообщения, на панели «Список ошибок». Эти сведения можно используйте для определения, следует ли изменять баз данных Access или процесс преобразования для получения результатов требуемое преобразование. Можно также использовать сведения в [Подготовка базы данных Access для миграции](http://msdn.microsoft.com/en-us/9b80a9e0-08e7-4b4d-b5ec-cc998d3f5114) раздел, чтобы определить, что будет и не будут преобразованы.  
  
## <a name="setting-conversion-options"></a>Установка параметров преобразования  
Перед преобразованием объектов, проверьте параметры преобразования проекта в **параметры проекта** диалоговое окно. В этом диалоговом окне, можно задать как SSMA преобразует memo индексированные столбцы, первичные ключи, ограничения внешнего ключа, отметки времени и таблицы без индексов. Дополнительные сведения см. в разделе [параметры проекта (преобразование)](http://msdn.microsoft.com/en-us/bcebc635-c638-4ddb-924c-b9ccfef86388)

## <a name="converting-access-objects"></a>Преобразование объектов доступа  
Чтобы преобразовать объекты баз данных Access, сначала необходимо выбрать объекты, которые требуется преобразовать, а затем выполните преобразование SSMA. Для просмотра выходных сообщений во время преобразования на **представление** последовательно выберите пункты **вывода**.  
  
**Чтобы выбрать и преобразование объектов базы данных Access в синтаксис SQL Server или SQL Azure**  
  
1.  В обозревателе метаданных доступа разверните **доступа к метабазе**, а затем разверните **баз данных**.  
  
2.  Выполните одно или несколько из следующих действий.  
  
    -   Чтобы преобразовать все базы данных, установите флажок рядом с **баз данных**.  
  
    -   Чтобы преобразовать или пропустите отдельных баз данных, установите или снимите флажок рядом с именем базы данных.  
  
    -   Преобразовать или исключив некоторые запросы, разверните базу данных, а затем установите или снимите **запросы** флажок.  
  
    -   Чтобы преобразовать или пропустите отдельных таблиц, разверните базу данных, разверните узел **таблиц**и затем установите или снимите флажок рядом с таблицей.  
  
3.  Выполните одно из следующих действий.  
  
    -   Чтобы преобразовать схемы, щелкните правой кнопкой мыши **баз данных** и выберите **преобразование схемы**.  
  
        Можно также преобразовать отдельных объектов. Для преобразования объекта, независимо от выбранных объектов, щелкните правой кнопкой мыши объект и выберите **преобразование схемы**.  
  
        При преобразовании объекта отображается полужирным шрифтом в обозревателе метаданных доступа.  
  
    -   Чтобы преобразовать, загрузить и перенос схемы и данные за один шаг, щелкните правой кнопкой мыши баз данных и выберите **преобразовать, загрузить и перенести**.  
  
4.  Просмотрите сообщения в **вывода** область и все ошибки и предупреждения в **список ошибок** области.  
  
## <a name="altering-tables-and-indexes"></a>Изменение таблиц и индексов  
После преобразования получить доступ к метаданным для [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или метаданных SQL Azure, и перед их загрузкой объектов в [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или изменить SQL Azure, [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или SQL Azure таблиц и индексов.  
  
**Изменить свойства таблицы или индекса**  
  
1.  В [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] или обозреватель метаданных SQL Azure, выберите таблицы или индекса, которые требуется изменить.  
  
2.  На **таблицы** щелкните свойство, необходимо изменить, а затем введите или выберите новый параметр. Например можно сделать nvarchar(15) nvarchar(20) или установите флажок, чтобы сделать столбец таблицы допускает значение NULL.  
  
    Переместите курсор за пределы ячейки измененного свойства. Это можно сделать, щелкнув другой строкой или нажав клавишу Tab.  
  
3.  Нажмите кнопку **Применить**.  
  
Теперь можно просмотреть изменения в коде на **SQL** вкладки.
