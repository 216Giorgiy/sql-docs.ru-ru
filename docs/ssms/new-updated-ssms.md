# <a name="new-and-recently-updated-sql-server-management-studio-ssms-for-sql-server"></a>Новые и недавно обновленные: SQL Server Management Studio (SSMS) для SQL Server



Почти каждый день Корпорация Майкрософт обновляет некоторые из его существующих статей на его [Docs.Microsoft.com](http://docs.microsoft.com/) документации веб-сайта. В этой статье отображает выдержки из недавно обновлены статьи. Ссылки на новые статьи также может быть указан.

- *Диапазон дат для обновлений:* &nbsp; **23.05.2017** &nbsp;—&nbsp; **17.07.2017**

**Выпуск SSMS 17.1**

Общедоступной | Номер сборки: 14.0.17119.0

**Усовершенствования**


- Profiler: Справка > около теперь отображает номер версии выпуска (например, 17,1)
- Анализ пользователей службы можно обновить учетные данные для своих источников данных для 1200 TM моделей и более поздних версий в контекстном меню для источника данных
- Теперь Показать журналы с горизонтальным масштабированием выполнения служб SSIS в CTP-версии 2.1 сообщает встроенных служб SSIS
- Приложения масштабирования управления служб SSIS
  - Общие сведения о master горизонтального масштабирования.
  - Легко добавьте рабочий процесс для масштабного развертывания.
  - Просмотр всех работников для масштабирования и основные сведения о них и можно также включить или отключить их легко.

**Исправления ошибок**

- Always On:
  - Устранена проблема, где свойства реплики доступности была всегда отображается как режим «Автоматическая отработка отказа» для групп доступности WSFC.
  - Устранена проблема, из которой был перезаписан списка маршрутизации только для чтения, при обновлении группы доступности
- Постоянное шифрование: исправлена проблема, где создан файл журнала отсутствуют данные, созданные в DacFx.
- Инструкции ShowPlan: исправлена в выпуске где пользовательский Интерфейс всегда показывают атрибута типа объединения фактический для операторы не адаптивной соединения.
- Программа установки:
  - Устранена проблема, где SSMS 17.0 разрыв SSDT для Visual Studio 2013 [подключения элемент 3133479]
  - Устранена проблема, где щелкните «Перезапуск» в конце установки не перезапускает компьютер
- Сценарии: временно препятствует SSMS случайное удаление объектов базы данных SQL Azure, при попытке создать скрипт удаления, отключив этот параметр.  В будущей версии SSMS будут соответствующие исправления.
- Обозреватель объектов: исправлена проблема, где узел «Базы данных» не был развернут при соединении с базой данных Azure, созданные с помощью «Копировать AS»
