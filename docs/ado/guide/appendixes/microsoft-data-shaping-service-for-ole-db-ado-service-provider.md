# <a name="microsoft-data-shaping-service-for-ole-db-overview"></a>Службы OLE DB Обзор формирования данных Microsoft
> [!IMPORTANT]
>  Этот компонент будет удален в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого приложения должны использовать XML.

 Служба формирования данных Microsoft для поставщика OLE DB для служб поддерживает конструирование иерархической (форму) [записей](../../../ado/reference/ado-api/recordset-object-ado.md) объектов из поставщика данных.

## <a name="provider-keyword"></a>Ключевое слово поставщика
 Для вызова службы Data Shaping Service для OLE DB, укажите следующие значения и ключевое слово в строке подключения.

```
"Provider=MSDataShape"
```

## <a name="dynamic-properties"></a>Динамические свойства
 При вызове этого поставщика услуг, добавляются следующие динамические свойства [свойства](../../../ado/reference/ado-api/properties-collection-ado.md) коллекцию[подключения](../../../ado/reference/ado-api/connection-object-ado.md) объекта.

|Имя динамического свойства|Описание|
|---------------------------|-----------------|
|**Изменения формы уникальные имена**|Указывает ли **записей** объектов с повторяющимися значениями для их **изменить имя** свойства допускаются. Если это динамическое свойство **True** и новый **записей** создается с тем же именем пользовательские изменения формы, что и у существующей **набора записей**, новый  **Набор записей** изменения формы имени объекта изменяется, чтобы сделать уникальным. Если это свойство имеет **False** и новый **записей** создается с тем же именем пользовательские изменения формы, как и существующие **записей**, оба **набора записей**  объектов будет иметь то же имя изменения формы. Таким образом, ни **записей** форму можно изменить до тех пор, пока существует в обоих наборах записей.<br /><br /> Значение свойства по умолчанию — **False**.|
|**Поставщик данных**|Указывает имя поставщика, который будет передавать строки, чтобы иметь форму. Это значение может быть NONE, если поставщик не будет использоваться для предоставления строк.|

 Можно также задать для записи динамические свойства, указав их имена в качестве ключевых слов в строке подключения. Например, в Microsoft Visual Basic, задайте **поставщик данных** динамических свойств для «MSDASQL», указав:

```
Dim cn as New ADODB.Connection
cn.Open "Provider=MSDataShape;Data Provider=MSDASQL"
```

 Также можно задать или получить динамических свойств, указав его имя в качестве индекса в [свойства](../../../ado/reference/ado-api/properties-collection-ado.md) свойство. Например, в следующем примере кода получает и печатает значение текущей **поставщик данных** динамических свойств, затем устанавливает новое значение, если cn. DataProvider присвоено значение «MSDataShape» (прямо или косвенно через строку подключения), и не было открыто соединение:

```
Debug.Print cn.Properties("Data Provider")
cn.Properties("Data Provider") = "MSDASQL"
```

> [!NOTE]
>  Динамическое свойство **поставщик данных**, может быть задан только на неоткрытый **подключения** объекта. После открытия подключения **поставщик данных** свойство становится доступным только для чтения.

 Дополнительные сведения о формировании данных см. в разделе [формирование](../../../ado/guide/data/data-shaping-overview.md).
