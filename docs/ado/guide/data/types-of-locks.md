# <a name="types-of-locks"></a>Типы блокировки
## <a name="adlockbatchoptimistic"></a>adLockBatchOptimistic  
 Указывает оптимистичный пакетных обновлений. Требуется для пакетного режима обновления.  
  
 Во многих приложениях получить несколько строк за один раз и затем нужно проверить скоординированного обновлений, которые включают полный набор строк для вставки, обновления или удаления. С курсорами пакета, только один цикл к серверу требуется, повышая производительность обновления и сократить сетевой трафик. Использование библиотеки курсоров пакета, можно создать статический курсор и затем отключиться от источника данных. На этом этапе можно вносить изменения в строки и впоследствии подключиться и поместить их в источник данных в пакете.  
  
## <a name="adlockoptimistic"></a>adLockOptimistic  
 Указывает, что поставщик использует оптимистической блокировки — блокировки записи только при вызове **обновление** метод. Это означает, что есть вероятность, что другой пользователь может изменить данные в промежутке между изменить запись и при вызове **обновление**, что создает конфликт. Используйте этот тип блокировки в случаях, когда не низкая вероятность конфликта или где можно легко разрешить конфликты.  
  
## <a name="adlockpessimistic"></a>adLockPessimistic  
 Указывает пессимистичных блокировок, по одной записи. Поставщик не то, что является обязательным для обеспечения успешного редактирования записей, обычно путем блокировки записей в источнике данных непосредственно перед редактированием. Конечно, это означает, что записи недоступны для других пользователей, после начала для редактирования, пока снять блокировку путем вызова **обновления.** Используйте этот тип блокировки в системе, где не может позволить себе иметь одновременных изменений данных, например, в системе резервирования.  
  
## <a name="adlockreadonly"></a>adLockReadOnly  
 Указывает записи только для чтения. Невозможно изменить данные. Блокировку только для чтения является тип блокировки «быстрый», так как он не сервер должен поддерживать блокировку записи.  
  
## <a name="adlockunspecified"></a>adLockUnspecified  
 Не указывает тип блокировки.
