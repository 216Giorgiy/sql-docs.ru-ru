# <a name="anticipating-errors"></a>Планирование ошибок
Предотвращение ошибок по крайней мере столь же важно, как обработка ошибок. В этом последнем разделе содержит краткого списка меры предосторожности, приложение может воспользоваться для того, чтобы вероятность возникновения ошибок.  
  
 Проверьте состояние объектов путем проверки значения **состояние** свойство перед попыткой выполнения операции с помощью этих объектов. Например, если приложение использует глобальный **подключения**, проверьте его **состояние** свойство, чтобы узнать, если он уже открыт перед вызовом метода **откройте** метод.  
  
-   Любая программа, которая принимает данные от пользователя необходимо включить код для проверки данных перед их отправкой в хранилище данных. Нельзя полагаться на хранилище данных, поставщик, ADO или даже на языке программирования для уведомления о проблемах. Необходимо проверить каждый байт, введенные на пользователей, убедившись, что данные являются правильный тип поля и обязательные поля не заполнены.  
  
 Проверьте данные, прежде чем пытаться выполнить запись данных в хранилище данных. Самый простой способ сделать это — для обработки **WillMove** событий или **WillUpdateRecordset** событий. Более подробные сведения об обработке событий ADO, в разделе [обработка событий ADO](../../../ado/guide/data/handling-ado-events.md).  
  
 Убедитесь, что **записей** объекты не находятся за пределами **записей** перед попыткой перемещения указателя записи. Если вы попытаетесь **MoveNext** при **EOF** имеет значение True или **MovePrev** при **BOF** имеет значение True, возникнет ошибка. При выполнении любого из **переместить** методов при обоих **EOF** и **BOF** имеют значение True, будет сформирована ошибка.  
  
 Также возникнут ошибки при попытке выполнения операций, таких как **Seek** и **найти** с пустым **записей**.
