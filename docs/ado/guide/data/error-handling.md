# <a name="error-handling"></a>Обработка ошибок
ADO использует несколько различных методов для оповещения приложения, возникающие ошибки. В этом разделе рассматриваются типы ошибок, которые могут возникнуть при использовании ADO, а также способ уведомления приложения. Заключительный шаг, делая предложения о том, как обрабатывать эти ошибки.  
  
## <a name="how-does-ado-report-errors"></a>Как ADO отчеты об ошибках  
 ADO уведомления об ошибках несколькими способами:  
  
-   ADO ошибки вызывают ошибку времени выполнения. Обработать ошибку ADO так же, как любая другая ошибка времени выполнения, например, с помощью **On Error** инструкции на языке Visual Basic.  
  
-   Программы могут получать ошибки OLE DB. Ошибка OLE DB создает ошибку времени выполнения.  
  
-   Если данная ошибка относится к поставщику данных, один или несколько **ошибка** объекты размещаются в **ошибки** коллекцию **подключения** объект, который был использован для доступа к данным хранилище, когда произошла ошибка.  
  
-   Если процесс, который вызывает событие также возникает ошибка, сведения об ошибке помещается в **ошибка** объекта и события, передаваемый как параметр. В разделе [обработка событий ADO](../../../ado/guide/data/handling-ado-events.md) Дополнительные сведения о событиях.  
  
-   Проблемы, возникающие при обработке пакета обновления или другие массовой операции, связанные с **записей** может быть указана **состояние** свойство **записей**. Например, нарушения ограничений схемы или недостаточно разрешений можно указать по **RecordStatusEnum** значения.  
  
-   Возможных неполадок, связанных с конкретным **поле** в текущей записи также обозначаются с **состояние** каждого экземпляра **поле** в **полей**  коллекцию **запись** или **записей**. Например, обновления, которые не удалось выполнить несовместимые типы данных можно указать или по **FieldStatusEnum** значения.
