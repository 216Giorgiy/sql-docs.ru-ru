# <a name="basic-rds-programming-model"></a>Модель программирования основных служб удаленных рабочих СТОЛОВ
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ больше не включаются в операционной системе Windows (в разделе Windows 8 и [руководство по Windows Server 2012 совместимости](https://www.microsoft.com/en-us/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будут удалены в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ необходимо перенести в [службы данных WCF](http://go.microsoft.com/fwlink/?LinkId=199565).  
  
 RDS адреса приложений, имеющихся в следующее окружение: клиентское приложение определяет программу, которая будет выполняться на сервере, а также параметры, необходимые для возвращать нужные сведения. Программа, вызванная при доступе повышение сервера для указанного источника данных, получает сведения о, при необходимости обрабатывает данные и затем возвращает полученные результаты в форме, которую можно легко использовать клиентское приложение. Служб удаленных рабочих СТОЛОВ предоставляет средства можно выполнить следующую последовательность действий:  
  
-   Укажите программу для вызова на сервере и получить ссылку на него из клиента. (Эта ссылка иногда называется *прокси-сервера*. Он представляет программу удаленного сервера. Клиентское приложение «вызовет» прокси-сервера, как если бы он был локальных программ, но вызывает программу удаленного сервера.)  
  
-   Необходимо вызовите программу сервера. Передача параметров определения источника данных и для выдачи команды программы-сервера. (Программа server фактически использует ADO для доступа к источнику данных. ADO устанавливает соединение с одной из заданных параметров, а затем выдает команды, указанной в другом параметре.)  
  
-   Программа сервер получает [записей](../../../ado/reference/ado-api/recordset-object-ado.md) из источника данных. При необходимости **записей** объект обрабатывается на сервере.  
  
-   Программа server Возвращает конечное **записей** объекта клиентскому приложению.  
  
-   На стороне клиента **записей** объект помещается в форму, которую можно легко использовать визуальные элементы управления.  
  
-   Изменения в **записей** объекта отправляются обратно в программе server, который использует их для обновления источника данных.  
  
 Эта модель программирования содержит функции, определенные удобства. Если программу сложного сервера для доступа к источнику данных не требуется, и если предоставить необходимые параметры подключения и команды служб удаленных рабочих СТОЛОВ будут автоматически найти нужные данные с простой, серверной программы по умолчанию.  
  
 Если требуется более сложную обработку, можно указать собственную программу на другой сервер. Например так как программа пользовательского сервера всю мощь ADO в ее распоряжении, он может подключаться к несколько различных источников данных, объединять свои данные каким-либо образом сложный и затем возвращают результат простой, обработанные в клиентское приложение.  
  
 Наконец Если вашим потребностям, где-то между ADO теперь поддерживает настройки поведения серверной программы по умолчанию.
