# <a name="understanding-the-customization-file"></a>Основные сведения о настройке файла
Заголовок каждого раздела в файле настройки состоит из квадратных скобок (**[]**) содержащего типа и параметров. Типы четыре раздела обозначаются строковых литералов **подключения**, **sql**, **userlist**, или **журналы**. Параметр является символьная строка, значение по умолчанию, пользовательским идентификатором или ничего.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ больше не включаются в операционной системе Windows (в разделе Windows 8 и [руководство по Windows Server 2012 совместимости](https://www.microsoft.com/en-us/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будут удалены в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ необходимо перенести в [службы данных WCF](http://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Таким образом каждый раздел помечен с помощью одного из следующих заголовков раздел:  
  
```  
  
[ connect default ] [ connect    
identifier   
] [ sql  default ] [ sql    
identifier   
] [ userlist    
identifier   
] [ logs ]  
  
```  
  
 Заголовки разделов состоят из следующих частей.  
  
|Часть|Description|  
|----------|-----------------|  
|**подключение**|Строковый литерал, который изменяет строку подключения.|  
|**SQL**|Строковый литерал, предназначенная для изменения командной строки.|  
|**USERLIST**|Строковый литерал, который изменяет права доступа для конкретного пользователя.|  
|**журналы**|Строковый литерал, указывающее оперативной ошибки записи файла журнала.|  
|**по умолчанию**|Строковый литерал, который используется, если идентификатор не указано или найдено.|  
|*Идентификатор*|Строка, соответствующая строка в **подключения** или **команда** строки.<br /><br /> -Используйте этот раздел, если длина заголовка содержит **подключения** , в строке подключения найдены строки идентификатора.<br />-Используйте этот раздел, если длина заголовка содержит **sql** и идентификатор строка найдена в командную строку.<br />-Используйте этот раздел, если длина заголовка содержит **userlist** и строка идентификатора соответствует **подключения** идентификатор секции.|  
  
 **DataFactory** вызывает обработчик, передача параметров клиента. Обработчик выполняет поиск полных строк в параметрах клиента, которые соответствуют идентификаторам в заголовках соответствующий раздел. Если соответствие найдено, содержимое этого раздела применяются для параметра клиента.  
  
 Определенный раздел используется в следующих случаях:  
  
-   Объект **подключения** раздел используется, если ключевое слово строки подключения значение часть клиента»**источник данных =***значение*», соответствует **подключения** Идентификатор раздела*.*  
  
-   **Sql** раздел используется, если строку команды клиента содержит строку, которая соответствует **sql** идентификатор секции.  
  
-   Объект **подключения** или **sql** раздел с параметром по умолчанию используется в том случае, если совпадающий идентификатор.  
  
-   Объект **userlist** раздел используется в том случае, если **userlist** статьи идентификатор соответствует **подключения** идентификатор секции. Если соответствие, содержимое **userlist** разделе применяются к соединению, принимается в соответствии с **подключения** раздела.  
  
-   Если строка в строке соединения или команда не соответствует идентификатор в любом **подключения** или **sql** раздел заголовка и не **подключения** или **sql**  раздел заголовка с параметром по умолчанию, то строки клиента используется без изменений.  
  
-   **Журналы** раздел используется каждый раз, когда **DataFactory** в операции.
