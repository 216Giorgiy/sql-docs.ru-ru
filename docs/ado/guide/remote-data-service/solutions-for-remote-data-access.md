# <a name="solutions-for-remote-data-access"></a>Решения для удаленного доступа к данным
## <a name="the-issue"></a>Проблема  
 ADO позволяет вашему приложению напрямую получать доступ к и изменение источников данных (иногда называемый двухуровневую систему). Например при подключении к источнику данных, содержащего данные, не в двухуровневую систему прямое подключение.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ больше не включаются в операционной системе Windows (в разделе Windows 8 и [руководство по Windows Server 2012 совместимости](https://www.microsoft.com/en-us/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будут удалены в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ необходимо перенести в [службы данных WCF](http://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Тем не менее может потребоваться доступ к источникам данных косвенно через посредника, такие как Microsoft® Internet Information Services (IIS). Такой подход иногда называют трехуровневые системы. IIS является система клиент сервер, который является эффективным способом для локальной или клиента, приложения для запуска программы удаленный или серверный, через Интернет или интрасеть. Серверная программа получает доступ к источнику данных и при необходимости обрабатывает данные.  
  
 Например веб-страницы интрасети содержит приложения, написанного в Microsoft® Visual Basic Scripting Edition (VBScript), который подключается к серверу IIS. IIS подключается к источнику данных в свою очередь, получает данные, обрабатывает его иным образом и затем возвращает обработанные данные в приложение.  
  
 В этом примере приложение никогда не прямое подключение к источнику данных; Было IIS. И IIS запрашиваемые данные с помощью ADO.  
  
> [!NOTE]
>  Клиент серверное приложение не имеет должна быть основана на Интернете или интрасети (то есть веб сервера) — оно может состоять только из скомпилированные программы в локальной сети. Тем не менее типичный случай — это веб-приложение.  
  
 Так как некоторые визуальному элементу управления, например сетки, флажок или список, может использовать возвращенные сведения, возвращенные сведения должны легко использоваться визуального элемента.  
  
 Вы хотите просто и эффективно интерфейса программирования приложений, поддерживает три уровня системы, который возвращает данные в виде легко как если бы он был загружен на двухуровневую систему. Служба удаленных данных (RDS) — это интерфейс.  
  
## <a name="the-solution"></a>Решение  
 Служб удаленных рабочих СТОЛОВ определяет модель программирования — последовательность действий, необходимых для получения доступа к и обновление источника данных, чтобы получить доступ к данным через посредника, например Internet Information Services (IIS). Модель программирования перечислены все функциональные возможности RDS.
