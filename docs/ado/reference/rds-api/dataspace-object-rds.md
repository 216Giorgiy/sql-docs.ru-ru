# <a name="dataspace-object-rds"></a>Объект пространства данных (RDS)
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ больше не включаются в операционной системе Windows (в разделе Windows 8 и [руководство по Windows Server 2012 совместимости](https://www.microsoft.com/en-us/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будут удалены в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ необходимо перенести в [службы данных WCF](http://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Создает клиентский прокси для пользовательских бизнес-объектов, расположенных на среднем уровне.  
  
 Удаленной службы данных должна бизнеса объект прокси-серверы, чтобы клиентские компоненты могут взаимодействовать с бизнес-объектов, расположенных на среднем уровне. Учетные записи-посредники упрощения упаковки, распаковке и транспорта (маршалинг) приложения [записей](../../../ado/reference/ado-api/recordset-object-ado.md) данных через границы процесса или компьютера.  
  
 Служба удаленного данных использует **RDS. Пространство данных** объекта [CreateObject](../../../ado/reference/rds-api/createobject-method-rds.md) метод для создания объекта business учетных записей-посредников. Business proxy объекта создается динамически при создании экземпляра его аналога среднего уровня бизнес-объекта. Служба удаленных данных поддерживает следующие протоколы: HTTP, HTTPS (HTTP протокола SSL), DCOM и в процессе (клиентские компоненты и бизнес-объекта, которые находятся на том же компьютере).  
  
> [!NOTE]
>  Служб удаленных рабочих СТОЛОВ «без отслеживания состояния» функционирует так же при **RDS. Пространство данных** объект использует протокол HTTP или HTTPS. То есть любые внутренние сведения о клиентском запросе удаляются после сервер возвращает ответ.  
  
> [!NOTE]
>  Хотя бизнес-объект существует в течение времени существования объекта business proxy, бизнес-объекта, существуют только до отправки ответа на запрос. Если выдается запрос (то есть, вызывается метод в бизнес-объекте), прокси-сервер открывает новое подключение к серверу, и сервер создает новый экземпляр бизнес-объекта. После бизнес-объект отвечает на запрос, сервер окончательно удаляет бизнес-объекта и закрывает соединение.  
  
> [!NOTE]
>  Это означает, что не может передавать данные из одного запроса, другой с помощью свойство бизнес-объекта или переменной. Необходимо использовать другой механизм, например файла или аргумент метода для сохранения данных состояния.  
  
 Идентификатор класса для **RDS. Пространство данных** объект является BD96C556 65A3 - 11 D 0-983A-00C04FC29E36.  
  
 **DataSpace** объекта безопасные для использования.
