# <a name="acquire-and-configure-a-backup-server-for-parallel-data-warehouse"></a>Получить и настроить резервный сервер для параллельного хранилища данных
В этой статье описывается настройка системы Windows не является специализированным как резервный сервер для использования с функциями резервного копирования и восстановления в Analytics Platform System (APS) и параллельные хранилища данных (PDW).  
  
  
## <a name="Basics"></a>Основы резервное копирование сервера  
Резервное копирование сервера:  
  
-   Предоставляется и управляется ИТ команды.  
  
-   Не требует определенного PDW программы или средства. PDW не устанавливает программное обеспечение на сервер резервного копирования.  
  
-   Расположен в стойку не является специализированным и не могут находиться внутри APS устройства.  
  
-   Можно подключить к сети InfiniBand устройства. Резервные копии могут быть выполнены над InfiniBand или Ethernet; Для повышения производительности рекомендуется InfiniBand.  
  
-   Находится в ваш домен клиента, не домен устройства. Нет отношения доверия между доменом клиента и домен устройства.  
  
-   Размещается в общей папке файла резервной копии, что является общей папкой Windows, который использует протокол Server Message Block (SMB) сети на уровне приложения. Разрешения для общего ресурса файла резервной копии присвойте пользователя домена Windows (обычно это пользователь выделенного резервного копирования) возможность выполнить резервное копирование и восстановление операций на этом ресурсе. Учетные данные пользователя и пароль пользователя домена Windows, хранятся в PDW, PDW можно выполнить резервное копирование и восстановление операций в папке файлов резервных копий.  
  
## <a name="Step1"></a>Шаг 1: Определите требования к емкости  
Требования к системе для резервного копирования сервера почти полностью зависят от рабочей нагрузкой. До приобретения и подготовки резервного копирования сервера необходимо понять требования к емкости. Резервное копирование не нужно быть предназначен только для резервных копий, при условии, что он будет обрабатывать требования к производительности и хранения рабочей нагрузки. Вы также можете несколько резервных серверов для резервного копирования и восстановления каждой базы данных на один из нескольких серверов.  
  
Используйте [лист планирования емкости сервера резервного копирования](backup-capacity-planning-worksheet.md) для определения требований к емкости.  
  
## <a name="Step2"></a>Шаг 2: Получение резервное копирование сервера  
Теперь, когда вы лучше понять требования к емкости, можно запланировать, серверами и сетевыми компонентами, которые потребуются для покупки или подготовки. Плана покупки с учетом перечисленных ниже требования и приобрести сервера или подготовить существующий сервер.  
  
### <a name="software-requirements"></a>Требования к программному обеспечению  
Любой файловый сервер, который использует протокол общего доступа к файлу Windows (SMB).  
  
Мы рекомендуем Windows Server 2012 или другие сведения для:  
  
-   Получите выигрыш в производительности предварительное выделение файлов по протоколу SMB.  
  
-   Мгновенная инициализация файлов (IFI) используется для операций резервного копирования. ИТ-специалисты управляет этот параметр на сервере резервного копирования. PDW Configuration Manager (dwconfig.exe) не настраивайте или управлять IFI на сервере резервного копирования. Предыдущие версии Windows не имеют IFI, но по-прежнему может использоваться в качестве резервного копирования серверов.  
  
### <a name="networking-requirements"></a>Требования к сети  
Хотя это необязательно, InfiniBand является тип подключения рекомендуется для серверов резервного копирования. Подготовка для подключения к сети InfiniBand устройства загрузка сервера:  
  
1.  Планирование для установки в стойку как можно ближе к устройству, чтобы вы могли подключаться к устройству переключается InfiniBand. Дополнительные сведения от технологий Mellanox о InfiniBand см. в документе [введение в InfiniBand](http://www.mellanox.com/pdf/whitepapers/IB_Intro_WP_190.pdf).  
  
2.  Приобрести одним или двумя порта сетевого адаптера InfiniBand FDR Mellanox ConnectX-3. Мы рекомендуем сетевой адаптер с двумя портами для обеспечения отказоустойчивости при передаче данных. Для обеспечения высокой доступности требуется два порта сетевого адаптера.  
  
3.  Купить 2 кабеля FDR InfiniBand двухпортовый карточки или 1 кабель FDR InfiniBand карточки один порт. Кабели FDR InfiniBand подключится загрузка сервера к сети InfiniBand устройства. Длина кабеля зависит от расстояния между загрузка сервера и коммутаторы InfiniBand устройства, в соответствии с вашей средой.  
  
## <a name="Step3"></a>Шаг 3: Подключите сервер к сети InfiniBand  
Для подключения к сети InfiniBand загрузка сервера, выполните следующие действия. Если сервер не использует сети InfiniBand, пропустите этот шаг.  
  
1.  Для монтажа в стойку сервер как можно ближе к устройству, чтобы вы могли подключаться к сети InfiniBand устройства.  
  
2.  Установка сетевого адаптера InfiniBand FDR InfiniBand Mellanox ConnectX-3 в загрузка сервера.  
  
3.  Используйте FDR кабели для подключения к одному из этих двух параметров InfiniBand в стойку устройства первого сетевого адаптера InfiniBand.  
  
4.  Установите и настройте Windows драйвер для сетевого адаптера InfiniBand.  
  
    -   InfiniBand драйверов для Windows разработанный Alliance OpenFabrics, Консорциум InfiniBand поставщиков.  Драйвер завершенных сетевого адаптера InfiniBand. В противном случае драйвер можно загрузить из www.openfabrics.org.  
  
5.  Настройте параметры для сетевых адаптеров InfiniBand и DNS. Инструкции по настройке см. в разделе [настройки сетевых адаптеров InfiniBand](configure-infiniband-network-adapters.md).  
  
## <a name="Step4"></a>Шаг 4: Настройка резервного копирования общей папки  
Резервное копирование сервера PDW доступ через общую папку UNC. Чтобы настроить общую папку:  
  
1.  Создайте папку на сервере резервного копирования для хранения резервных копий.  
  
2.  Создайте общую папку с именем общую папку резервной копии для папки резервного копирования.  
  
3.  Назначение или создайте учетную запись домена Windows в домене клиента, который вы хотите использовать в целях выполнения резервное копирование и восстановление. По соображениям безопасности рекомендуется использовать выделенную учетную запись как пользователя с резервным копированием.  
  
4.  Добавление разрешений для резервного копирования организации общего доступа только доверенным учетным записям и учетную запись домена для резервного копирования могут получить доступ, чтение и запись в общую папку.  
  
5.  Добавьте учетные данные учетной записи домена к PDW.  
  
    Например:  
  
    ```sql  
    EXEC sp_pdw_add_network_credentials '10.192.147.63', 'seattle\david', '********';  
    ```  
  
    Дополнительные сведения см. в разделе эти хранимые процедуры:  
  
    -   [sp_pdw_add_network_credentials](../relational-databases/system-stored-procedures/sp-pdw-add-network-credentials-sql-data-warehouse.md)  
  
    -   [sp_pdw_remove_network_credentials](../relational-databases/system-stored-procedures/sp-pdw-remove-network-credentials-sql-data-warehouse.md)  
  
## <a name="Step5"></a>Шаг 5: Начать резервное копирование данных  
Теперь вы готовы начать резервное копирование данных на сервер резервного копирования.  
  
Чтобы создать резервную копию данных, используйте запрос клиента для подключения к SQL Server PDW и последующей отправки резервного КОПИРОВАНИЯ базы данных или ВОССТАНОВЛЕНИЕ базы данных команды. Использовать ДИСК = предложений, чтобы указать расположение резервной копии сервера и резервного копирования.  
  
> [!IMPORTANT]  
> Не забывайте использовать InfiniBand IP-адрес резервного сервера. В противном случае данные будут скопированы через сеть Ethernet вместо InfiniBand.  
  
Например:  
  
```sql  
BACKUP DATABASE Invoices TO DISK = '\\10.172.14.255\backups\yearly\Invoices2013Full';  
  
RESTORE DATABASE Invoices2013Full  
FROM DISK = '\\10.172.14.255\backups\yearly\Invoices2013Full'  
```  
  
Дополнительные сведения см. в разделе: 
  
-   [РЕЗЕРВНОЕ КОПИРОВАНИЕ БАЗЫ ДАННЫХ](../t-sql/statements/backup-database-parallel-data-warehouse.md)   
  
-   [ВОССТАНОВЛЕНИЕ БАЗЫ ДАННЫХ](../t-sql/statements/restore-database-parallel-data-warehouse.md)  
  
## <a name="Security"></a>Уведомления безопасности  
Резервный сервер не присоединен к частному домену для устройства. Он находится в собственной сети, а не задано отношение доверия между домен и устройства частного домена.  
  
Поскольку резервные копии PDW не хранятся на устройстве, ИТ-специалисты отвечает за управление все аспекты безопасности резервного копирования. Например в нем Управление безопасностью резервных копий данных, безопасность сервера, используемого для хранения резервных копий и безопасность сетевой инфраструктуры, который подключается к APS устройства резервного копирования сервера.  
  
### <a name="manage-network-credentials"></a>Управление учетными данными в сети  
  
Сетевой доступ к каталогу резервного копирования осуществляется по стандартным принципам безопасности при обмене файлами в Windows. Прежде чем выполнять резервное копирование, необходимо создать или указать учетную запись Windows, которая будет использоваться для проверки подлинности PDW в каталог резервного копирования. Эта учетная запись Windows должна иметь разрешения на доступ к каталогу резервного копирования, создание такого каталога и запись в него.  
  
> [!IMPORTANT]  
> Чтобы снизить риски безопасности в работе с данными, рекомендуется назначить одну учетную запись Windows исключительно для выполнения резервного копирования и восстановления. Предоставьте этой учетной записи разрешения исключительно для расположения резервного копирования.  
  
Чтобы сохранить имя пользователя и пароль в PDW, используйте [sp_pdw_add_network_credentials](../relational-databases/system-stored-procedures/sp-pdw-add-network-credentials-sql-data-warehouse.md) хранимой процедуры. PDW диспетчер учетных данных Windows использует для хранения и шифрование имен пользователей и паролей на узел элемента управления и вычислительных узлов. Учетные данные не архивируются с помощью команды BACKUP DATABASE.  
  
Чтобы удалить PDW сетевые учетные данные, используйте [sp_pdw_remove_network_credentials](../relational-databases/system-stored-procedures/sp-pdw-remove-network-credentials-sql-data-warehouse.md) хранимой процедуры.  
  
Чтобы получить список всех сетевых учетных данных, хранящихся в SQL Server PDW, [sys.dm_pdw_network_credentials](../relational-databases/system-dynamic-management-views/sys-dm-pdw-network-credentials-transact-sql.md) динамическое административное представление.  
  
### <a name="secure-communications"></a>Безопасные подключения  
  
Операции на сервере загрузки могут использовать UNC-путь для извлечения данных из вне доверенной внутренней сети. Он в сети или с возможностью влияют разрешения имен можно перехватывать или изменять данные, отправляемые PDW. Это представляет угрозу раскрытия подмены и сведения. Чтобы снизить риск подмены:

- Требуется подпись для соединения. 
- На сервере загрузки задать следующий параметр групповой политики в параметры безопасности\Локальные Политики\параметры безопасности: клиент сети Microsoft: использовать цифровую подпись (всегда): включено.
