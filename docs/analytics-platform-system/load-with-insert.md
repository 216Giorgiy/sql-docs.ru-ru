# <a name="load-data-with-insert"></a>Загрузка данных с помощью вставки

Для загрузки данных в SQL Server Parallel Data Warehouse (PDW) распределенного либо репликацию таблицы можно использовать инструкцию INSERT tsql. Дополнительные сведения об инструкции INSERT см. в разделе [вставить](../t-sql/statements/insert-transact-sql.md). Реплицированные таблицы и все столбцы без распределения распределенные таблицы PDW использует SQL Server для неявного преобразования значения данных, указанных в инструкции, чтобы тип данных целевого столбца. Дополнительные сведения о правилах преобразования данных в SQL Server см. в разделе [преобразования типа данных в SQL](http://msdn.microsoft.com/library/ms191530&#40;v=sql11&#40;.aspx). Однако для столбцов распределения PDW поддерживает только подмножество неявных преобразований, поддерживаемых SQL Server. Таким образом при использовании инструкции INSERT для загрузки данных в столбец распределения источника данных необходимо указать в одном из форматов, определенных в следующих таблицах.  
  
  
## <a name="InsertingLiteralsBinary"></a>Вставьте в двоичные типы литералов  
Следующая таблица определяет допустимый тип литерала, формат и правила преобразования для вставки литеральное значение в столбец распределения типа **двоичных** (*n*) или **varbinary** (*n*).  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Двоичный литерал|0x*hexidecimal_string*<br /><br />Пример: 0x12Ef|Двоичные литералы должны иметь префикс 0 x.<br /><br />Длина источника данных не может превышать число байтов, указанное для типа данных.<br /><br />Если длина источника данных меньше, чем размер **двоичных** тип данных, данные дополняются нулями вправо до размера типа данных.|  
  
## <a name="InsertingLiteralsDateTime"></a>Вставьте литералов в типы даты и времени  
Литералы даты и времени представлены с помощью значения символов в специального формата, заключенных в одинарные кавычки. В следующих таблицах допустимые типы литералов, формат и правила преобразования для вставки в столбец распределения SQL Server PDW типа даты или времени литерала **datetime**, **smalldatetime**, **даты**, **время**, **datetimeoffset**, или **datetime2**.  
  
### <a name="datetime-data-type"></a>Тип данных DateTime  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **datetime**. Любой пустая строка ("") преобразуется в значение по умолчанию "12:00:00.000 1900-01-01". Строки, содержащие только пустые значения ("") вызывают ошибку.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **datetime** формат|«ГГГГ-мм-дд чч [.Ннн]»<br /><br />Пример: "2007-05-08 12:35:29.123"|При вставке значение отсутствует цифр дробной части задается значение 0. Например, литерал "2007-05-08 12:35 ° вставляется как ' 2007-05-08 12:35:00.000".|  
|Размер строкового литерала в **smalldatetime** формат|«ГГГГ-мм-дд чч: мм»<br /><br />Пример: "2007-05-08 12:35"|Секунды и оставшиеся цифры дробной части задается значение 0 при вставке значение.|  
|Размер строкового литерала в **даты** формат|'YYYY-MM-DD'<br /><br />Пример: "2007-05-08"|Значения времени (час, минуты, секунды и доли секунды), присваивается 12:00:00.000 при вставке значение.|  
|Размер строкового литерала в **datetime2** формат|«ГГГГ-мм-дд чч:мм:сс.ннннннн»<br /><br />Пример: "2007-05-08 12:35:29.1234567"|Источник данных не может превышать трех цифр дробной части. Например, литерал "12:35:29.123 2007-05-08" будет вставлен, но значение "2007-05-08 12:35:29.1234567" приводит к ошибке.|  
  
### <a name="smalldatetime-data-type"></a>Тип данных smalldatetime  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **smalldatetime**. Любой пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00". Строки, содержащие только пустые значения ("") вызывают ошибку.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **smalldatetime** формат|«ГГГГ-мм-дд чч: мм» или «Гггг-мм-дд hh:mm:00»<br /><br />Пример: "2007-05-08 12:00" или "2007-05-08 12:00:00"|Источник данных должен иметь значения для года, месяца, даты, час и минуту. Секунд являются необязательными и если он имеется, должно быть присвоено значение 00. Любое другое значение приводит к ошибке.|  
|Размер строкового литерала в **даты** формат|'YYYY-MM-DD'<br /><br />Пример: "2007-05-08"|При вставке значение значения времени (час, минуты, секунды и доли секунды) равным 0.|

### <a name="time-data-type"></a>Тип данных времени  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **время**. Значение по умолчанию «00:00:00.0000» преобразуется любое пустая строка (""). Строки, содержащие только пустые значения ("") вызывают ошибку.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **время** формат|«чч:мм:сс.ннннннн»<br /><br />Пример: «12:35:29.1234567»|Если источник данных имеет больше точность (количество цифр дробной части) чем точность значения типа **время** тип данных, данные дополняются нулями справа. Например значение-литерал «12:35:29.123» добавляются в виде «12:35:29.1230000».<br /><br />Значение, которое имеет большую точность, чем у целевого типа данных отклоняется.|  
  
### <a name="datetimeoffset-data-type"></a>Тип данных DateTimeOffset  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **datetimeoffset** (*n*). По умолчанию используется формат "гггг-мм-дд чч:мм:сс.ннннннн {+ |-} чч: мм". Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00:00.0000000 + 00:00". Строки, содержащие только пустые значения ("") вызывают ошибку. Число цифр дробной части зависит от определения столбца. Например, столбец, определенный как **datetimeoffset** (2) будут иметь две цифры дробной части.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **datetime** формат|«ГГГГ-мм-дд чч [.Ннн]»<br /><br />Пример: "2007-05-08 12:35:29.123"|Отсутствует цифр дробной части и значения смещения задается значение 0 при вставке значение. Например, литерал "2007-05-08 12:35:29.123" добавляются в виде "2007-05-08 12:35:29.1230000 + 00:00".|  
|Размер строкового литерала в **smalldatetime** формат|«ГГГГ-мм-дд чч: мм»<br /><br />Пример: "2007-05-08 12:35"|Секунды, оставшиеся цифры дробной части и значения смещения равным 0 при вставке значение.|  
|Размер строкового литерала в **даты** формат|'YYYY-MM-DD'<br /><br />Пример: "2007-05-08"|При вставке значение значения времени (час, минуты, секунды и доли секунды) равным 0. Например, литерал "2007-05-08' вставляется как ' 2007-05-08 00:00:00.0000000 + 00:00".|  
|Размер строкового литерала в **datetime2** формат|«ГГГГ-мм-дд чч:мм:сс.ннннннн»<br /><br />Пример: "2007-05-08 12:35:29.1234567"|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняются нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
|Размер строкового литерала в **datetimeoffset** формат|"Гггг-мм-дд чч:мм:сс.ннннннн {+&#124;-} чч: мм"<br /><br />Пример: "2007-05-08 12:35:29.1234567 + 12:15 '|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняются нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
  
### <a name="datetime2-data-type"></a>Тип данных datetime2  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **datetime2** (*n*). Форматом по умолчанию является «Гггг-мм-дд чч:мм:сс.ннннннн». Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00:00". Строки, содержащие только пустые значения ("") вызывают ошибку. Число цифр дробной части зависит от определения столбца. Например, столбец, определенный как **datetime2** (2) будут иметь две цифры дробной части.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **datetime** формат|«ГГГГ-мм-дд чч [.Ннн]»<br /><br />Пример: "2007-05-08 12:35:29.123"|Доли секунды, являются необязательными и устанавливаются в значение 0 при вставке значение.<br /><br />Значение, которое имеет больше цифр дробной части, чем у целевого типа данных отклоняется.|

|Размер строкового литерала в **даты** формат|'YYYY-MM-DD'<br /><br />Пример: "2007-05-08"|При вставке значение значения времени (час, минуты, секунды и доли секунды) равным 0. Например, литерал "2007-05-08' вставляется как ' 2007-05-08 12:00:00.0000000".|  
|Размер строкового литерала в **datetime2** формат|«ГГГГ-мм-дд hh:mm:ss:nnnnnnn»<br /><br />Пример: "2007-05-08 12:35:29.1234567"|Если источник данных содержит компоненты даты и времени, которые меньше или равно значению, указанному в **datetime2**(*n*), все данные вставлены; в противном случае возникает ошибка.|

### <a name="money-and-smallmoney-data-types"></a>типов данных Money и smallmoney  
Литеральные значения Money, представляются в виде числа с необязательный десятичный разделитель и префикс в виде знака валюты. Источник данных не может превышать диапазона, разрешенного для заданного типа данных. Например, диапазон для **smallmoney** —-214,748.3648 214748,3647 и допустимый диапазон для **money** является -922,337,203,685,477.5808 до 922,337,203,685,477.5807. В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **money** или **smallmoney**.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Размер строкового литерала в **целое** формат|«nnnnnnnn»<br /><br />Пример: "123433"|При вставке значение отсутствует цифр после десятичной запятой имеют значение 0. Например литерал "12345" добавляются в виде 12345.0000.|  
|Размер строкового литерала в **десятичное** формат|«nnnnnn.nnnnn»<br /><br />Пример: «123344.34455"|Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения. Например значение «123344.34455"добавляются в виде 123344.3446.|  
|Размер строкового литерала в **money** формат|«$nnnnnn.nnnn»<br /><br />Пример: «$123456.7890"|Значение вставляется символ валюты.<br /><br />Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения.|  
|Целочисленный литерал|nnnnnnnn<br /><br />Пример: 123433|При вставке значение отсутствует цифр после десятичной запятой имеют значение 0. Например литерал 12345 вставляется как 12345.0000.|  
|Десятичный литерал|nnnnnn.NNNNN<br /><br />Пример: 123344.34455|Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения. Например значение 123344.34455 вставляется как 123344.3446.|  
|Денежный литерал|$nnnnnn.nnnn<br /><br />Пример: $123456.7890|Значение вставляется символ валюты.<br /><br />Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения.|  
  
## <a name="InsertLiteralsString"></a>Вставка строковых литералов  
Следующие таблицы определяют допустимые форматы и правила преобразования для вставки литеральное значение в столбец SQL Server PDW, который использует тип string.  
  
### <a name="char-varchar-nchar-and-nvarchar-data-types"></a>типы данных char, varchar, nchar и nvarchar  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения типа **char**, **varchar**, **nchar** и **nvarchar**. Длина источника данных не может превышать размер, указанный для типа данных. Если длина источника данных меньше, чем размер **char** или **nchar** тип данных, данные дополняются пробелами вправо до размера типа данных.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Строковый литерал|Формат: «символ строка»<br /><br />Пример: «abc»| Нет|  
|Юникод строковый литерал|Формата: Строка N'character "<br /><br />Пример: N'abc'|  Нет |  
|Целочисленный литерал|Формат: nnnnnnnnnnn<br /><br />Пример: 321312313123| Нет |  
|Десятичный литерал|Формат: nnnnnn.nnnnnnn<br /><br />Пример: 12344.34455| Нет |  
|Денежный литерал|Формат: $nnnnnn.nnnnn<br /><br />Пример: $123456.99|Символ валюты не вставляется значение. Чтобы вставить символ валюты, вставьте значение в виде строкового литерала. Это будет соответствовать формату **dwloader** средства, которое обрабатывает каждый литерал как строковый литерал.<br /><br />Не допускаются запятые.<br /><br />Если количество цифр после десятичной запятой превышает 2, значение округляется до ближайшего значения. Например значение 123.946789 вставляется как 123.95.<br /><br />При использовании функции CONVERT для вставки литералы money разрешается только стиль по умолчанию 0 (без запятых и 2 цифры после десятичной запятой).|
