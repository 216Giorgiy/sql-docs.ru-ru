# <a name="data-type-conversion-rules-for-dwloader---parallel-data-warehouse"></a>Тип данных правила преобразования для dwloader - Parallel Data Warehouse
В этом разделе описываются форматы входных данных и неявное преобразование типов данных, [dwloader командной строки загрузчика](dwloader.md) поддерживает при загрузке данных в PDW. Неявные преобразования данных возникают, когда входные данные не соответствует типу данных в целевой таблице SQL Server PDW. Используйте эти сведения при разработке процесс загрузки, чтобы данные будет загрузить в SQL Server PDW.  
   
  
## <a name="InsertBinaryTypes"></a>Вставка двоичные типы литералов  
Следующая таблица определяет допустимый тип литерала, формат и правила преобразования для загрузки литеральное значение в столбец типа SQL Server PDW **двоичных** (*n*) или **varbinary** (*n*).  
  
|Тип входных данных|Примеры входных данных|Преобразование в binary или varbinary, тип данных|  
|-------------------|-----------------------|-----------------------------------------------|  
|Двоичный литерал|[0x]*hexidecimal_string*<br /><br />Пример: 12Ef или 0x12Ef|Префикс 0 x является необязательным.<br /><br />Длина источника данных не может превышать число байтов, указанное для типа данных.<br /><br />Если длина источника данных меньше, чем размер **двоичных** тип данных, данные дополняются нулями вправо до размера типа данных.|  
  
## <a name="InsertDateTimeTypes"></a>Вставка литералов в типы даты и времени  
Литералы даты и времени могут быть представлены с помощью строковых литералов в специального формата, заключенных в одинарные кавычки. В следующих таблицах допустимые типы литералов, формат и правила преобразования для загрузки в столбец типа даты или времени литерала **datetime**, **smalldatetime**, **даты**, **время**, **datetimeoffset**, или **datetime2**. Таблицы определяют формат по умолчанию для заданного типа данных. Другие форматы, которые могут быть указаны определены в разделе [форматов даты и времени](#DateFormats). Литералы даты и времени не может содержать начальные или конечные пробелы. **Дата**, **smalldatetime**, и значения null не удается загрузить в режиме фиксированную ширину.  
  
### <a name="datetime-data-type"></a>Тип данных datetime  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **datetime**. Пустая строка ("") преобразуется в значение по умолчанию "12:00:00.000 1900-01-01". Строки, содержащие только пустые значения ("") вызывают ошибку.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных datetime|  
|-------------------|-----------------------|------------------------------------|  
|Размер строкового литерала в **datetime** формат|«гггг мм дд чч [.fff]»<br /><br />Пример: "2007-05-08 12:35:29.123"|При вставке значение отсутствует цифр дробной части задается значение 0. Например, литерал "2007-05-08 12:35 ° вставляется как ' 2007-05-08 12:35:00.000".|  
|Размер строкового литерала в **smalldatetime** формат|«гггг мм дд чч: мм»<br /><br />Пример: "2007-05-08 12:35"|Секунды и оставшиеся цифры дробной части задается значение 0 при вставке значение.|  
|Размер строкового литерала в **даты** формат|«гггг мм дд»<br /><br />Пример: "2007-05-08"|Значения времени (час, минуты, секунды и доли секунды), присваивается 12:00:00.000 при вставке значение.|  
|Размер строкового литерала в **datetime2** формат|«yyyy-MM-dd hh:mm:ss.fffffff»<br /><br />Пример: "2007-05-08 12:35:29.1234567"|Источник данных не может превышать трех цифр дробной части. Например, литерал "2007-05-08 12:35:29.123" будут вставлены значения, кроме "12:35:29.1234567 2007-05-8" приводит к ошибке.|  
  
### <a name="smalldatetime-data-type"></a>Тип данных smalldatetime  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **smalldatetime**. Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00". Строки, содержащие только пустые значения ("") вызывают ошибку.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных smalldatetime|  
|-------------------|-----------------------|-----------------------------------------|  
|Размер строкового литерала в **smalldatetime** формат|«гггг мм дд чч: мм» или «гггг мм дд чч»<br /><br />Пример: "2007-05-08 12:00" или "2007-05-08 12:00:15"|Источник данных должен иметь значения для года, месяца, даты, час и минуту. Секунд являются необязательными и если он имеется, должно быть присвоено значение 00. Любое другое значение приводит к ошибке.<br /><br />Секунд не являются обязательными. При загрузке данных в столбец типа smalldatetime, dwloader округлит секунды и доли секунды. Например 1999-01-05 20:10:35.123 загрузит как 01-05 20:11.|  
|Размер строкового литерала в **даты** формат|«гггг мм дд»<br /><br />Пример: "2007-05-08"|При вставке значение значения времени (час, минуты, секунды и доли секунды) равным 0.|

### <a name="datetimeoffset-data-type"></a>Тип данных DateTimeOffset  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **datetimeoffset** (*n*). По умолчанию используется формат "гггг мм дд hh:mm:ss.fffffff {+ |-} чч: мм". Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00:00.0000000 + 00:00". Строки, содержащие только пустые значения ("") вызывают ошибку. Число цифр дробной части зависит от определения столбца. Например, столбец, определенный как **datetimeoffset** (2) будут иметь две цифры дробной части.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных datetimeoffset|  
|-------------------|-----------------------|------------------------------------------|  
|Размер строкового литерала в **datetime** формат|«гггг мм дд чч [.fff]»<br /><br />Пример: "2007-05-08 12:35:29.123"|Отсутствует цифр дробной части и значения смещения задается значение 0 при вставке значение. Например, литерал "2007-05-08 12:35:29.123" добавляются в виде "2007-05-08 12:35:29.1230000 + 00:00".|  
|Размер строкового литерала в **smalldatetime** формат|«гггг мм дд чч: мм»<br /><br />Пример: "2007-05-08 12:35"|Секунды, оставшиеся цифры дробной части и значения смещения равным 0 при вставке значение.|  
|Размер строкового литерала в **даты** формат|«гггг мм дд»<br /><br />Пример: "2007-05-08"|При вставке значение значения времени (час, минуты, секунды и доли секунды) равным 0. Например, литерал "2007-05-08' вставляется как ' 2007-05-08 00:00:00.0000000 + 00:00".|  
|Размер строкового литерала в **datetime2** формат|«yyyy-MM-dd hh:mm:ss.fffffff»<br /><br />Пример: "2007-05-08 12:35:29.1234567"|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняются нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
|Размер строкового литерала в **datetimeoffset** формат|"yyyy-MM-dd hh:mm:ss.fffffff {+&#124;-} чч: мм"<br /><br />Пример: "2007-05-08 12:35:29.1234567 + 12:15 '|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняются нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|

Число цифр дробной части зависит от определения столбца. Например, столбец, определенный как **datetime2** (2) будут иметь две цифры дробной части.

-   Буквы в квадратных скобках необязательны.

-   Буквы zzz обозначают смещение часового пояса для текущего часового пояса системы в формате {+ |-} HH:ss].

|Целочисленный литерал|fffffffn<br /><br />Пример: 1 или 321|Значение меньше 0 или больше 1 преобразуется в 1. Например значения -123 и 123 в преобразуются в 1.|
