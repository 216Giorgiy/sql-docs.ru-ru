# <a name="acquire-and-configure-a-loading-server-for-parallel-data-warehouse"></a>Приобретать и настраивать загрузка сервера для параллельного хранилища данных
В этой статье описывается приобретать и настраивать загрузка сервера как система Windows не является специализированным по отправке загружаемых данных параллельного хранилища данных (PDW).  
  
## <a name="Basics"></a>Основы  
Загрузка сервера:  
  
-   Не быть одиночного сервера. Можно загрузить одновременно с несколькими серверами загрузки.  
  
-   Предоставляется и управляется ИТ команды. Уже установлена на сервере или серверах, которые можно использовать для загрузки данных в PDW.  
  
-   Расположен в стойку не является специализированным и не может быть помещен в устройстве Analytics Platform System.  
  
-   Подключен к устройству через сети InfiniBand устройства или через Ethernet. Для повышения производительности рекомендуется использовать InfiniBand.  
  
-   Находится в ваш домен клиента, не домен устройства. Нет отношения доверия между доменом клиента и домен устройства.  
  
## <a name="Step1"></a>Шаг 1: Определение требований к емкости  
Загрузка системы могут быть разработаны как один или несколько серверов загрузки, выполняющие параллельных загрузок. Каждый сервер загрузки не имеет должен использоваться только для загрузки, при условии, что он будет обрабатывать требования к производительности и хранения рабочей нагрузки.  
  
Требования к системе для загрузки сервера почти полностью зависят от рабочей нагрузкой. Используйте [загрузке листа планирования емкости сервера](loading-server-capacity-planning-worksheet.md) для определения требований к емкости.  
  
## <a name="Step2"></a>Шаг 2: Получение элемента  
Теперь, когда вы лучше понять требования к емкости, можно запланировать, серверами и сетевыми компонентами, которые потребуются для покупки или подготовки. Плана покупки с учетом перечисленных ниже требования и приобрести сервера или подготовить существующий сервер.  
  
### <a name="R"></a>Требования к программному обеспечению  
Поддерживаемые операционные системы  
  
-   Windows Server 2012 или Windows Server 2012 R2. Эти операционные системы требуют FDR сетевого адаптера.  
  
-   Windows Server 2008 R2. Этой операционной системе требуется сетевой адаптер DDR.  
  
Для использования средства командной строки загрузке dwloader сервер должен использовать языковой стандарт EN-US. dwloader не поддерживает другими языковыми стандартами.  
  
### <a name="networking-requirements-for-windows-server-2012-and-windows-server-2012-r2"></a>Требования к сети для Windows Server 2012 и Windows Server 2012 R2  
Несмотря на то, что не требуется для загрузки, InfiniBand — это тип рекомендуемые подключение для загрузки серверов. Для повышения производительности рекомендуется используйте Windows Server 2012 или Windows Server 2012 R2 и сетевого адаптера FDR InfiniBand для подключения к сети InfiniBand устройства загрузка сервера.  
  
Подготовка для подключения к Windows Server 2012 или Windows Server 2012 R2 InfiniBand:  
  
1.  Планирование для установки в стойку как можно ближе к устройству, чтобы вы могли подключаться к устройству переключается InfiniBand. Дополнительные сведения от технологий Mellanox о InfiniBand см. в документе [введение в InfiniBand](http://www.mellanox.com/pdf/whitepapers/IB_Intro_WP_190.pdf).  
  
2.  Приобрести одним или двумя порта сетевого адаптера InfiniBand FDR Mellanox ConnectX-3. Мы рекомендуем сетевой адаптер с двумя портами для обеспечения отказоустойчивости при передаче данных. Для обеспечения высокой доступности требуется два порта сетевого адаптера.  
  
3.  Купить 2 кабеля FDR InfiniBand двухпортовый карточки или 1 кабель FDR InfiniBand карточки один порт. Кабели FDR InfiniBand подключится загрузка сервера к сети InfiniBand устройства. Длина кабеля зависит от расстояния между загрузка сервера и коммутаторы InfiniBand устройства, в соответствии с вашей средой.  
  
## <a name="Step3"></a>Шаг 3: Подключите сервер к сети InfiniBand  
Для подключения к сети InfiniBand загрузка сервера, выполните следующие действия. Если сервер не использует сети InfiniBand, пропустите этот шаг.  
  
1.  Для монтажа в стойку сервер как можно ближе к устройству, чтобы вы могли подключаться к сети InfiniBand устройства.  
  
2.  Установка сетевого адаптера InfiniBand FDR InfiniBand Mellanox ConnectX-3 в загрузка сервера.  
  
3.  Используйте FDR кабели для подключения к одному из этих двух параметров InfiniBand в стойку устройства первого сетевого адаптера InfiniBand.  
  
4.  Установите и настройте Windows драйвер для сетевого адаптера InfiniBand.  
  
    -   InfiniBand драйверов для Windows разработанный Alliance OpenFabrics, Консорциум InfiniBand поставщиков.  Драйвер завершенных сетевого адаптера InfiniBand. В противном случае драйвер можно загрузить из www.openfabrics.org.  
  
5.  Настройте параметры для сетевых адаптеров InfiniBand и DNS. Инструкции по настройке см. в разделе [сетевых адаптеров InfiniBand Настройка](configure-infiniband-network-adapters.md).

## <a name="performance"></a>Производительность  
Для лучшего производительность на Windows Server 2012 и более поздних загрузки Включите мгновенную инициализацию файлов, чтобы при перезаписи данных, операционная система не перезаписывает существующие данные нулями. Если это представляет угрозу безопасности, так как по-прежнему существует предыдущих данных на дисках, необходимо отключить быструю инициализацию файлов.  
  
## <a name="Security"></a>Уведомления безопасности  
Поскольку данные для загрузки не хранятся на устройстве, ИТ-специалисты отвечает за управление все аспекты безопасности данных для загрузки. Например это включает в себя управление безопасность данных для загрузки, безопасность сервера, используемого для хранения загружает и безопасность сетевая инфраструктура, которая подключается к SQL Server PDW appliance загрузка сервера.  
  
> [!IMPORTANT]  
> Это особенно важно для обеспечения безопасности каждого загрузка сервера, который будет использовать средство командной строки загрузке dwloader. При загрузке dwloader данных он сначала выполняет проверку подлинности с узла управления, а затем после успешной проверки подлинности он перемещает данные из загрузка сервера непосредственно для вычислительных узлов по каналам данных. Проверка сертификата не возникает при наличии встряхивания между каждой загрузка сервера и каждом вычислительном узле. При этом остается уязвимым для потенциальных атак в середине по каждому каналу данных при загрузке вычислительных узлов. Эти атаки может привести к поддельных данных или раскрытие информации.  
  
Чтобы снизить риски безопасности, с данными, рекомендуется следующие:  
  
-   Назначить одну учетную запись Windows, который используется исключительно для загрузки данных в PDW. Разрешить этой учетной записи необходимо предоставить разрешения на место загрузки и нигде больше.  
  
-   Назначьте один PDW пользователя с разрешениями для загрузки данных. В зависимости от требований безопасности имеется один конкретного пользователя на одну базу данных.  
  
-   Операции на сервере загрузки могут принимать UNC-путь, из которого для извлечения данных из вне доверенной внутренней сети. И он в сети или с возможностью влияют разрешения имен можно перехватывать или изменять данные, отправляемые в SQL Server PDW. Это представляет угрозу раскрытия подмены и сведения. Изменение данных должен быть минимизирован требуя подписи для соединения. Чтобы уменьшить этот риск, установите следующий параметр групповой политики **Параметры безопасности\Локальные Политики\параметры безопасности** на загрузка сервера: **клиента сети Microsoft: использовать цифровую подпись (всегда): Включен**  
  
-   Отключите мгновенную инициализацию файлов на Windows Server 2012 и более поздних. Это компромисс между производительностью и безопасностью, как указано в разделе «производительность». Необходимо решить, что лучше в соответствии с требованиями безопасности.
