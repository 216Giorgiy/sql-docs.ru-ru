# <a name="configure-windows-server-update-services-wsus"></a>Настройка Windows Server Update Services (WSUS)
Эти инструкции пошаговые инструкции по использованию мастера настройки Windows Server Update Services (WSUS) для настройки служб WSUS для Analytics Platform System. Необходимо настроить службы WSUS, перед применением обновлений программного обеспечения с устройством. На виртуальной машине VMM устройства уже установлена служба WSUS.  
  
Дополнительные сведения о настройке служб WSUS см. в разделе [руководство по установке служб WSUS пошаговое](http://go.microsoft.com/fwlink/?LinkId=202417) на веб-сайт WSUS. После настройки WSUS, в разделе [загрузки и применения обновлений Майкрософт &#40; Система платформы аналитики &#41; ](download-and-apply-microsoft-updates.md) для запуска обновления.  
  
> [!WARNING]  
> Если возникают ошибки во время этого процесса настройки, остановите и обратитесь за помощью в службу поддержки. Не пропускать ошибки и продолжения процесса после получения ошибки.  
  
## <a name="before-you-begin"></a>Перед началом  
Чтобы настроить службы WSUS, необходимо:  
  
-   У имени входа данные учетной записи администратора система платформы аналитики appliance домена.  
  
-   Наличие разрешений на доступ к имени входа Analytics Platform System **консоли администрирования** и просмотр сведений о состоянии устройства.  
  
-   Знаете IP-адрес вышестоящий сервер WSUS, если вы планируете синхронизировать обновления с вышестоящего сервера WSUS вместо синхронизации обновлений непосредственно из центра обновления Майкрософт. Убедитесь, что вышестоящий сервер WSUS допускает анонимные соединения и поддерживает SSL.  
  
-   Знаете IP-адрес прокси-сервера, если устройства будут использовать прокси-сервер для доступа к вышестоящему серверу или Центра обновления Майкрософт.  
  
-   В большинстве случаев WSUS требуется для доступа к серверам вне устройства. Для поддержки этого сценария использования системы DNS платформы Analytics можно настроить для поддержки пересылки внешнее имя, которое позволит Analytics Platform System узлов и виртуальных машин (ВМ) для использования внешнего DNS-серверы для разрешения имен вне устройство. Дополнительные сведения см. в разделе [использовать DNS-сервер пересылки для разрешения DNS-имена устройств не &#40; Система платформы аналитики &#41; ](use-a-dns-forwarder-to-resolve-non-appliance-dns-names.md).  
  
## <a name="to-configure-windows-server-update-services-wsus"></a>Настройка службы Windows Server Update Services (WSUS)  
  
1.  Войдите на **консоли администрирования**. На **состояние устройства** убедитесь, что **кластера** и **сети** столбцы показывают зеленый (или **NA**) для всех узлов. Проверьте индикаторы состояния для всех узлов на **состояние устройства**.  
  
    -   Можно безопасно продолжать работу с зеленым или индикаторы н/д.  
  
    -   Оцените некритические ошибки (желтый) предупреждения. В некоторых случаях предупреждающие сообщения не будет блокировать обновлений. Если имеется ошибка некритические диска тома, не на диске C:\, можете перейти к следующему шагу до разрешения ошибки тома диска.  
  
    -   Большинство красный индикаторы необходимо устранить перед продолжением. В случае сбоев диска, используйте **предупреждения консоли администратора** страницу, чтобы проверить, происходит сбой не более одного диска в течение каждого сервера или массивом SAN. Если ошибка не более одного диска в пределах каждого сервера или массивом SAN, можно продолжить следующий шаг перед исправлением сбоев диска. Убедитесь, что в службу поддержки Майкрософт для устранения сбоев дисков, как можно быстрее.  
  
2.  Войдите на виртуальной машине VMM с правами администратора домена устройства.  
  
3.  Запуск мастера настройки.  
  
    #### <a name="to-launch-the-configuration-wizard"></a>Чтобы запустить мастер настройки  
  
    1.  В **панели мониторинга диспетчера сервера**на **средства** меню, нажмите кнопку **Windows Server Update Services**.  
  
    2.  В левой области **Update Services** окна, разверните узел сервера управления виртуальными машинами (***appliance_domain*- VMM**) и нажмите кнопку **Параметры**.  
  
    3.  В **параметры** области, нажмите кнопку **мастер настройки сервера WSUS** для запуска мастера настройки.  
  
        ![Меню панели мониторинга диспетчера сервера](./media/configure-windows-server-update-services-wsus/WSUS_Wiz0.png "WSUS_Wiz0")  
  
    4.  Если при первом запуске мастера WSUS, возможно, потребуется настроить каталог для хранения обновлений. `C:\wsus`является подходящее место; Однако может предоставить другой путь.  
  
        ![Путь WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz1.png "WSUS_Wiz1")  
  
    5.  Просмотрите **перед началом** список элементов для выполнения до завершения работы мастера.  
  
        ![WSUS перед началом](./media/configure-windows-server-update-services-wsus/WSUS_Wiz2.png "WSUS_Wiz2")  
  
    6.  На **присоединиться к программе улучшения Microsoft Update** выберите **Да, я хочу присоединиться к программе улучшения Microsoft Update**, а затем нажмите кнопку **Далее**.  
  
        ![Программа улучшения WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz3.png "WSUS_Wiz3")  
  
    Теперь вы увидите **Выбор вышестоящего сервера** страницы. Следующий снимок экрана является отправной точкой мастера настройки.  
  
    ![Синхронизация с вышестоящим сервером WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4a.png "WSUS_Wiz4a")  
  
4.  Выбор вышестоящего сервера.  
  
    На **Выбор вышестоящего сервера** страницы мастера настройки WSUS будет выбирать, как WSUS на этом узле управления виртуальной машины будут подключаться к вышестоящему серверу для получения обновлений программного обеспечения. Два возможны для синхронизации с вышестоящим сервером [центра обновления Майкрософт](http://go.microsoft.com/fwlink/?LinkId=133349) или синхронизировать обновления с другим сервером Windows Server Update Services.  
  
    #### <a name="to-update-by-using-microsoft-update"></a>Для обновления с помощью центра обновления Майкрософт  
  
    1.  Если выбран для синхронизации с центром обновления Майкрософт, необходимо внести изменения в **Выбор вышестоящего сервера** страницы. Нажмите кнопку **Далее**.  
  
        ![Синхронизация с вышестоящим сервером WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4a.png "WSUS_Wiz4a")  
  
    #### <a name="to-update-from-another-wsus-server"></a>Для обновления с другого сервера WSUS  
  
    1.  Если выбран для синхронизации с источником, отличные от центра обновления Майкрософт (сервер), укажите сервер (введите IP-адрес) и порт, на котором данный сервер будет связываться с вышестоящим сервером.  
  
        ![Синхронизация с вышестоящим сервером WSUS из WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4b.png "WSUS_Wiz4b")  
  
    2.  Чтобы использовать протокол Secure Sockets Layer (SSL), выберите **использовать SSL при синхронизации данных об обновлениях** флажок. В этом случае для синхронизации серверы будут использовать порт 443.  
  
        ![Синхронизация с вышестоящим сервером WSUS из WSUS SSL](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4c.png "WSUS_Wiz4c")  
  
    3.  Если сервер-реплику, выберите **это реплика вышестоящего сервера** флажок. Можно выбрать оба **использовать SSL при синхронизации данных об обновлениях** и **это реплика вышестоящего сервера**.  
  
        ![Реплика вышестоящего сервера WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4d.png "WSUS_Wiz4d")  
  
    4.  На этом этапе вы закончили с вышестоящим сервером конфигурации. Нажмите кнопку **Далее**, или выберите **прокси-сервера укажите** в левой области навигации.  
  
5.  Укажите прокси-сервера.  
  
    Если этот сервер требует прокси-сервер для доступа к центра обновления Майкрософт или другой вышестоящего сервера, можно настроить параметры прокси-сервера в противном случае нажмите кнопку **Далее**.  
  
    ![Прокси-сервер WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz5a.png "WSUS_Wiz5a")  
  
    #### <a name="to-configure-proxy-server-settings"></a>Чтобы настроить параметры прокси-сервера  
  
    1.  На **Настройка прокси-сервера** страницы мастера настройки выберите **использовать прокси-сервер при синхронизации** флажок, а затем введите прокси-сервера IP-адрес сервера (а не имени) и номер порта (порт 80 по по умолчанию) в соответствующих полях.  
  
    2.  Если вы хотите подключиться к прокси-серверу с помощью определенных учетных данных пользователя, выберите **использовать учетные данные пользователя для подключения к прокси-серверу** флажок, а затем введите имя пользователя, домен и пароль пользователя в соответствующем поля. Если вы хотите включить обычную проверку подлинности для пользователя, подключающегося к прокси-серверу, установите **разрешить обычную проверку подлинности (пароль передается открытым текстом)** флажок.  
  
        ![Учетные данные прокси-сервера WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz5b.png "WSUS_Wiz5b")  
  
    3.  На этом этапе вы закончили с конфигурацию прокси-сервера. Нажмите кнопку **Далее** для перехода на следующую страницу, где можно приступать к настройке процесса синхронизации.  
  
6.  Запустите подключение.  
  
    ![Прокси-сервер WSUS начала подключения](./media/configure-windows-server-update-services-wsus/WSUS_Wiz6.png "WSUS_Wiz6")  
  
    Нажмите кнопку **начала подключения**.  
  
    После успешного соединения щелкните **Далее** для перехода на следующую страницу, где можно выбрать языки интерфейса.  
  
7.  Выбор языков.  
  
    Выберите **загружать обновления только для этих языков**.  
  
    Выберите **английского**, а затем нажмите кнопку **Далее**.  
  
    ![Выбор языков](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChooseLanguages.png "SQL_Server_PDW_WSUSChooseLanguages")  
  
8.  Выберите продукты.  
  
    > [!NOTE]  
    > При использовании с вышестоящим сервером, может не появиться возможность выбор продуктов. Если этот параметр недоступен, пропустите этот шаг.  
  
    Снимите флажки для всех выбранных обновлений.  
  
    Выберите **SQL Server 2014**, **Windows Server 2012 R2**, и **System Center 2012 R2 — диспетчер виртуальных машин**, а затем нажмите кнопку **Далее**.  
  
9. Выберите классификации.  
  
    > [!NOTE]  
    > При использовании с вышестоящим сервером, может не появиться возможность выбор классов.  Если этот параметр недоступен, пропустите этот шаг.  
  
    Отмените выбор всех ранее выбранных обновлений.  
  
    Выберите **критические обновления** и **обновления для системы безопасности** обновлений, которые будут синхронизированы для устройства система платформы аналитики и нажмите кнопку **Далее**.  
  
    ![Выберите классификации](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChooseClassifications.png "SQL_Server_PDW_WSUSChooseClassifications")  
  
10. Настройте расписание синхронизации.  
  
    Выберите **вручную синхронизировать**, а затем нажмите кнопку **Далее**.  
  
    ![Задайте расписание синхронизации](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSyncSchedule.png "SQL_Server_PDW_WSUSSyncSchedule")  
  
11. Запустить первоначальную синхронизацию.  
  
    Выберите **запустить первоначальную синхронизацию**, нажмите кнопку **Далее**.  
  
12. "Готово".  
  
    Нажмите кнопку **Готово**.  
  
## <a name="bkmk_WSUSGroup"></a>Группы серверов устройстве в службах WSUS  
После настройки WSUS для Analytics Platform System, следующим шагом является для группирования серверов устройства. Добавляя все серверы, устройства в группу, WSUS будет иметь возможность обновления программного обеспечения на всех серверах в устройстве.  
  
> [!NOTE]  
> Система WSUS предназначена для асинхронного выполнения. Инициирование действия не всегда приводит к появлению немедленно обновить. Таким образом может потребоваться подождать некоторое время, пока компьютеры будут отображаться в диалоговых окнах служб WSUS. Под управлением `setup.exe /action=ReportMicrosoftUpdateClientStatus /DomainAdminPassword="<password>"` команды, описанные в конце этого раздела [загрузки и применения обновлений Майкрософт &#40; Система платформы аналитики &#41; ](download-and-apply-microsoft-updates.md) может помочь обновление диалоговые окна.  
  
#### <a name="to-group-the-appliance-servers"></a>Для группировки серверов устройства  
  
1.  Откройте консоль служб WSUS, щелкните правой кнопкой мыши **все компьютеры** и нажмите кнопку **добавить группу компьютеров**.  
  
    ![Добавьте группу компьютеров. ] (./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSAddComputerGroup.png "SQL_Server_PDW_WSUSAddComputerGroup")  
  
2.  Введите имя для группы компьютеров «APS» и нажмите кнопку **добавить**.  
  
    ![Введите имя для новой группы компьютеров. ] (./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSpecifyGroupName.png "SQL_Server_PDW_WSUSSpecifyGroupName")  
  
3.  Нажмите кнопку **все компьютеры** снова, измените состояние в **состояние** раскрывающееся меню для **любой**и нажмите кнопку **обновление**. Может потребоваться развернуть **все компьютеры** , щелкнув его в структуре дерева в левой части экрана для отображения в новую группу только что добавлен.  
  
    ![Измените состояние на любой и щелкните "Обновить". ] (./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeStatusToAny.png "SQL_Server_PDW_WSUSChangeStatusToAny")  
  
4.  Выберите все компьютеры, которые являются частью устройства, щелкните правой кнопкой мыши и выберите команду **изменить членство**.  
  
    ![Измените членство всех компьютеров PDW. ] (./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeMembership.png "SQL_Server_PDW_WSUSChangeMembership")  
  
5.  Выберите группу компьютеров, создать, щелкнув флажок и нажмите кнопку **ОК**.  
  
    ![Членства в группе компьютеров набор](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSetComputerGroupMembership.png "SQL_Server_PDW_WSUSSetComputerGroupMembership")  
  
6.  Выберите новую группу компьютеров, изменить его **состояние** для **любой**и нажмите кнопку **обновление**. Теперь все компьютеры должны назначены этой группе и правой панели. Обычно безопасно продолжать, если узлы показывать предупреждения, такие как **этот узел еще не сообщил состояние**.  
  
    ![Измените состояние на любой и щелкните "Обновить". ] (./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeStatusAnyRefresh.png "SQL_Server_PDW_WSUSChangeStatusAnyRefresh")
