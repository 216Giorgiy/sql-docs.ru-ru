# <a name="read-scale-availability-groups"></a>Группы доступности для чтения и масштабирования
[!INCLUDE[tsql-appliesto-ssvnxt-xxxx-xxxx-xxx](../../../includes/tsql-appliesto-ssvnxt-xxxx-xxxx-xxx.md)]

Помимо объединения лучших в своем классе возможностей обеспечения высокого уровня доступности для SQL Server, группа доступности представляет собой комплексное решение, предлагающее интегрированные решения по масштабируемости. В обычном приложении базы данных существует несколько клиентов, выполняющих различные типы рабочих нагрузок, что иногда может привести к возникновению узких мест из-за нехватки ресурсов. Можно высвободить ресурсы и добиться более высокой пропускной способности для рабочей нагрузки OLTP или обеспечить более высокую производительность и масштабируемость в доступных только для чтения рабочих нагрузках. Это можно сделать за счет использования самой быстрой технологии репликации для SQL Server — создания группы реплицируемых баз данных для разгрузки рабочих решений по аналитике и отчетности на реплики только для чтения. 

С помощью групп доступности можно настроить одну или несколько вторичных реплик для поддержки доступа только для чтения к базам данных-получателям.

Клиентские приложения с рабочими нагрузками в области аналитики и отчетности могут напрямую подключаться к базам данных-получателям. Или клиент может настроить список маршрутизации только для чтения и подключить его к первичной реплике, которая будет пересылать запрос на подключение к каждой из вторичных реплик в списке маршрутизации методом циклического перебора.

## <a name="read-scale-availability-groups-without-cluster"></a>Группы доступности для чтения и масштабирования без кластера

В [!INCLUDE[sssql15-md](..\..\..\includes\sssql15-md.md)] и более ранних версиях для всех групп доступности требовался кластер. Кластер обеспечивал непрерывность бизнеса — высокую доступность и аварийное восстановление (HADR). Кроме того, можно было настраивать вторичные реплики для операций чтения. Если обеспечение высокой доступности не было целью, настройка и работа кластера были связаны с множеством операционных издержек. В SQL Server 2017 появились группы доступности для чтения и масштабирования без кластера. 

Если бизнес-требование заключается в необходимости экономить ресурсы для критически важных рабочих нагрузок, выполняющихся на сервере-источнике, пользователь теперь может использовать маршрутизацию только для чтения или напрямую подключаться к вторичным репликам для чтения независимо от интеграции с какой-либо технологией кластеризации. Эти новые возможности доступны для SQL Server 2017, запущенного на платформах Windows и Linux.

>[!IMPORTANT]
>Они не имеют отношения к настройке высокого уровня доступности. Инфраструктура для отслеживания и координации обнаружения сбоев и автоматического перехода на другой ресурс отсутствует. Пользователям, которым необходимы возможности HADR, следует использовать диспетчер кластера (WSFC в Windows или Pacemaker в Linux). 

## <a name="use-distributed-availability-groups-for-geographic-read-scale"></a>Использование распределенных групп доступности для географического чтения и масштабирования

Географически разделенные решения могут реализовывать решения для чтения и масштабирования с помощью распределенных групп доступности. Это позволяет разгрузить рабочие процессы чтения из первичной реплики на доступные для чтения вторичные реплики для сайтов, которые находятся ближе к источнику рабочих нагрузок чтения. При этом не только снижается использование ресурсов на сервере-источнике, но и повышается пропускная способность чтения за счет сокращения сетевых задержек и использования выделенных ресурсов.

Одна распределенная группа доступности может иметь до 17 доступных для чтения вторичных реплик. Для повышения масштабируемости реализуется последовательное соединение нескольких групп доступности и еще больше увеличивается количество доступных для чтения реплик. Можно также развернуть две распределенные группы доступности из одной группы доступности для обеспечения низкой задержки чтения в географически распределенных средах.
