# <a name="operation-of-non-parameterized-commands"></a>Операции без параметров команд
Команды, без параметров, выполняются все команды поставщика и **наборы записей** создаются во время выполнения команды. Если команда выполняется синхронно, все **наборы записей** будет полностью заполнен. Если был выбран режим асинхронное заполнение, заполненных состояние **наборы записей** будет зависеть от режима заполнения и размера **наборы записей**.  
  
 Например *родительской команды* может вернуть **записей** заказчиков компании из таблицы Customers и *команда дочерних* может вернуть **Записей** заказов для всех клиентов из таблицы Orders.  
  
```  
SHAPE {SELECT * FROM Customers}   
   APPEND ({SELECT * FROM Orders} AS chapOrders   
   RELATE customerID TO customerID)  
```  
  
 Для непараметризованных родительско дочерних отношений, в каждой родительской и дочерней **записей** объект должен быть столбец в том, чтобы связать их. Столбцы имеют имена в предложении RELATE *родительский столбец* первый и затем *дочерний столбец*. Столбцы могут иметь разные имена в соответствующих им **записей** объектов, но должен ссылаться на те же сведения, чтобы указать отношение может применяться. Например **клиентов** и **записей заказов** объекты могут иметь поля customerID. Так как членство в дочерней **набора записей** определяется с помощью команды поставщика дочернего **набора записей** может содержать появлению потерянных строк. Эти потерянные строки являются недоступными без дополнительных изменения формы.  
  
 Формирование данных добавляет столбец к родительскому **записей**. Значения в столбце главы — ссылки на строки в дочерней **записей**, который удовлетворяет предложении RELATE. То же значение является в *родительский столбец* данной родительской строки, как в *дочерний столбец* всех строк дочерних главы. При использовании нескольких предложения TO в том же предложении RELATE они неявно объединяются с помощью оператора AND. Если родительские столбцы в предложении RELATE не представляют ключ к родительскому **записей**, и строка один дочерний элемент может иметь несколько родительских строк.  
  
 При доступе к ссылку в столбце главе, ADO автоматически извлекает **записей** представлены по ссылке. Обратите внимание, что в команде без параметров, несмотря на то что все дочерние **записей** была извлечена, глава показан только подмножество строк.  
  
 Если добавленный столбец не имеет *главе псевдоним*, имя будет создан для него автоматически. Объект [поле](../../../ado/reference/ado-api/field-object.md) объекта для столбца будут добавлены к **записей** объекта [поля](../../../ado/reference/ado-api/fields-collection-ado.md) коллекции, а также тип данных будет **adChapter**.  
  
 Сведения о навигации по иерархической **набора записей**, в разделе [доступ к строкам в иерархических записей](../../../ado/guide/data/accessing-rows-in-a-hierarchical-recordset.md).
