# <a name="data-mining-extensions-dmx-syntax-conventions"></a>Соглашения о синтаксисе расширений интеллектуального анализа данных
[!INCLUDE[ssas-appliesto-sqlas](../includes/ssas-appliesto-sqlas.md)]

  В справочной документации расширений интеллектуального анализа данных (DMX) в [!INCLUDE[msCoName](../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../includes/ssnoversion-md.md)] [!INCLUDE[ssASnoversion](../includes/ssasnoversion-md.md)] использует следующие правила для описания языка расширений интеллектуального анализа данных.  
  
|Обозначение|Использование|  
|----------------|-----------|  
|**полужирный**|Ключевые слова расширений интеллектуального анализа данных и текст, которые необходимо вводить точно так, как указано.|  
|*курсив*|Пользовательские аргументы синтаксиса расширений интеллектуального анализа данных.|  
|&#124; (вертикальная черта)|Служит для разделения элементов синтаксиса внутри квадратных или фигурных скобок. Можно выбрать только один из элементов.|  
|`[ ]` (квадратные скобки)|Содержат необязательные элементы синтаксиса. Скобки не вводятся.|  
|{ } (фигурные скобки)|Содержат обязательные элементы синтаксиса. Скобки набирать не следует.|  
|, ...|Указывает на то, что предшествующий запятой элемент может повторяться любое количество раз. Элементы разделяются запятыми.|  
|\<Метка >:: =|Имя синтаксического блока. Это обозначение используется для группирования и маркировки сегментов с длинным синтаксисом или элемента синтаксиса, который может использоваться в нескольких местах в пределах одной инструкции. Каждое место, в котором используется этот синтаксический блок, обозначается меткой, заключенной в угловые скобки, такие как \<метка >.|
